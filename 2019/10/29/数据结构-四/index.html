背景介绍
Java的运行环境就是基于栈结构的虚拟机。
网络浏览器将用户最近访问过的地址组织成一个栈，新访问的页面存放在栈顶，“后退”按钮可以沿着相反方向返回之前访问的页面。
文本编辑器可以将用户的编辑操作记录在一个栈中，“撤销”功能可以返回上一个编辑记录。

调度银行或医院的服务窗口，管理轮耕的田地和轮伐的森林，要使用到队列。
多进程共享CPU时间，多用户共享打印机，需要借助队列结构实现优化的分配。

栈

ADT接口

入栈与出栈
栈（stack）是存放数据对象的一种特殊的容器，其中的数据元素按照线性的逻辑次序排列。
栈结构仅支持对对象在某一特定端进行插入或删除等操作。（只能从某一端插入新元素，同时只能从这一端删除已有的元素）
禁止操作的一端，称作盲端。
栈中可操作的一端称为栈顶（stack top），而无法直接操作的盲端称为栈底（stack bottom）。

栈ADT支持的操作接口

操作接口 功能
size() 报告栈的规模
empty() 判断栈是否为空
push(e) 将e插入栈顶
pop() 删除栈顶对象
top() 引用栈顶对象

栈的特点
后进先出（last-in-first-out,LIFO）：更晚出栈的元素，为更早入栈者，更晚入栈者应更早出栈。

栈与递归

函数调用与栈密切相关，具体有：调用与被调用之间的关系，调用的返回，维护所有函数的实例…

函数调用栈

调用栈（call stack）/执行栈（execution stack）：

  1. 跟踪同一程序的所有函数；
  2. 记录函数之间的相互调用关系；
  3. 保证调用实例执行完成后可以准确返回。

帧（frame）：

调用栈的基本单位。

功能：

  1. 记录函数实例的返回地址（return address）；
  2. 函数实例的局部变量；
  3. 函数实例的传入参数。

各帧之间的关系：

各帧对应于尚未返回的调用实例（当前的活跃函数实例（active function instance））。
栈底的帧对应于入口主函数main()，其弹出栈意味着整个程序的结束。

调用栈自底到顶对应于到函数的最新活跃实例的一条调用路径。

函数、函数实例与帧：
在程序中，一个函数可能有多个函数实例，每个活跃函数实例对应于帧中的一帧（frame）。
递归就是同一函数的多个实例对应于栈中的多个帧。

逆序输出

特征

  1. 解答以线性序列的形式给出。
  2. 该序列都是依逆序计算输出。
  3. 输入和输出的规模不确定。

进制转换
对于一个n进制数，转化为m进制数。
N(n) -> N(m)
设N(n) = d_m (n^m) + … + d_1 (n^1) + d_0 (n^0)
设N_i(n) = d_m (n^{i-m}) + … + d_i * (n^{0})
从而有N(n) = N_0(n); d_i = N_i(n) % n; N_{i+1}(n) = N_{i}(n) / n。

迭代嵌套

具有自相似性的问题多可以嵌套地递归描述。

两个迭代嵌套的问题：

  1. 栈混洗
  2. 括号匹配

栈混洗

描述：
有三个栈A、B和S，其中A中含有n个元素，自顶向下分别为{a_1, a_2, …, a_n}。B和S为空。若只允许通过弹出A的顶元素并压入B或S的方式（B.push(S.pop(),S.push(S.pop())），当所有元素都转移到栈B时，B的元素构成的序列称为一个栈混洗（stack permutation）。

要点要求：实施出栈操作的栈不得为空。

模型抽象：对于空栈而言，在由push和pop构成的操作序列中，任意一个前缀中push的数量不少于pop。

括号匹配

描述：左括号与右括号的数量相同和次序相反。

递归实现

算法复杂度
最坏情况是当左、右括号匹配，则需要递归的次数为括号对的数量，每个递归的复杂度为O(n),故总的时间复杂度为O(n^2)(设序列全是括号)。

延迟缓冲

栈具有数据缓冲区的功能，当数据在需要时弹出处理，否则入栈等待。

实例：在计算算术表达式时，要按照规定的优先级依次处理，而非一直是“从左到右”。
