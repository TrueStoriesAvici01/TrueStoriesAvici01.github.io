Java编程思想(一)：对象导论

Posted on 2019-10-29

对象导论

抽象过程

面向对象方式

实质：程序通过添加新类型的对象使自身使用于某个特定问题。
在阅读描述解决方案的代码的同时，也是在阅读问题的描述。
OOP 是根据问题来描述问题，而不是解决方案的计算机来描述问题。

面向对象的五个基本特性

  1. 万物皆对象：将对象视为变量，它可以存储数据，同时在自身上执行操作。
  2. 程序是对象的集合，彼此通过消息来相互通信：发消息是对某个特定对象的方法的调用请求。
  3. 每个对象可由其他对象构成：通过创建包含现有对象的包的方式来创建新类型的对象。
  4. 每个对象都拥有其类型：每个对象都是某个类(class)的一个实例(instance)。不同类的重要区别是发送消息的类型。
  5. 每个特定类型的所有对象可以接受相同的消息：消息是区别不同的类型的重要标志。

对象具有状态，行为和标识：

  1. 状态：拥有内部数据，给出对象的状态。
  2. 方法：产生方法。
  3. 表示：每个对象具有可以唯一的和其他对象区分开。(在内存中有唯一的地址)

每个对象都有一个接口

对程序进行抽象，发现所有的程序都在重复进行两个行为：

  1. 创建某一类型的变量（对象，实例）：这个过程称为实例化。
  2. 操作这些变量（发消息，请求）

那么如何判断一个变量是否为有效变量：即能够实现上面的第二个行为。我们在创建对象的时候通过接口(interface)定义了对于某个对象所能进行的请求，而构成对象的各种特性和行为决定了接口。

对象与类的关系：

  1. 对象和类是所属关系，特定的对象属于某类，而某类又可以包含多个相同类型对象。
  2. 对象和类是具体和抽象的关系，类是对于具有相同特征（数据元素）和相同行为（功能*）的对象的抽象，对象是某个类的具体实现。类中定义了最具代表性的方面，而对象在满足这些类所定义的方面的同时又有自己的特点，用来区别同类的不同对象。
    *一个典型的实例是：人类中定义了最具代表性的方面，如：眼睛，鼻子，行动，等…而到某个具体对象，在继承类中的共同方面的同时又有自己的具有辨识的方面，如：同样有眼睛，有的人眼睛大，有的人眼睛小…等

每个对象都提供服务

将对象看成“服务提供者”，这样不同的对象提供不同的“服务”，而整个程序就是一个“服务的集合”。程序向用户提供服务，而程序通过调用一系列对象将其所提供的“服务”进行整合而实现。

基于对象为“服务提供者”这一观点，对象的设计中存在难点如下：

通过将问题看成对象，将对象提供的“服务”进行分解，由不同的对象提供的“服务”协作完成

  1. 分解太细，需要的对象变多，费时费力，单个对象提供的功能单一，内聚性差
  2. 分解太粗，单个对象要提供的功能复杂、冗余，无法处理灵活多变的问题

被隐藏的具体实现

程序员可以分为两种：

  1. 类创建者：创建新数据类型的程序员
  2. 客户端程序员：使用数据类型的类消费者

通过访问控制来区别两类程序员：

  1. 帮助类使用者快速了解哪些可以忽略，哪些值得重视
  2. 降低两类程序员的紧耦合关系，这样大家可以通过统一规范的接口来相互协调，各司其职，互不影响。

类中有四类访问控制表示(访问指定词access specifier)

  1. public: 任何人可用
  2. default:(不指定时默认) 同一个包的不同类可用
  3. protected: 本类和继承的类可用
  4. private: 仅本类可用

复用具体实现

创建成员变量: 将一个类的一个对象置于某个新的类中。

组合(composition): 新的类是由任意数量、类型的其他对象以任意方式实现所需功能组成。

继承

为什么要继承:

通过继承，减少不同类之间的相似功能，消除冗余。

继承过程的关系：

源类(基类，超类，父类): 被继承者，包含其所有导出类所共享的特性和行为。
导出类(继承类，子类): 继承者，表示核心概念可以被实现的各种不同的方式。

继承中基类和导出类的两种关系:

  1. 在导出类中添加新方法。(关键词extends)
  2. 改变现有基类的方法(覆盖(overriding))。

多态

通常把一个对象不当成其所属的特定类型看待，而是将其当成其基类的对象来看待。

解决不同对象的相同方法的不同实现问题:
通过后期绑定：向对象发消息时，知道运行时具体的代码段才确定，编译器只保证被调用方法的存在以及对调用参数和返回值进行类型检查。

单根继承结构

所有对象最终都继承自单一的基类(Object)

优势

  1. 在每个对象中执行某些基本操作
  2. 所有对象可以在堆上创建
  3. 简化参数传递
  4. 垃圾回收器实现简单

容器

背景

在不知道解决特定问题时需要的对象个数，或者其存活时间，也就不知道如何以及用多大的空间存储这些对象。
容器: 可以在任何需要的时候扩充自己以容纳置于其中的所有东西。

Q: 单个类型的容器可以满足所有需求，为什么还要设计不同类型？
A:

  1. 不同容器提供不同类型的接口和外部行为
  2. 不同的容器对于某些操作具有不同的效率

参数化类型

Object类型的容器可以存储任何东西，但是在其中置入对象引用时，需要将其向上转型为Object类型，从而丢失其原有类型。在取回时，获得的是一个Object对象的引用，需要进行类型转换。

参数化类型: 一种编译器可以自动定制作用于特定类型上的类，从而编译器可以定制只接纳和取出特定类型的对象。

范式: Java中的参数化类型。一对尖括号，中间包含类型消息。

对象的创建和生命期

Java 在堆(heap)的内存池中动态地创建对象，直到运行时才知道需要对象的个数，生命期以及具体类型。在需要一个对象时直接在堆中创建，存储空间是在运行时被动态管理。

异常处理

在Java中，异常是一种对象，它从出错点被“抛出”，并被专门设计用来处理特定类型错误的相应的异常处理器“捕获”。
异常处理就像与程序正常执行路径并行的，在出错时执行另一个路径，它是一条完全分离的执行路径，不会干扰正常执行的代码。

并发编程

对于大量的问题，可以把问题切分成多个可独立运行的部分，提高程序的响应能力。
在程序中，这些独立运行的部分称为线程，多线程同时运行称为“并发”。

并发的一个隐患
共享资源可能出现问题：可能约束某些资源不能同时被访问。
一个简单的解决方案：在某个线程使用时将该资源锁定，完成任务后，释放资源锁。
