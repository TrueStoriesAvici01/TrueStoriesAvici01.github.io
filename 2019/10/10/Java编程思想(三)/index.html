Java编程思想(三)：操作符

Posted on 2019-10-10 Edited on 2019-10-28

Java操作符

操作符接受一个或多个参数，并生成一个新值。
有些操作符可能改变操作数自身的值。
“=”，“==”，“!=”能够操作所有对象，其他的操作符只能操作“基本类型”。
String类支持“+”和“+=”。

优先级

先乘除后加减。
使用括号明确计算顺序。
“+”可能涉及“字符串转换”：当一个String后面有“+”，其后又有非String类型元素，会试图将非String类型的元素转换为String。

赋值

“=”：用右边的值复制给左边的值。

右值可以是任何常数、变量或者表达式。
左值必须为明确的、已命名的变量。

基本数据类型的赋值就是将一个地方的内容复制到另一个地方。a = b后，修改a的值不会影响b。

赋值的为对象时有所不同，对一个对象进行操作时，操作的是对象的引用，赋值操作则是将“引用”复制一份，两者都是指向相同的对象，修改其中一个都会对另外一个产生相同的影响。（别名现象）

避免“别名问题”：使用对象中的域进行赋值，而不是使用对象进行赋值。

方法调用中的别名问题

在方法中传入的参数为对象时，方法会作用于该对象，从而改变对象的实际值（传进来的是对象的引用，而非对象的复制）

示例

算术操作符

  1. 包含加（+），减（-），乘（*），除（/）以及取模操作符（%从整数除法中产生余数）。
  2. 支持运算与赋值同时操作。

一元减号（-）和一元加号（+）：一元减号用于转变数据的符号，一元加号仅将较小类型的操作数提升为int。

自动递增和递减

  1. ++：增加一个单位
  2. - -：减少一个单位
  1. 前缀式
    ++a, - -a：操作符位于变量的前面。先执行运算，再生成值。
  2. 后缀式
    a++, a- -：操作符位于变量的后面。先生成值，再执行运算。

关系操作符

关系操作符有：

  1. 小于（<）
  2. 大于（>）
  3. 小于或等于（<=）
  4. 大于或等于（>=）
  5. 等于（==）
  6. 不等于（!=）

产生一个boolean值：关系成立则为true，否则为false。
=, !=适合所有基本数据类型，其他不适合boolean类型。

测试对象的等价性

若 == 两侧的是对象，则实际上比较的是对象的引用，除非使用赋值=创建两个对象，否则相同class创建的两个对象的引用是不同的。
使用“==” 或 “equal” 来对比相同类的两个对象时，其比较的都是引用，而创建的两个引用是不同的，所以为false。只有使用“=”来生成新的对象时，两个对象的引用才是一样的。

逻辑操作符

  1. 与（&&）
  2. 或（||）
  3. 非（！）
    生成的是boolean值：ture或者false

“与”，“或”，“非”操作只可应用于布尔值。
不可将一个非布尔值当作布尔值在逻辑表达式中使用。

短路

使用逻辑操作符时，一旦能够明确无误地确定整个表达式的值（&&时遇到false，||遇到true），就不再计算表达式剩下的部分。

直接常量

后缀字符标志着其类型：

  1. 大写（或小写）的L代表long。
  2. 大写（或小写）的F代表float。
  3. 大写（或小写）的D代表double。
  4. 十六进制数以前缀0x（或0X）表示。

指数计数法

使用e（E）表示“10的幂次”。

按位操作符

  1. 按位“与”（&）
  2. 按位“或”（|）
  3. 按位“异或”（^）
  4. 按位“非”（~）（只对一个操作符操作）

可与=连用：
&=
|=
^=
没有~=

移位操作符

只可处理整数类型

  1. 左移位操作符（<<）：将操作数向左移动指定的位数（低位补0）
  2. “有符号”右移位操作符（>>）：向右移动操作数，符号为正，高位插0；符号为负，高位插1。
  3. “无符号”右移位操作符（>>>）：高位只插0。

char、byte或short类型的数值：
先转成int类型，得到结果为int类型，只有低5位有效。
long类型：
结果为long类型，只有低6位有效。

移位与等号组合使用
先进行移位操作，再将得到的结果赋给左边的变量。

注:进行“无符号”右移位结合赋值符号时，当对于byte或short类型的值处理时，会先转换为int类型，再进行移位，然后被截断，赋值给原来的类型，可能产生-1的情况。

三元操作符 if-else

表达式为：boolean-exp ? value0 : value1
若boolean-exp为true，就计算value0，否则就计算value1。

字符串操作符 + 和 +=

在Java中的一个特殊用途：

连接不同的字符串。

如果表达式以一个字符串开头，那么后续所有操作数都必须时字符串型。（编译器会自动将字符序列转化为字符串类型）

使用操作符时常犯的错误

  1. while (x = y){...}，实际上是想测试两个数x与y是否相等while (x == y){...}。
  2. 把按位与（&）、或（|）当成逻辑与（&&）、或（||）使用。

类型转换操作符

显式地进行类型的转换，或者在不能自动转换的时候进行强制类型转换。

方法

将需要的数据类型置于圆括号内，放在要进行类型转换的值的左边。

两种类型转换

  1. 窄化转换（narrowing conversion）：强制进行显式类型转换。
  2. 扩展转换（widening conversion）：不必进行显式类型转换。

注：

  1. 允许任何基本数据类型转化为基本数据类型（布尔类型除外）。
  2. boolean类型不允许进行任何类型转换。
  3. “类”数据类型不允许进行数据类型转换。

截尾和舍入
一般的，将浮点数转换为整型值时，对该数值进行“截尾”：即，直接舍弃小数部分。
如果需要舍入的结果，需要使用java.lang.Math中的round()方法。

提升
通常情况下，较小的数据类型与较大的数据类型进行运算时，较小的数据类型会被转换为较大的数据类型。

表达式中出现的最大的数据类型决定了表达式最终结果的数据类型。

Java没有sizeof

Java中所有的数据类型在所有机器中的大小都相同，而sizeof()是考虑到不同机器的数据类型的位数不同而设计的，所以Java中不存在sizeof()。

操作符小结

  1. 布尔类型值只能进行赋值（false或true），对其进行测试，不能相加或其他操作。
  2. 对于char、byte、short，复合赋值不需要进行类型转换。
