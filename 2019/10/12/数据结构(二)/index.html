    数据结构
    数据项之间的相互联系及作用（或某种逻辑次序）。

    分类

    1. 线性结构
    2. 半线性结构
    3. 非线性结构

    线性结构：各线性项按照一个线性次序构成一个整体。
    示例

    序列（sequence）：

    1. 向量（vector）：所有数据项的物理存放位置与其逻辑次序一致。逻辑次序称为秩（rank）。
    2. 列表（list）：逻辑上相邻的数据项在物理上未必相邻。采用间接定址的方式通过封装后的位置（position）相互引用。

    从数组到向量

    数组

    定义

    若集合S由n个元素组成，且各元素之间具有一个线性次序，则可将它们存放于起始于地址A、物理地址连续的一段存储空间。

    数组A[]中的每个元素都唯一对应于一个下标编号，一般编号从0开始。

    Notations

    1. 前驱（predecessor）：对于 0 <= i < j < n, A[i]都是A[j]的前驱。
    2. 后继（successor）：同理，A[j]是A[i]的后继。
    3. 直接前驱（intermediate predecesor）：A[i-1]为A[i]的直接前驱。
    4. 直接后继（intermediate successor）：A[i]为A[i-1]的直接后继。
    5. 前缀（prefix）：任一元素的所有前驱构成其前缀。
    6. 后缀（suffix）：所有的后继构成其后缀。

    访问数组元素
    设数组A[]存放空间的起始地址为A，且每个元素占用s个单位的空间，则元素A[i]对应的物理地址为：A + i*s。

    线性数组（linear array）：元素的物理地址与其下标之间满足线性关系。

    向量

    向量（vector）
    由具有线性次序的一组元素构成的集合V = {v0, v1, …, vn-1}，其中的元素分别由秩相互区分。

    秩（rank）：元素e的前驱元素共计r个，则其秩为r。
    循秩访问（call-by-rank）：通过秩r唯一确定对应的元素。

    ADT接口

    操作接口 功能 使用对象
    size() 元素总数 向量
    get(r) 获取秩为r的元素
    put(r, e) 用e替换秩为r元素的值
    inset(r, e) 将e插入秩为r位置，其后元素依次后移
    remove(r) 删除秩为r的元素，返回该元素中原存放的对象
    disorderd() 判断所有元素是否按照非降排序
    sort() 调整各元素的位置，使之按照非降序排列
    find(e) 查找等于e且秩最大的元素
    search(e) 查找目标元素e，返回不大于e且秩最大的元素 有序向量

    uniquify() 剔除重复元素 有序向量
    traverse() 遍历向量并统一处理所有元素

    过程

    动态空间管理

    静态空间管理

    静态空间管理策略
    内部数组所占用物理空间的容量，再向量的生命期内不允许调整。（容易导致上溢overflow）。

    装填因子（load factor）
    向量实际规模与其内部数组容量的比值（_size/_capacity）(保证装填因子不超过1，也不接近0)

    可扩充向量

    扩充向量（extendable vector）
    若内部数组仍有空余，则操作正常进行。一旦空间耗尽，就动态地扩大内部数组的容量。

    方法
    申请一个容量更大的数组，将原数组中的成员搬迁到新的空间。原数组占用的空间，需要及时的释放。

    分摊分析

    时间成本
    每次扩容，需要花费O(n)的时间。随着向量的规模增大，需要扩容的概率在降低。平均而言，时间成本不高。

    分摊复杂度

    1. 分摊运行时间（amortized running time）：足够多次连续操作，将其所消耗的时间，分摊至所有的操作，平均单次操作的时间成本。
    2. 平均运行时间/期望运行时间（average running time/expected running time）：按照某种假定的概率分布，对各种情况下所执行时间的加权平均。

    O(1)分摊时间
    设向量的初始规模为N，数组的初始容量为N，连续进行n次insert()操作。

    size(n) = 连续插入n个元素后向量的规模
    capacity(n) = 连续插入n个元素后数组的容量
    T(n) = 连续插入n个元素而花费的时间
    则 size(n) = N + n
    有 size(n) <= capacity(n) < 2·size(n)
    则 capacity(n) = Θ(n)
    总共做过Θ(log2 n)次扩容，消耗的时间：
    T(n) = 2N + 4N … < 2·capacity(n) = Θ(n)
    分摊到连续n次操作，单次操作所需的时间为O(1)。

    其他扩容策略
    追加固定数目的单元。
    最坏情况下，分摊时间复杂度为Ω(n)。

    缩容

    向量的实际规模远小于内部数组的容量。

    实现


    当空间利用率降至阈值之下时，即申请一个容量减半的新数组，将原数组中的元素逐一迁移过去。

    判等器与比较器

    1. 判等器
      判断两个对象是否相等，“对比”操作。
    2. 比较器
      判断两个对象的相对大小，“比较”操作。

    推广至可比较或可比对的任何数据类型
    方法：在定义对应的数据类型时，通过重载“<”和“==”等操作符，给出大小和相等的关系的具体定义及其判别方法。

    当元素本身就是指向其他对象的指针时，往往关注的是其指向对象的大小，而不是指针的数值（指向对象的物理地址）。

    无序查找

    查找与数据对象e相等的元素，此类仅支持对比，未必支持比较的向量，称为无序向量（unsorted vector）。

    顺序查找
    顺序查找（sequential search）：依次逐个对比的查找方法。
    从末位元素自后向前，逐个取出与目标元素e对比，相等则直接返回，否则直到检查完所有元素后结束。

    实现

    1. 存在多个命中元素时，只返回秩最大的元素。
    2. 一旦命中，立即返回，避免不必要的比对。
    3. hi--而不是--hi保证到达对比结束的最后一个元素时，直接结束，不会出现index为 -1 的错误。

    复杂度
    最坏情况：时间复杂度O(n)
    最好情况：时间复杂度O(1)

    输入敏感的（input sensitive）：对于规模相同、内部组成不同的输入，渐进运行时间有本质区别的算法。

    插入

    实现

    复杂度
    时间消耗主要在后继元素的后移，与后缀的长度成正比。
    当插入为最后一个位置，则时间复杂度为O(1)
    若插入的位置等概率分布，平均时间复杂度为O(n)

    删除

    删除操作重载有两个接口：

    1. remove(lo, hi)：用以删除区间[lo, hi)内的元素
    2. remove(r)：用以删除秩为r的单个元素

    区间删除:remove(lo, hi)

    单元素删除:remove(r)

    复杂度
    remove(lo, hi)：所需时间取决于后缀元素的数目，与被删区间宽度无关。时间复杂度线性正比于后缀的长度，总体为O(n)

    remove(r)：与删除的元素位置有关，越靠后时间越短，最好O(1)，最坏O(n)

    错误与意外处理
    输入的参数必须满足 0 <= lo <= hi <= _size

    唯一化

    实现

    正确性
    正确性是由不变性保证的：当前的前缀中各元素彼此互异。
    对于当前的元素有两种情况：

    1. 与前缀相同，删去，前缀保持互异性。
    2. 与前缀不同，加入前缀中，保持互异性。

    复杂度

    1. while()循环最坏情况下需要执行n-1次，时间复杂度为O(n)；
    2. find()和remove()分别与前缀的长度和后缀的长度成正比，则两种总的时间复杂度为O(n)；
    3. 总的时间复杂度为O(n^2)。

    功能
    将向量作为一个整体，对其中所有元素实施某种统一的操作。

    实现

    复杂度
    遍历所需的时间线性正比于所指定的基本操作与数组的长度，若单个元素的操作时间复杂度为O(1)，则总的时间复杂度为O(n)。

    有序向量

    有序向量（sorted vector）：向量S[0, n)中的所有元素不仅按线性次序存放，而且其数值大小也按照次序单调分布。

    比较器

    先决条件：
    各元素之间能够比较大小。

    有序性甄别

    有序性带来某些操作的更快实现。

    可以考虑先判别当前向量是否有序，再依此采用更加高效的算法。

    实现
    顺序扫描整个向量，逐个比较每一对相邻元素。

    唯一化

    消除重复元素

    正确性
    有序向量中的重复元素必然前后紧邻，可以从前之后逐个删去相邻元素中的重复元素，只要相邻元素都不相同，再加上有序性，保证整个向量没有重复元素。

    复杂度
    最坏情况下，正比于向量元素的个数，循环的时间复杂度为O(n)，而单次循环中remove()的时间复杂度为O(n)，故总的时间复杂度为O(n^2)。

    改进思路
    对于多个重复元素，每次删除一个重复元素，调用remove()一次，后缀仅移动一个单位，可以一次移动多个单位，减少调用remove()的次数。

    解决：以区间为单位成批删除相邻的多个重复元素，再将后缀统一向前大跨度迁移。

    实现

    复杂度
    while()循环的时间复杂度为O(n)，即从头到尾遍历一次。
    每次循环中的时间复杂度为O(1)，即将一个元素复制到另一个地方。
    所以总的时间复杂度为O(n)。

    查找

    二分查找

    减而治之

    1. 以任一元素为界，将区间分为三个部分，“<”，“=”，“<”。
    2. 将目标元素与中间元素比较，从而判断其可能出现的区间。
    3. 重复多次，直到找到目标元素的位置。

    实现

    复杂度
    每次迭代之后，如果没有找到目标元素，则新的区间减少到原区间的1/2，所以最多经过(log2 (hi - lo))次后，算法结束，则迭代的次数为logn，即时间复杂度为O(logn)。
    而每次迭代只要比较中间元素与目标元素的大小，即单次迭代的时间复杂度为O(1)。总的时间复杂度为O(logn)。

    查找长度
    时间复杂度只要取决于元素的大小比较操作的次数，即查找长度（search length）。（通过最好、最坏和平均情况考察查找算法的总性能）

    1. 成功查找长度
      平均查找长度为O(1.5logn)
    2. 失败查找长度
      平均失败查找长度O(1.5logn)

    Fibonacci查找

    按照黄金分割比来确定前后区间分割位置，设向量长度为n=fib(k) - 1。
    以fib(k-1) - 1 作为前后子向量的切分点，前后子向量的长度为：fib(k-1)-1 和 fib(k-2)-1 = (fib(k) - 1)-(fib(k-1)-1)-1

    实现

    复杂度
    平均查找长度为O(logΦ 2·log2 n) = O(1.44logn)

    二分查找（版本B）

    在切分点处仅做一个元素比较，若目标元素小于切分点元素，则在前一部分区间[lo, mi)查找，否则在后一区间[mi, hi)查找。

    实现

    复杂度
    整个算法的复杂度为：O(logn)。

    新的要求

    1. 当目标元素重复出现时，只能＂随机＂返回其中之一，取决于算法的分支策略和向量组成。
    2. 要求：

      在出现重复元素时：返回序号在最后的一个元素。
      查找失败时：返回一个不大于（不小于）目标元素的最后一个元素位置。

    二分查找（版本C）

    实现

    正确性
    每次查找中，A[lo, mi)中的元素都不大于A[mi]；A[mi , hi)中的元素都大于e，从而使得每次e都落在A[lo, hi)之间。
    最终结束时，lo = hi。此时A[lo - 1]不然不大于e，而A[lo]不然大于e，故A[lo - 1]是不大于e的最后一个元素。

    下界

    复杂度下界
    任一问题在最坏情况下的最低计算成本，为该问题的复杂度下界（lower bound）。一旦算法达到这一下界，即意味着已是最坏情况下最优的（worst-case optimal）。

    基于比较的分支
    用节点表示算法过程中不同的状态，用有向线段表示不同状态之间的相互转换。

    比较树（comparison tree）

    • 每一内部节点各对应于一次对比操作；
    • 内部节点的左右分支，分别对应于在两种对比结果下的执行方向；
    • 叶节点（根到叶节点的路径）对应于算法某次执行的完整过程及输出；
    • 算法的每一运行过程都对应于从根到某一叶节点的路径。

    算法所有可能的执行过程都可表示为一颗比较树，此算法称为基于比较式算法（comparison-based algorithm, CBA式算法）。

    估计下界
    对于CBA式算法A，设CT(A)为与之对应的一颗比较树。
    算法A每一次运行所需的时间都取决于其对应叶节点到根节点的距离（叶节点的深度）。
    算法A在最坏情况下的运行时间，取决于比较树中所有叶节点的最大深度（称为该树的高度，h(CT(A)))。
    渐进意义下，算法的时间复杂度应不低于Ω(h(CT(A)))。
    估计算法时间复杂度下界等价于求比较树的最小高度。
    只需要知道树中所含叶节点（可能的输出结果）的数目。

    对于一个高度为h的二叉树，叶节点的数目不可能多于2^h。故对于数据结果不少于N种，其叶节点不可能少于N个，树高h不可能低于log2 N。

    对于n个元素的排序问题，可能的输出共有N = n!种，比较树为三叉树（相等，比较大、小），即每个内部节点都有三个分支，对应的比较树的高度为：h >= ceil(log3 (n!)) = ceil(log3 e×ln(n!)) = Ω(nlogn)。

    重复元素与稳定性
    稳定性（stability）是算法对于重复元素的处理效果。

    归并排序

    归并排序是通过反复调用二路归并（2-way merge）算法来实现的。
    二路归并：将两个有序序列合并成为一个有序序列。
    每次迭代中，只需要比较两个待归并向量的首元素，将小者取出并追加至输出向量的末尾，而该向量在原向量中的后继则成为新的首元素。直到某一向量为空。再将另一个非空向量整体接至输出向量的末尾。

    二路归并接口的实现

    归并时间
    二路归并的渐进时间成本取决于其中循环迭代的总次数。
    每次迭代总长加一，而最多增加所有待归并的元素，即时间复杂度不超过O(n)。

    排序时间
    设归并排序算法处理长度为n的向量所需的时间记为T(n)。
    对于长度为n的向量，需要递归地对长度各为n/2的两个子向量做递归排序，并进行一次二路递归，递推关系：T(n) = 2T(n/2) + O(n)，
    且有T(1) = O(1)：可得T(n) = O(n*log    n)。

    从递归图示可以看出：每层递归实例耗时为O(n)，而层数共O(log2 n)，则总的时间复杂度：O(nlongn)。
