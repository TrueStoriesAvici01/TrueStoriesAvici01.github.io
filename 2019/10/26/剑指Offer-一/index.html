剑指Offer(一)：面试流程

Posted on 2019-10-26 Edited on 2019-10-28

面试官谈面试

  1. 初级程序员：考查算法与数据结构；高级程序员：关注专业技能和项目经验。
  2. 准备：公司近况，项目情况有所了解；准备好合适的问题问面试官。
  3. 面试：不要紧张；了解清楚所解决的问题；做一些整体的设计与规划；不要马上提交，测试用例检查代码；

面试的3种形式

流程
电话面试 -> （共享桌面远程面试） -> 现场面试

电话面试

两种形式：

  1. 预约时间
  2. 突然袭击

建议

  1. 投简历后的一两个星期内，保持手机至少能够连续通话1小时。
  2. 不要长期处于嘈杂环境中。
  3. 尽量用形象化的语言把细节说清楚。
  4. 不要不懂装懂，答非所问，要大胆向面试官多提问。

共享桌面远程面试

注意要点

  1. 思考清楚再开始编码。（想清楚解决问题的思路，算法的时间、空间复杂度，有哪些特殊情况需要处理）
  2. 良好的代码命名和缩进对齐习惯。（一目了然的变量和函数名，合理的缩进和括号对齐）
  3. 能够进行单元测试。（先写单元测试用例，再写解决问题的函数）
  4. 出错时的表现。（设置断点、单步跟踪、查看内存、分析调用栈）

现场面试

要点

  1. 规划好路线并估算出行时间。（预留半小时应对突发情况）
  2. 准备好得体的衣服。（不必要穿正装）
  3. 注意面试邀请函里的面试流程。（准备缓解面试疲劳的东西）
  4. 准备几个问题。（提前准备好问题）

面试的3个环节

一轮面试通常分为三个环节：
行为面试 -> 技术面试 -> 应聘者提问

行为面试环节

时间：面试开始的 5 ～ 10 分钟。
特点：性格；了解简历的项目经历；简短的自我介绍（主要学习，工作经历， 30s～1min）；

三个注意点

应聘者的项目经验
STAR模型：

  • Situation（简短的项目背景）：项目规模，软件功能，目标用户；
  • Task（自己完成的任务）：参与（只是写了几行代码）；负责（总体框架设计、核心算法、团队合作）。
  • Action（做了哪些工作，怎么做的）：什么工具，哪些平台，哪些技术…
  • Result（贡献）：完成多少功能，性能提升多少，修改多少bug。

一些值得注意的问题

  • 项目中遇到的最大问题是什么，怎么解决？
  • 从项目中学到什么？
  • 与团队成员有什么冲突，怎么解决？

应聘者掌握的技能

技能掌握的三种程度

  1. 了解：只是看过书，没有做过实际项目。
  2. 熟悉：实际项目中使用较长时间，通过查阅相关文档可以独立解决大部分问题。
  3. 精通：使用得心应手。尽量少使用

为什么跳槽

注意：

  • 不要抱怨，或流露负面情绪。
  • 避免四种原因：老板太苛刻；同事难相处；加班太频繁；工资太低。

建议回答：希望寻找一份更有挑战的工作，对于面试公司的方向感兴趣。

技术面试环节

关注五种素质

  • 基础知识扎实全面：编程语言、数据结构、算法。
  • 能写出正确、完整、鲁棒的高质量代码
  • 能思路清晰地分析、解决复杂问题
  • 能从时间、空间复杂度两方面优化算法效率
  • 具备优秀的沟通能力、学习能力、发散思维能力

扎实的基础知识

  • 至少掌握一两门编程语言。
  • 熟练掌握数据结构：链表、树、栈、队列和哈希表及其相关操作。
  • 注重查找、排序等算法。

高质量的代码
关注边界条件、特殊输入等方面。
如：各种特殊输入，考虑溢出情况，错误处理，对空指针、空字符串的处理。
对策：写代码之前想好测试用例，把各种可能的输入事先想好。

清晰的思路
形成清晰思路的建议：

  1. 一两个具体的例子模拟操作的过程，从具体的例子中寻找规律。
  2. 用图形表示抽象的数据结构。
  3. 把复杂的问题分解成若干个简单的子问题，一一解决。

优化效率的能力

  1. 要知道如何分析效率。
  2. 要熟知各种数据结构的优缺点。
  3. 熟练掌握常用的算法。

优秀的综合能力

  1. 沟通能力
  2. 团队合作意识
  3. 学习能力：最近学习的技术，理解新概念的能力
  4. 知识迁移能力：从简单的问题中找到解决复杂问题的能力
  5. 建模能力和发散思维能力

应聘者提问环节

几点建议

  1. 不要问和自己的职位没有关系的问题；
  2. 不要问薪水；
  3. 不要立即打听面试结果；
  4. 问应聘职位或项目相关的问题：主要业务、职位要求、项目进展。
