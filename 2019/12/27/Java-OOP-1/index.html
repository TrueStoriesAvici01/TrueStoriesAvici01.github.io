Java-面向对象(一)

2019-12-27

Java 对象与类

面向对象程序设计
程序：由对象组成。
对象：包含对用户公开的特定功能部分和对用户隐藏的实现部分。

类(class)是构造对象的模板和蓝图。

类构造对象的过程称为创建类的实例(instance)。

class -(构造)-> instance

封装：将数据域和行为组合，对象的使用者不知道实现的细节。

实例域(instance field)：对象中的数据。
方法(method)：对象中对这些数据的操作。

提示：
实现封装的一个准则就是：
不要在类外直接访问该对象的实例域，而是只通过该对象的方法实现与实例域的交互。

继承(inheritance)：
通过扩展(extends)一个已有的类来构造一个新的类。

类

类可以看作是一种自定义的数据类型.

类与对象

  • 类: 是一批对象的抽象.
  • 对象: 是一个具体存在的实体.

定义类的格式:

类中成员的组成:

  • 成员变量
  • 构造器
  • 方法

成员变量:
类或对应实例的数据.

定义成员变量的格式:

构造器:
一个类创建对象(实例)的途径.(没有则系统默认发配)

定义构造器的格式:

方法:
类或该类对象的行为或功能.

定义方法的格式:

注:

  • 修饰符: 可以是{public, protected, private,}中选一和{static,final,abstract}中选一组合.
  • 类型: 可以是基本类型和引用类型.
  • 形参列表: 由形参类型 形参名构成,多组形参之间用,隔开.
  • 构造器名: 必须与类名保持一致.

static修饰的成员变量和方法为类变量和类方法,表示其属于这个类,而非某个特定的对象.
没有static修饰的变量和方法称为实例变量和实例方法.
static修饰的方法不能访问未被static修饰的变量或方法.

对象

其中:
new Person(Bob,20): 创建一个Person类的对象(实例).
Person person1: 定义一个Person类型的引用变量.
结果: 引用变量指向新创建的对象.(类似于指针)
内存: 引用变量存放在栈内存中(Stack),对象(实例)存放在堆内存(heap)中.

对象的this引用

使用情景:

  • 构造器中引用该构造器正在初始化的对象.
  • 方法中引用调用该方法的对象.

作用:
类中的一个方法调用该类中另一个方法或实例变量.
一般情况下,调用一个方法不需要使用this关键词,只有当方法内有同名的局部变量时才加this指明该变量是类中的成员变量, 如:

与static的关系:

  • static修饰的方法中不能使用this引用.
  • static修饰的方法中不能访问没有static修饰的成员变量或方法.

方法

  • 类中的方法类似于函数,但是其必须依赖于对应的类或对象,不能独立使用.
  • static修饰的方法属于类,故该类的不同的对象调用时其结果都相同.
  • 非static方法,调用者为对象,static方法,调用者为类.

参数传递
方法的参数传递只有一种: 值传递.
值传递: 将实际参数值的副本传入方法中,原来值不受影响.

注:
当方法的参数是一个对象时,其传入的是对象的引用,对该引用变量进行修改时,不会影响引用所指向的对象. 因此是为引用的值传递,而非对象传递.

形参个数可变
方法中的参数列表中的一个形参的个数是可变的.
形式:参数类型... 参数名,如int... arg.
注:

  • 方法的参数列表中的个数可变的形参只能放在列表的最后.
  • 有且只有一个个数可变的形参.
  • 其等价于对应参数的数组参数类型[] 参数名.

递归方法

一个方法在其内部调用自己.注意避免无穷递归.

方法重载

一个类中定义多个同名的方法, 其参数列表不同.

对于一个重载了无参参数列表,个数可变的参数和只有一个参数的方法:

  • 对于无参的方法的调用最终会调用到个数可变的参数的方法.
  • 对于只有一个参数的调用,最终调用的是只有一个参数的方法.
  • 其他情况,会调用个数可变的参数的方法.
  • 若传入一个参数,而想调用个数可变的参数的方法,可以构造一个只含一个参数的数组.

变量

分类:

所有变量:

  • 成员变量
  • 局部变量

成员变量:

  • 实例变量(无static)
  • 类变量(有static)

局部变量:

  • 形参(方法形参列表中定义)
  • 方法局部变量(方法内定义)
  • 代码块局部变量(代码块内定义)

各变量参数:

变量 初始化时期 生命周期 默认初始值 存储位置
类变量 首次加载类 整个类 有 堆内存
实例变量 实例创建时 整个实例 有 堆内存
形参 方法被调用 整个方法 无
方法局部变量 定义该变量起 从定义起,到方法结束为止 无 栈内存
代码块局部变量 定义该变量起 从定义起,到代码块结束为止 无 栈内存

同名

  • 相同方法的不同代码块局部变量可以重名.
  • 局部变量和类变量可以重名.(局部变量覆盖类变量)
  • 其他情况不可重名

使用规则:
减少变量的生存周期.

成员变量使用情景:

  • 所有实例共有的属性.
  • 用来保存类的状态.
  • 同内中多个方法共享的信息.

封装(Encapsulation)

举例:
内存条: 内部实现细节和数据被隐藏在其内部,外部只需通过金手指就可以实现数据的存取.

将对象的状态信息隐藏到对象内部,不允许外部程序直接访问对象的内部信息,而是通过该类所提供的方法来实现对内部信息的操作.

目的:

  • 隐藏类的实现细节.
  • 使用者对数据的访问受限,保证数据的合法性.
  • 便于修改,提高可维护性.

访问控制级别表

private default protected public
同一个类中 Y Y Y Y
同一个包中 Y Y Y
子类中 Y Y
全局范围内 Y

选择原则:

  • private: 大多数成员变量,以及用于辅助实现其他方法的工具方法.
  • protected: 作为其他类的父类,需要被子类重写方法,而非直接调用.
  • public: static修饰的,全局变量; 供外部使用的方法.

package,import和import static

package: 一组功能相关的类放在一起,组成逻辑上的类库, 可避免重名.

import: 导入包名,其后可省略包名而直接使用类名.

import static: 导入static的类变量,方法,可省略类名.

注:

  • package 只能放在类中的首句.
  • 一个类只能指定一个package.
  • 一个类中可以使用多个import导入.
  • import导入时使用 * 可导入包中所有的类,但其子包需额外导入.
  • 导入的多个包有重名类,使用时要加包名指定哪个包中的类.

构造器

作用:
创建对象时对对象进行初始化操作.

系统 => 创建对象(分配内存,默认初始化)
构造器 => 变量初始化

构造器重载

使用不同的参数列表实现变量的不同的初始化.

通过this调用其他构造器

this语句只能放在首句,不同的变量初始化位于其后.

继承

目的:
子类通过继承父类,获得父类的全部成员变量和方法.

特点:

  • 只能继承一个父类.
  • 子类只有一个直接父类,但可以有多个间接父类.
  • 所有类都继承于java.lang.Object.

重写父类的方法

情景:
需要继承父类的方法,但又有所不同.

重写(Override)/方法覆盖: 子类中定义父类同名的方法.

特点:

  • 方法名,形参列表相同.
  • 子类返回值和异常类型比父类小.
  • 子类方法的访问权限比父大.
  • 父类private对子类隐藏,子类的重名方法不是重写.

重载与重写

  • 重载(Overload): 同一个类中多个同名方法之间.
  • 重写(Override): 子类与父类之间同名方法.

调用父类的方法

情景:
子类重写父类的方法,子类的实例默认使用子类的方法.若想要调用父类的方法,则需要super.

注:

  • 不能用于static的方法中.
  • 系统创建子类时,会首先创建对应的父类,直到其父类为java.lang.Object为止,然后在创建对应的子类.
  • 向父类依次查找成员变量时,若其中有一个private变量,则无法在继续向上查找,提示无法访问private变量.

未指明调用者,则变量的查找顺序为:

  1. 该方法内部的局部变量.
  2. 当前类的成员变量.
  3. 查找直接父类或间接父类的成员变量.

多态(Polymorphism)

情景:
引用变量的类型和其所引用的对象的类型不同,此为多态.

Object obj = new String();

obj 编译时为Object类型, 运行时为String类型.

注:

  • 只能用父类的引用变量引用其子类的对象.
  • 引用变量在编译时为声明该引用变量的类型(父类),运行时为其引用的对象的类型(子类).
  • 编译阶段,引用变量只能使用其声明的类中的方法或其父类中的方法,其子类中的方法不可用.
  • 运行时,引用变量可以默认使用其引用对象中的方法,否则再去使用其父类的方法.
  • 调用引用变量的成员变量,使用的是声明该引用变量的成员变量(父类中的成员变量),与方法在运行时调用子类中的方法不同.

类型转换

格式:
(type)variable

注:

  • 基本类型只能是数值类型(整数,字符,浮点)之间相互之间转换,数值与布尔不能相互转化.
  • 引用类型之间的相互转换只能发生在两个有继承关系的类型之间.
  • 将父类引用变量转换为子类引用变量,只能是其所指向的对象为要转换的子类类型,否则异常.
  • 子类转换为父类可忽略类型转换.(向上转型,upcasting)

instanceof 运算符

格式:
引用变量 instanceof 类

功能:
判断前面的引用变量是否是后面的类的类,子类,实现类的实例.是则为true,否则false.

组合

继承的缺陷: 破坏封装.

父类设计原则:

  • 隐藏父类的内部数据,避免直接访问父类的成员变量.
  • 避免子类随意修改父类的方法: public: 外部调用; final: 被调用而不能重写; protected: 能重写而不能自由访问.
  • 不要在父类构造器中调用子类重写的方法.

派生子类的原则:

  • 子类需要增加额外的属性.
  • 子类需要增加自己独有的方法.

组合实现复用
把旧类对象作为新类的成员变量组合起来,实现新类的功能.
隐藏(private)旧类对象,开放新类的方法.

组合与继承的关系

  • 继承: 对已有的类的改造,获得特殊的版本,是将抽象的类改造成适应于特定需求的类.
  • 组合: 两个类之间有明确的整体,部分的关系.
  • 继承: 抽象-具体;
  • 组合: 整体-部分.

初始化块

情景:
多个构造器使用相同的初始化语句,可以合并成一个初始化块,减少冗余.

格式:

static初始化块
类初始化阶段执行,只能对static变量进行初始化.

次序:

  1. 执行父类的static初始化块.
  2. 执行子类的static初始化块.
  3. 执行父类的普通初始化块.
  4. 执行父类的构造器.
  5. 执行子类的普通初始化块.
  6. 执行子类的构造器.

原则:

  • 先执行父类,再执行子类.
  • 先执行static初始化块,再执行普通初始化块,最后执行构造器.
  • 多个构造器依据调用情况选择构造器.
  • 初始化块之间按照原本的次序先后执行.

对象构成:

  • 特定功能部分(公开)
  • 实现部分(隐藏)

从一个类构造一个对象,该对象称为类的一个实例(instance).

封装(Encapsulation, information hiding):
将数据和行为组合在一个包类,对用户隐藏实现的细节.
表现:

  • 用户无法直接与实例域交互.
  • 用户只能通过特定的方法与实例域交互.

实例域(instance fileds): 对象的数据.
方法(methods): 对象的行为.

继承(inheritnce): 通过已有类来构造(extends)新的类.
(所有类都是Object类的子类)

对象的特征

  • 行为(behaviors)
  • 状态(state)
  • 标识(identity)

OOP流程

分辨不同的类,并向类中添加对应的方法.
名词 => 类
动词 => 方法

类间关系

  • 依赖(dependence): 其方法使用或操作另外一个类的对象.(需要降低依赖/耦合(coupling))
  • 聚合(aggregation): 一个类包含另外一个类的对象.
  • 继承(inheritance): 特殊和一般的关系. 继承的子类获取父类的方法,并且具有特有的行为.

更改器方法和访问器方法

  • 更改器方法(mutator method): 调用该方法后,对象的状态发生改变.
  • 访问器方法(accessor mathod): 只访问对象,而不修改对象的状态.

构造器

  • 构造器与类同名.
  • 一个类可有多个构造器.
  • 一个构造器可有0或多个参数.
  • 构造器没有返回值.
  • 构造器只被new操作符调用.

构造器或方法内同名变量
构造器或方法中定义实例域(instance field)同名的局部变量:

  • 在对该变量进行修改时,为局部变量改变,对应的同名实例域变量保持不变.
  • 使用this.变量名可以对实例域变量进行修改.

隐式参数和显式参数(Implicit & Explicit)

  • 隐式参数: 调用方法时在方法名之前,为方法所属类的对象.(方法内可用this代替)
  • 显式参数: 方法的参数列表.

良好的封装

  • private的数据域.
  • public的访问器方法.
  • public的修改器方法.

封装的优点

  1. 类的实现的更改不会影响其使用.
  2. 可以对实例域的修改进行检查,确保其合法.

注: 不要使用可变对象作为方法的返回值
任何对该对象的修改将会导致所有引用该对象的对象的状态随之改变.
建议: 使用clone来进行拷贝.

一个对象的方法可以访问同类对象的私有域.

私有方法

  • 与其他方法联系紧密,如: 是其他方法的一部分.
  • 需要特定的调用次序,如: 两个方法之间有先后次序要求.

final实例域
确保构造器执行完后,其值已经初始化.
特点: 用于一旦设定完后不再修改的数据.

static域和static方法

  • static域: 为类变量,该类的所有对象共用同一个域.对象不存在static域也存在.
  • static常量: 可以通过类直接访问该变量而无需创建对象.
  • static方法: 不能向对象操作的方法,不使用对象访问,只可访问自身类中的static域.

static方法的使用

  • 不需要访问对象,其参数都是显式参数.
  • 只访问类的静态域.

main方法
main方法是static方法, 不对任何对象进行操作.
常用于进行单元测试.

方法参数

Java中只有值调用（call by value）。

方法参数分类：

  • 基本数据类型（数字，布尔值）
  • 对象引用

使用情况：

  1. 一个方法不能修改一个基本数据类型的参数。
  2. 一个方法可以改变一个对象参数的状态。
  3. 一个方法不能让对象参数引用一个新的对象。

对象构造

重载（overloading）

特征： 多个方法具有相同的名字、不同的参数。
注： 不同的返回值不能构成重载。

默认域初始化

实例域没有初始化，默认自动初始化为：

  • 数值 => 0
  • 布尔值 => false
  • 对象引用 => null

无参数的构造器

  • 没有构造器时，系统默认提供一个无参构造器
  • 拥有有参构造器，则默认的无参构造器则无法使用。

显式域初始化

可以调用方法对域进行初始化。

参数名

参数变量名与实例域变量重名时，方法内使用this.变量名调用实例域变量。

调用其他构造器

  • 当前构造器中使用this(...)调用其他的构造器。
  • this只能放在第一句。

初始化块（initialization block）

处理次序：

  1. 所有数据域被初始化为默认值。
  2. 按照类中出现的次序，依次执行所有域初始化语句和初始化块。
  3. 若构造器调用其他构造器，则执行其他的构造器。
  4. 执行构造器中的其他内容。

导入包

格式：

  • import 包名/类名
  • import 包名.*：将包内所有的类导入。
  • 不能使用 * 导入多个包。
  • 多个包中重名的类使用时需要指定完整的包名。

static导入

import static 包名: 导入静态方法或静态域。
