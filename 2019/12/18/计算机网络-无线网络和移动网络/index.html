计算机网络-无线网络和移动网络

2019-12-18

无线网络和移动网络

概述

无线网络要素

  1. 无线主机
  2. 无线链路

无线网络要点

  • 基站(base station): 负责向与之关联的无线主机收发数据. 如: 蜂窝网络中的蜂窝塔(cell tower)和802.11无线LAN中的接入点(access point).
  • 基础设施模式(infrastructure mode): 与基站关联的主机运行的模式.
  • 自组织网络(ad hoc network): 无线主机没有与基础设施相连接.
  • 切换(hand off): 从一个基站到另一个基站,移动主机改变其接入点.

无线网络分类:

  1. 单挑,基于基础设施: 无线主机与基站相连,主机与基站只有一个无线跳. 如: 3G蜂窝网络和802.11.
  2. 单跳,无基础设施: 不与基站相连. 如: 蓝牙网络,自组织模式的802.11.
  3. 多跳,基于基础设施: 无线节点通过其他节点中继来与基站通信. 如: 无线传感器网络和无线网状网络(wireless mesh network).
  4. 多跳,无基础设施: 没有基站,节点通过其他节点中继来到达目的地. 如: 移动自组织网络(mobile ad hoc network, MANET), 车载自组织网络(vehicular ad hoc network).

无线链路和网络特征

无线链路的特点

  1. 递减的信号强度: 电磁波穿透物体是强度会减弱,会随着距离增加而减弱(路径损耗(path loss))
  2. 来自其他源的干扰: 同一频段发送的信号的电磁波会相互干扰.
  3. 多径传播: 电磁波由于发生发生经过不同路径到达目的地会产生多径传播(multipath propagation).
  4. 掩藏终端问题(hidden terminal problem): 两个站点同时向同一站点发送信息,由于物理阻挡,导致两个站点无法听到对方的传输.
  5. 衰减(fading): 两者的距离太远导致信号衰减无法相互检测对方的传输.

CDMA

流程:
发送的每个比特都要通过乘以M个比特长的CDMA编码,输出为编码后的序列.
接收方对每个M长的序列进行解码,从而得到对应的一个比特.

数学表示:
发送方: $Z_{i,m} = d_i \times c_m$.
接收方: $d_i = \frac{1}{M} \sum_{m=1}^{M}Z_{i,m}\times c_m$.
$c_m$: CDMA编码的第m个比特.
$d_i$: 要发送的未编码的第i个比特.
$c_m(m=1,…,M)$:收发双方事先知道.

多用户传输:
对于每个用户选择恰当的CDMA编码,可以从聚合的信号中恢复出给定的发送方发送的数据.
数学表示:
N个发送方传输的比特总和:
$Z^\star_{i,m} = \sum_{i=1}^NZ^s_{i,m}$
获取第k个用户的第i个数据比特:
$d_i^k = \frac{1}{M}\sum_{m=1}^M Z^\star_{i,m}\times c_m^k$

WiFi: 802.11无线LAN

标准

  • 802.11b
  • 802.11a
  • 802.11g

关系

  • 使用相同的CSMA/CA
  • 链路层帧使用相同的帧结构
  • 都可降低传输速率而达到更远的距离
  • 都允许”基础设施模式”和”自组织模式”
  • b,g的工作频率较低,传输速率低,传播距离远;a的工作频率高,传输速率快,传播距离近.
  • 802.11n: 使用多输入多输出MIMO,传输速率快.

802.11体系结构

基本服务集(Basic Service Set,BSS): 一个BSS包含多个无线站点和一个接入点(Access Point,AP)(中央基站(base station),其接入网络).

基础设施无线LAN(infrastructure wireless LAN): 配备AP的无线LAN.

自组织网络: 多个站点构成的无中心控制(AP),与外部网络不连通的局域网络.

信道与关联

服务集标识符(Service Set Identifier,SSID): 即AP名(如:WiFi名)

WiFi丛林(WiFi jungle): 多个AP覆盖的位置.

关联(associate): 无线站点从WiFi丛林中选择一个AP,并与之创建一个虚拟线路.

关联方式

  • 被动扫描(passive scanning): AP发送信标帧(beacon frame,包含AP的SSID和MAC地址),无线站点通过扫描不同信道获得信标帧,选择一个AP关联.
  • 主动扫描(active scanning): 无线站点广播探测请求帧,AP发送探测响应帧进行响应.无线主机选择一个AP,并向其发送一个关联请求帧,AP发送关联响应帧进行回应.

关联后:
无线站点通过AP向子网发送DHCP发现报文,获取子网的一个IP地址,无线站点加入子网中.

关联过程中的身份鉴别:

  • 站点的MAC地址
  • 用户名和密码: 需要鉴别服务器(协议: RADIUS或DIAMETER)

802.11 MAC协议

802.11的多路访问协议

  1. 使用碰撞避免.
  2. 使用链路层确认/重传(ARQ).

链路层确认(link-layer acknowledgment)
目的: 检测数据帧是否无损到达目的地.
方法: 目的地收到CRC校验的帧后等待短帧间间隔(Short Inter-Frame Spacing,SIFS)后发送一个确认帧.若发送站点在规定时间内未收到确认帧,则认为发送失败,进行重传.多次重传无确认,则放弃发送该帧.

带碰撞避免的CSMA,CSMA/CA(CSMA with collision avoidance)
CSMA: “载波侦听多路访问”,站点在传输前侦听信道,信道忙则抑制传输.
CA: “碰撞避免”,802.11中站点一旦发送一个帧,则完全发送完该帧.

CSMA/CA流程

  1. 若监听信道空闲,则在一个分布式帧间间隔(distributed Inter-Frame Space,DIFS)的时间后发送该帧.
  2. 否则: 选择一个随机回退值并在信道空闲时递减该值.信道忙则值保持不变.
  3. 计数值为0时发送数据帧并等待确认.
  4. 收到确认,则帧被正确接收.否则,重新进入第二步的回退阶段,在更大的范围内选取随机值.

第二步中随机回退值在信道空闲仍然抑制传输:
理由: 避免多个站点同时等待正忙的信道时导致的信道一空闲就同时发送而导致碰撞.
问题: 两个站点相互隐藏或随机值接近.

隐藏终端导致的碰撞

使用短请求发送(Request to Send,RTS)控制帧和一个短允许发送(Clear to Send,CTS)控制帧来预约信道.

流程:

  1. 发送方发送DATA帧前,先向AP发送一个RTS帧(指示传输DATA帧和确认(ACK)帧需要的总时间).
  2. AP收到RTS帧,广播一个CTS帧作为响应.(发送方获得发送许可,其他站点被指示不可发送)

IEEE 802.11 帧

图示:

有效载荷与CRC字段
有效载荷: 放置一个IP数据包和一个ARP分组.
CRC: 检测帧中的比特错误.

地址字段

  • 地址1: 接收该帧的无线站点的MAC地址.
  • 地址2: 传输该帧的站点的MAC地址.
  • 地址3: BSS中AP连接的路由器接口的MAC地址.
  • 地址4: 自组织模式中相互转发时使用.

序号,持续期和帧控制字段
序号: 区分新传输的帧和以前帧的重传.
持续期: 包含传输数据帧的时间和传输确认的时间.
帧控制字段: 类型和子类型用于区分关联,RTS,CTS,ACK和数据帧.To和From定义不同地址字段的含义.WEP指示是否加密.

相同的IP子网中的移动性

移动站点在同一子网的不同BSS中移动,移动站点会解除与原AP的关联,然后与新BSS中的AP关联.
新AP在关联形成后,会以站点的源地址向交换机发送一以太网广播帧.交换机收到帧后,更新转发表.

802.11 中的高级特色

802.11 速率适应
当连续两个帧没有收到确认,传输速率降低到前一个较低的速率.
当10个帧连续得到确认,或用于跟踪上次降速的定时器超时,传输速率提高到上一个较高的速率.

功率管理
结点将其功率管理比特设为1,指示其即将睡眠,并设置定时器使得在AP发送信标帧前唤醒.
接入点将先缓存睡眠结点的帧,等其唤醒后.
结点唤醒后查询是否有缓存帧,没有则继续睡眠,否则接收帧.

蓝牙和ZigBee

蓝牙

IEEE 802.15.1 用于小范围,低功耗和低成本运行.被称为无线个人区域网络(Wireless Personal Area Network,WPAN).

每个时隙选择一个信道传输,并在下一个时隙以随机方式变更信道(调频扩展频谱Frequency-Hopping Spread Spectrum,FHSS).

传输过程:
设备组织成一个8个活动设备的皮可网(piconet),其中一个设备为主设备,其余为从设备,其在奇数时隙发送消息,从设备只有在主设备与其通信后才能发送,只能发送给主设备.其余的设备为存放设备(parked),只有主结点将其转换为活动之后才能进行通信.

ZigBee

结点: 多个”简化功能设备”在单个”全功能设备”控制下作为从设备运行. 多个全功能设备能够配置网状(mesh)网络,之间发送帧.

蜂窝因特网接入

全球移动通信系统(GSM)

  1. 第一代(1G): 模拟FDMA系统,用于语音通信.
  2. 第二代(2G): 语音通信,扩展对数据的支持(2.5G).
  3. 第三代(3G): 支持语音和数据.

2G蜂窝网络体系结构

小区(cell): 一个蜂窝网覆盖的区域
收发基站(Base Transceiver Station,BTS): 向小区内的移动站点收发信号.(适用组合的FDM/TDM: 信道被分成多个子带,时间被划分成多个时隙)
基站控制器(Base Station Controller,BSC): 服务多个基站. 功能: 为用户发配BTS信道,执行寻呼(paging,找到另一个移动用户),执行移动用户切换.
GSM基站系统(Base Station System,BSS): 基站控制器和其控制的收发基站.
移动交换中心(Mobile Switching Center,MSC): 用户鉴别和账户管理,呼叫建立和切换.

3G蜂窝数据网

结构:
保留核心GSM蜂窝语音网,增加并行的附加蜂窝数据功能.

新结点:

  • 服务通用分组无线服务支持节点(Serving Generalized packet radio service Support Node,SGSN): 在移动节点和GGSN之间转发数据报,与MSC交互,提供用户认证和切换,维护活跃节点的位置.
  • 网关GPRS支持节点(Gateway GPRS Support Node,GGSN): 网关作用,将多个SGSN连接更大的因特网.
  • 无线电网络控制器(Radio Network Controller,RNC): 通过MSC与电路交换蜂窝语音网连接,通过SGSN与分组交换因特网连接.

差异:
在TDMA时隙中使用直接序列宽带CDMA(Direct Sequence Wideband CDMA,DS-WCDMA).

4G LTE

LTE: Long-Term Evolution.

创新

  1. 演化的分组核(Evolved Packet Core,EPC): 语音和数据在IP数据报中承载.
  2. LTE无线电接入网(LTE Radio Access Network): 下行信道采用正交频分复用(Orthogonal Frequency Division Multiplexing,OFDM): 频分复用和时分复用结合.

移动管理

概念

  1. 归属网络(home network): 移动结点的永久居所.
  2. 归属代理(home agent): 归属网络中代表移动结点执行移动管理功能的实体.
  3. 外部网络(foreign network)/被访网络(visited network): 移动结点当前所在的网络.
  4. 外部代理(foreign agent): 外部网络中帮助移动结点进行移动管理的实体.

寻址

方法:

  1. 通过交换域内与域间路由选择信息实现,通告其他网络有一条正确的路径可将数据报导向该移动结点的固定地址.
  2. 外部代理为移动结点创建一个交换地址(Care-Of Address,COA).移动结点与两个地址:永久地址(parmanent address)和COA(外部地址). 外部代理告诉归属代理移动结点具有的COA.COA用于将数据报通过外部地址重路由到移动结点.

路由选择到移动结点

间接路由选择(indirect routing)

  • 移动节点到外部代理的协议: 移动节点到外部网络,向外部代理注册.
  • 外部代理到归属代理的注册协议: 外部代理向归属代理注册移动结点的COA.
  • 归属代理数据报封装协议: 将原始数据报封装在目的地址为COA的数据报中,转发数据报.
  • 外部代理拆封数据报: 将数据报拆封出原始数据报,转发给移动结点.

缺陷: 三角路由选择问题(triangle routing problem): 通信者和移动节点存在更有效的路径时,数据报仍然先发给归属代理,再转发到外部网络.

直接路由选择(direct routing)

  • 通信者所在的网络中有一个通信者代理(correspondent agent),其通过向归属代理查询获得移动节点COA.
  • 通信者通过通信者代理将数据报直接发送到移动结点COA.

缺陷: 当移动结点在多个外部网络移动时COA变化后通信者代理的更新.
解决: 移动结点到达的第一个外部网络的外部代理为锚外部代理(anchor foreign agent). 移动结点的新外部代理将新COA发送给锚外部代理,锚外部代理将数据报重新封装转发给新的外部代理.

移动IP

移动IP: 支持移动性的因特网体系结构与协议.

组成:

  1. 代理发现: 归属代理/外部代理通告移动结点的协议,移动结点请求归属代理/外部代理的协议.
  2. 向归属代理注册: 向归属代理注册或注销COA所用的协议.

  3. 数据报的间接路由选择: 数据报由归属代理转发至移动节点的方法.

    代理发现

    代理发现(agent discovery): 移动结点中的网络层知道其已进入一个新的外部网络.

    方法:

  4. 代理通告

  5. 代理请求

    代理通告

    外部代理或归属代理周期性的广播一个ICMP报文,包括路由器的IP地址和移动性代理通告扩展.

  • 归属代理比特(H)
  • 外部代理比特(F)
  • 注册要求比特(R)
  • M,R封装比特

  • 转交地址(COA)字段

    代理请求(agent solicitation)

    移动结点广播一个代理请求报文,代理节点单播一个代理通告.

    向归属代理注册

    步骤:

  1. 收到外部代理通告后,移动结点向外部代理发送一个移动IP注册报文.
  2. 外部代理收到注册报文并记录移动结点的永久IP地址.
  3. 归属代理收到注册请求并检查真实性和正确性.

  4. 外部代理接收注册响应,然后将其转发给移动结点.

    蜂窝网中的移动性管理

定义

  • 归属公共地域移动网络(home Public Land Mobile Network, homePLMN)/归属网络(home network): 移动用户的归属网络.
  • 被访网络(visited network): 移动用户当前所在网络.
  • 归属位置注册器(Home Location Register,HLR): 维护用户的个人信息, 通信者与归属网络的网关移动服务交换中心(Gateway Mobile services Switching Center,GMSC/归属MCS)联系.
  • 访问者位置注册(Visitor Location Register,VLR): 维护当前服务网络中的移动用户.

移动用户呼叫的路由选择

  1. 通信者拨打移动用户的电话号码.
  2. 归属MSC收到该呼叫并查询HLR来确定移动用户的位置.(HLR返回移动站点漫游号码(Mobile Station Roaming Number,MSRN)/*漫游号码).
  3. 给定一个漫游号码,归属MSC通过网络到达被访网络的MSC建立呼叫.

GSM中的切换

切换(handoff): 移动结点从一个基站到另一个基站.

原因:

  1. 消耗减弱,可能中断呼叫.
  2. 蜂窝过载.

步骤:

  1. 旧基站通告MSC即将进行一个切换.
  2. MSC发起建立到新BS的路径,发配资源.
  3. 新BS发配并激活信道.
  4. 新BS发出信令到被访MSC和旧BS,通知可以切换.
  5. 移动用户被告知应当切换.
  6. 移动用户与新BS交换报文,激活新的信道.
  7. 移动用户向新BS发送切换完成报文,报文被转发到被访MSC.
  8. 旧BS的资源被释放.

多MSC间切换
移动用户初始访问的MSC为锚MSC(anchor MSC).
呼叫从归属MSC路由选择到锚MSC,然后到当前被访MSC.

无线和移动性对高层协议的影响

链路层和网络层有重要区别.
运输层和应用层影响较小.

TCP的拥塞控制反应在无线情况下存在缺陷,解决方案:

  • 本地恢复: 在比特出错出现的当时和本地将其恢复.
  • TCP发送方知晓无线链路: 发送方和接收方知道无线链路的存在,将有线网络中发生的拥塞性丢包和无线网络中的差错/丢包区分.
  • 分离连接方法: 将端到端的连接分成两个运输层连接:一个移动主机到无线接入点,一个无线接入点到其他通信端点.
