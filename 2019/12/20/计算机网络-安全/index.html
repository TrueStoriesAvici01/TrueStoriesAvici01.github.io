计算机网络-安全

2019-12-20

计算机网络中的安全

网络安全

安全通信(secure communication):

  1. 机密性(confidentiality): 仅有通信双方能够理解报文的内容.
  2. 报文完整性(message integrity): 通信内容在传输过程中未被修改.
  3. 端点鉴别(end-point authentication): 通信双方能够证实对方的身份.
  4. 运行安全性(operational security): 抵御来自公共因特网的威胁.

潜在的威胁:

  1. 窃听: 监听,记录信道上传输的控制报文和数据报文.
  2. 修改,插入和删除报文或其内容.

密码学原则

对称密钥系统(symmetric key system): 通信双方的密钥是相同的.
公开密钥系统(public key system): 使用一对密钥,其中一个公开,另外一个只有一人知道.

对称密钥密码体制

凯撒密码(Caesar cipher): 每个字母由其后的第k个字母进行替换.

单码替换密码(monoalphabetic cipher): 使用非规则的模式进行替换.

多码替换规则(polyalphabetic encryption): 使用多个单码替换密码.一个单码替换密码用于加密某明文报文中一个特定位置的一个字母.

入侵分类:

  1. 唯密文攻击(ciphertext-only attack): 入侵者只知道密文.
  2. :已知明文攻击(konwn-plaintext attack): 入侵者知道(明文,密文)的一些匹配.
  3. 选择明文攻击(chosen-plaintext attack): 入侵者能够选择某一明文报文并得到该明文报文对应的密文.

分类:

  1. 流密码(stream ciphers)
  2. 块密码(block ciphers)

块密码:
将报文分成若干个k比特的块,对每个块单独进行一一映射加密.

密码块链接(Cipher Block Chaining,CBC):

目的:
防止相同的块加密生成相同的密文.

做法:

  1. 加密之前,生成一个初始向量(Initialization Vector,IV).
  2. 第一个块,使用明文与初始向量异或后加密.
  3. 后续的块,使用前一个加密后的块和明文以后后加密.

公开密钥加密

被通信者有两个密钥: 公钥(公开的)和私钥(私有的). 通信者通过被通信者的公钥加密明文,被通信者用私钥解密密文得到明文.

RSA
公钥,私钥的生成:

  1. 选择大素数p和q.
  2. 计算 n=pq 和 z=(p-1)(q-1).
  3. 选择小于n的数e,使得e和z互素.
  4. 选择d,使得 ed mod z =1.
  5. 公钥对为(n,e);私钥对为(n,d).

加密:
待加密的明文为m:
密文c=(m^e) mod n.

解密:
解密得: m = (c^d) mod n.

会话密钥(session key)

用来传输大量数据的密钥.

方法:

  1. 选择会话密码(对称密钥).
  2. 通过RSA加密会话密钥.
  3. 利用会话密钥加密数据.

报文完整性和数字签名

报文完整性(message integrity): 报文鉴别.

密码散列函数

密码散列函数(cryprographic hash function): 计算上不可能实现两个不同的报文x和y使得H(x) = H(y).

分类:

  • MD5
  • SHA-1
  • …

报文鉴别码

维拉保证报文完整性,需要使用鉴别密钥(authentcation key).

报文完整性步骤:

  1. 将报文m和鉴别密钥s级联后计算散列H(m+s),称为报文鉴别码(Message Authentication Code,MAC).
  2. 明文和MAC发送给对方.
  3. 对方计算H(m+s),若与发送的报文界别码相同,则正常.

数字签名(digital signature)

作用:
类似现实生活中的签名,指明某个文件的所有者或某人认可一个文件的内容.

要求:

  • 可证实性
  • 不可伪造性

应用:

身份认证

步骤:
签名:

  1. 签名者对报文m计算其”指纹”H(m)(哈希值).
  2. 使用签名者的私钥对H(m)进行加密K(H(m)).
  3. 发送报文m和加密后的密文K(H(m)).

验证:

  1. 用公钥对明文解密,获得散列值H(m).
  2. 对报文计算其散列值H’(m).
  3. 两者相同则身份验证成功.

公钥认证(public key certification)

目的: 证实公钥是对应于某一实体,防止冒名顶替.

认证中心(Certification Authority,CA): 识别和发行证书合法化.
功能:

  • 证实一个实体的真实身份.
  • 生成身份和公钥绑定的证书(certificate).

端点鉴别

回放攻击(playback attack): 攻击者记录下带有口令的加密报文,再向目标回放该报文,则可实现伪装.

解决方案:

  1. 被认证者发送认证报文.
  2. 认证者回应一个不重数(nonce,在协议生存期中不会再次使用的数).
  3. 被认证者使用共享的密钥加密不重数,并发送.
  4. 认证者解密,不重数正确则确认身份.

安全电子邮件

PGP(Pretty Good Privacy)
一种电子邮件加密方案.

使用MD5或SHA计算报文摘要;
使用CAST,三重DES或IDEA进行对称密钥加密.
使用RSA进行公开密钥加密.
允许为所信任的用户鉴别更多密钥提供担保.
通过哦保持密钥签署方相互签署对方的密钥.

使TCP连接安全:SSL

安全套接字层(Secure Socket Layer,SSL): 安全性服务加强TCP.
运输层安全性(Transport Layer Security,TLS): SSL版本3的修改版本.

步骤:
SSL握手

  1. 客户发送其支持的密码算法列表,连同一个客户的不重数.
  2. 从列表中,服务器选择一个对称算法,一种公钥算法和一种MAC算法.将选择和证书以及服务器不重数返回给客户.
  3. 客户验证证书,提取公钥,生成前主密钥(Pre-Master Secret,PMS),使用服务器公钥加密PMS,并发送给服务器.
  4. 使用相同的密钥导出函数,客户和服务器从PMS和不重数计算主密钥(Master Secret,MS),之后的报文用MAC加密和鉴别.
  5. 客户发送所有握手报文的一个MAC.
  6. 服务器发送所有握手报文的一个MAC.

连接关闭:
在类型字段中指出该记录是否是用于终止该SSL会话(接收方可用MAC对其进行鉴别,防止截断攻击(truncation attack))

网络层安全性: IPsec和虚拟

IP安全(IP Security,IPsec): 为网络层提供安全性.

应用:
使用IPsec创建虚拟专用网(virtual private network,VPN).
提供机密性,源鉴别,数据完整性,重放攻击防护.

IPsec和虚拟专用网

IP数据报在网关路由器中被转换为IPsec数据报,并被转发进因特网.
IPsec数据报具有传统IPv4首部,能被互联网路由器处理.
IPsec数据报载荷包括一个IPsec首部,用于被IPsec处理.
IPsec数据报载荷是被加密的.

AH协议和ESP协议

IPsec协议簇中的两个主要协议:

  1. 鉴别首部(Authentication Header,AH): 提供源鉴别和数据完整性服务.
  2. 封装安全性载荷(Encapsulation Security Payload,ESP): 提供源鉴别,数据完整性和机密性.

安全关联

安全关联(Security Association,SA): 源和目的实体创建一个网络层的逻辑连接.

安全关联数据库(Security Assocaition Database,SAD): 存储所有SA的状态信息.

IPsec数据报

分组形式:

  1. 隧道模式(tunnel mode)
  2. 运输模式(transport mode)

IPv4数据报->IPsec数据报:

  1. 尾部附上”ESP尾部”字段
  2. 使用SA规定的密钥加密.
  3. 在前面附上”ESP首部”的字段.
  4. 使用SA规定的密钥生成鉴别MAC
  5. MAC附加在后面形成载荷.
  6. 生成一个IPv4首部字段.附加在载荷之前.

接收:

  1. 由协议字段,确定使用IPsec ESP处理.
  2. 使用SPI确定数据报所属的SA.
  3. 计算MAC并验证是否与ESP MAC字段相同.(防篡改)
  4. 检查序号(防重放)
  5. 使用SA关联密钥解密.
  6. 删除填充,提取原始IP报文.

IKE: IPsec中的密钥管理

因特网密钥交换(Internet Key Exchange,IKE): 自动生成SA.

  1. 在路由器之间创建agiel双向的IKE SA,创建加密和鉴别密钥,以及主密钥.
  2. 协商加密和鉴别算法.
  3. 两侧生成每个方向的一个SA.

无线LAN安全

有线等效保密(Wired Equivalent Privacy,WEP).

鉴别步骤:

  1. 主机通过接入点请求鉴别.
  2. 接入点以一个128字节的不重数响应该鉴别请求.
  3. 主机用与接入点共享的密钥加密不重数.
  4. 接入点解密主机加密的不重数.

IEEE 802.11i

阶段:

  1. 发现: AP通告鉴别和加密形式,客户请求.
  2. 相互鉴别和主密钥(MK)生成: 鉴别发生在客户和鉴别服务器之间,接入点起中继作用.
  3. 成对主密钥(Pairewise Master Key,PMK): 使用MK生成PMK.
  4. 临时密钥(Temporal Key,TK): 使用PMK,生成用于通信的密钥.

运行安全性:防火墙和入侵检测系统

防火墙

防火墙(firewall): 将内部网络与因特网隔离,允许某些分组通过而阻止另外一些分组.

目标:

  1. 从外部到内部和从内部到外部的所有流量都经过防火墙.
  2. 仅被授权的流量允许通过.
  3. 防火墙自身免于渗透.

分类:

  1. 传统分组过滤器(traditional packet filter)
  2. 状态过滤器(stateful filter)
  3. 应用程序网关(application gateway)

传统的分组过滤器

分组过滤(packet filtering):
位置:
在内部网络与ISP之间的网关路由器,进入或离开内部网都要经过该路由器.
因素:

  • IP源/目的地址
  • IP数据报中的协议类型字段.
  • TCP或UDP的源和目的端口.
  • TCP标志比特: SYN,ACK.
  • ICMP报文类型.
  • 数据报离开或进入网络的不同规则.
  • 不同路由器接口的不同规则.

状态分组过滤器

跟踪TCP连接,从而作出过滤决定.
通过访问控制列表和连接表确定分组是否是正在进行的TCP连接的一部分.

应用程序网关

应用程序网关(application gateway): 一个应用程序特定的服务器,所有应用程序数据必须通过它.

流程:

  1. 用户输入要连接的外部主机的主机名.
  2. 网关与外部主机建立会话.
  3. 应用程序网关中继用户和外部主机的数据.

缺陷:

  1. 不同的应用程序使用不同的应用程序网关.
  2. 所有数据由网关转发,负担重.

入侵检测系统

深度分组检查(deep packet inspection): 查看分组携带的实际应用数据.

入侵检测系统(Intrusion Detection System,IDS): 观察到潜在恶意流量并产生告警.

入侵防止系统(Intrusion Prevention System,IPS): 滤除可疑流量的设备.

结构:

  • 高度安全区域: 由分组过滤器和应用程序网关保护,由IDS监视.
  • 非军事区(DeMilitarized Zone,DMZ): 仅有分组过滤器保护.

IDS系统分类:

  1. 基于特征的系统(signature-based system)
  2. 基于异常的系统(anomaly-based system)

基于特征的系统: 维护一个攻击特征数据库,每个特征是一个规则集.
缺陷: 对新攻击缺乏判断力;可能产生虚假警告;可能处理过载而难以检查.

基于异常的系统: 观察正常流量,生成概况文件.然后寻找不寻常分组流.不依赖现有攻击的以前知识.
