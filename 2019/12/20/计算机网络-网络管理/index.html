计算机网络-网络管理

2019-12-20

网络管理

概念

领域:

  • 性能管理(performance management): 量化,测量,报告,分析和控制不同网络部件的性能.
  • 故障管理(fault management): 记录,检测和响应网络中的故障.
  • 配置管理(configuration management): 跟踪网络中的设备,跟踪设备的硬件和软件配置.
  • 账户管理(accounting management): 允许网络管理员定义,记录,控制用户访问网络资源.
  • 安全管理(security management): 定义明确的策略控制对网络资源的访问.

基础设施

组成:

  1. 管理实体(managing entity): 控制网络管理信息的收集,处理,分析和显示,发起控制网络行为的动作.
  2. 被管设备(managed device): 有被管对象(managed object).对象具有的与之相关的部分信息,收集在管理信息库(MIB)中. 被管设备驻留网络管理代理(network management agent),在管理实体的命令和控制下,对被管设备采取本地行动.
  3. 网络管理协议(network management protocol): 允许管理实体查询被管设备的状态.

因特网标准管理框架

框架:

  1. 网络管理对象的定义: 管理对象形成一个信息库,为管理信息库(MIB).
  2. 数据定义语言: 管理信息结构(SMI): 定义数据结构,对象模型,写入和修改管理信息的规则.
  3. 协议SNMP: 传递信息和命令.
  4. 安全性和管理能力.

SMI

目的: 确保网络管理数据的语法和语义定义明确.

基本数据类型

具有专门用途的数据定义语言.

  • Integers
  • Octet strings
  • Object IDs
  • Network addresses
  • Counters
  • Gauges
  • Time ticks

较高层结构

OBJECT-TYPE结构定义被管对象的数据类型,状态和语义.
SYNTAX: 基本数据类型.
MAX-ACCESS: 被管对象是否能读写,创建…
STATUS: 对象的状态(当前的,合法的…)
DESCRIPTION: 可读的描述文本.

对象命名:OID和OID树
层次命名: 一个对象可用树根到树叶的节点名(数字)来标识.

语法:

  • 采用ASN.1的一些基本内容
  • ASN.1采用变量和声明.
  • 提供供应商中立,跨平台的语言.
  • 允许不同类型的计算机能够更有效的分享信息.

被管对象的类型

  • sequence: 简单数据类型的组合
  • sequence of: 简单类型的序列

MIB

按照层次组织起来的OID集合.

MIB-I
定义了100个被管对象
定义了8个对象组(group)

MIB-II
11个对象组,包含171个对象

MIB对象细节:

  • System Group: 系统信息
  • Interface Group: 接口信息
  • IP Group: IP分组信息
  • ICMP Group: ICMP报信息
  • TCP Group: TCP段信息
  • UDP Group: UDP数据报信息

BER

SMI使用BER对被管对象值编码成位串.

每个被传输的数据都有:

  • Type: 数据类型
  • Length: 数据的字节长度
  • Value: 数据的值

SNMP: 互联网网络管理协议

分类:

  1. 请求/响应模式
  2. 陷阱模式

报文类型:

  • GetRequest/GetNextRequest/GetBulkRequest
  • InformRequest
  • SetRequest
  • Response
  • Trap

PDU格式:

  • PDU type
  • request id
  • error status
  • error index
  • variable bind

编码SNMP报文
用tags(type)定义报文,Type字段包含:

  • class
  • format
  • number

安全性

  • 报文完整性: 报文不在传输中修改
  • 可认证性: 判断报文是否是一个有效的源发送的
  • 加密: 保证私密性
  • 基于视图的访问控制
