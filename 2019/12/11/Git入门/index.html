Git理论基础

2019-12-11

Git记录的是什么

  1. Git是将每个版本独立保存
  2. 工作区域、暂存区域和Git仓库

    工作区域 (Working Directory)：日常编程时的工作目录
    暂存区域 (Stage (Index)): 保存改动的目录
    Git仓库 (Repository(HEAD)): 最终存放所有数据的位置

Git 的工作流程

  1. 在工作目录中添加、修改文件
  2. 将需要进行版本管理的文件存放到暂存区域
  3. 将暂存区域的文件提交到Git仓库

Git 管理的文件有三个状态

  • 已修改 (modified)
  • 已暂存 (staged)
  • 已提交 (committed)

Git 的第一次实战

  1. 创建初始文件夹，作为本地仓库(即工作目录)
  2. 在cmd 中cd 到该文件夹，运行 git init
  3. git add 带提交的文件 将需要提交的文件提交到暂存区域
  4. git commit -m "提交说明"

Git 命令

  1. git status: 用来查看各个文件的状态。
    1. Untracked files: 为未添加到暂存区或提交的文件
    2. Changes not staged for commit: 工作目录的文件与暂存目录的文件不匹配(存在修改但为提交至暂存区域)
    3. Changes to be committed: 暂存区域存在未提交的文件
  2. git reset 要撤销的文件: 用来撤销被加入暂存区域的文件
  3. git checkout -- <从暂存区恢复的文件>: 用暂存区域的指定的文件覆盖工作区域
  4. git log: 查看历史提交

Git 回到过去

Working Directory - add -> Stage(Index)
Stage(Index) - commit -> Repository(HEAD)
Repository(HEAD) - reset -> Stage(Index)
Stage(Index) - checkout -> Working Directory

Git 命令

  1. git reset [--mixed] HEAD~: 回滚到上一个快照，~的个数代表回滚的代数~2等于~~。
    移动HEAD的指向，将其指向上一个快照
    将HEAD移动后指向的快照回滚到暂存区域
    1. git rest --soft HEAD~:
      移动HEAD的指向，将其指向上一个快照
    2. git reset --hard HEAD~:
      移动HEAD的指向，将其指向上一个快照
      将HEAD移动后指向的快照回滚到暂存区域
      将暂存区域的文件还原到工作目录
  2. git reset <快照ID>: 回滚到指定快照，也可前滚(通过版本ID确定)
  3. git reset <版本快照> <文件名/路径>: 回滚指定快照的个别文件
  4. git diff: 比较 工作区 和 暂存区 文件的差异
  5. git diff <快照ID1> <快照ID2>: 比较两个历史快照
  6. git diff <快照ID>: 比较当前工作目录和Git仓库中的快照
  7. git diff HEAD: 比较当前工作目录的内容和最新提交的快照的差别
  8. git diff --cached <快照ID>: 比较暂存区域和Git仓库快照
  9. git commit --amend -m "修改内容": “修正” 最近的一次提交 shift + zz保存退出。
  10. git rm <文件名>:
    1. 删除的只是工作目录和暂存区域的文件，也就是取消跟踪，在下次提交时不会纳入版本管理。
    2. 当工作区域和暂存区域的同一个文件在不同内容时，执行 git rm -f <文件名> 可以把两个都删除。
    3. 如果只删除暂存区域的文件(保留工作目录)，则可通过 fit rm --cached <文件名>。
  11. git mv <旧文件名> <新文件名>: 重命名文件。相当于：
    1. ren/mv <旧文件名> <新文件名>
    2. git rm <旧文件名>
    3. git add <新文件名>

分支

创建分支

git branch <分支名>: 创建分支
git checkout <分支名>: 切换分支
git log --decorate --oneline --graph --all: 以图像化方式显示所有分支的情况

合并分支

  1. git merge <待合并的分支>: 合并分支到主分支master中去（若存在同名的不同文件，则合并失败）

删除分支

git branch -d <分支名>: 删除分支
