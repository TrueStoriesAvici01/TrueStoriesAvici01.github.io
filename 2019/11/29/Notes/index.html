Posted on 2019-11-29 Edited on 2019-12-10

JBIG2

Model-Based Coding

  1. 若想获得最佳的压缩结果，使用的数据模型要与实际数据相匹配。
  2. T.4 和 T.6 将每一行看作黑白像素游程的组合。
  3. 为了进一步利用不同行之间的关联，二维的游程模型(MR和MMR)提供更佳的文本压缩效果。
  4. JBIG1 使用邻近的像素(模板)来预测单个像素。
  5. 自适应的方式提高周期性的半色调图像的压缩表现，使用的游程模型和基于上下文的预测模型具有通用性，但是通用性影响了某些图像的压缩能力。
  6. JBIG2 并不对文件做前提性假设, 而是提取最具代表性的比特图(bitmaps): 对不同类型的文字(不同的语言文字)和图像(周期抖动, 误差扩散)更加鲁棒。

Pattern Matching for Text Image Data

  1. 使用“基于字符的模式匹配”技术：
    对重复出现的字符中最具代表性的实例的比特图(bitmaps, symbol, mark, pixel block)进行编码，并存入字典中。

  2. 两种编码方式：

    1. pattern matching and substitution(PM&S)
    2. Soft pattern matching(SPM)

    要点
    两者之间的差异在于如何编码比特图。
    JBIG2允许两个编码器使用相同的编码格式。
    允许混合使用两者。

Pattern Matching and Substitution (匹配替换模式)

  1. 相同字符的两个实例不会按照像素进行匹配。
  2. 对于每个字符，编码一个指向字典中最具代表性的比特图的指针和其在文件中的位置（一般是相对于前一个相同字符的相对位置）。
  3. 若该字符未出现过，则直接编码比特图，并加入到字典中。
  4. 比特流不将编码的数值和字典数据放在一起，而是产生一个或多个字典片段，和一个或多个图像数据（指示比特图的数值信息）。

分割
图像会分割成比特图，提取其中的长，宽，面积，位置等信息。

字典搜索

  1. 预扫描潜在的比特图，跳过特征不符（黑色像素差异较大的，长宽高不符的）。
  2. 计算匹配分数，获得最佳匹配的比特图（Hamming distance）。
    当最佳匹配的得分超过阈值，认为匹配有效。（阈值与字符的特征有关（尺寸））

数值型数据编码
数值型数据（index，position）可进行huffman编码。

比特图编码
比特图可用MMR或JBIG1进行编码。

问题
由于替换，导致不可避免的替换误差。（可使用Residue Coding 或 soft pattern matching）

Soft Pattern Matching (软匹配)

  1. 加入细节信息(refinement data), 可用于重构原始字符。

  2. 细节信息包含：
    当前期望字符的像素；
    使用当前字符和匹配字符的比特图的编码。（有匹配字符，预测精度增加）
    使用匹配的字符对当前字符进行无损编码。
    匹配的比特图只用于预测模板中，增加无损压缩的精确度。
    数值型数据使用算术编码或huffman编码。
    比特图使用 逐像素的基于上下文模型的算术编码（JBIG1）或MMR（T.6）。

    编码细节

    1. 将待编码的比特图和匹配的比特图的几何中心进行匹配。
    2. 使用算术编码，逐行进行编码（使用不同于JBIG1的模板）
      模板由当前比特图的causal region（已编码的像素）和匹配的比特图中邻近的像素。

Halftones

两种压缩halftone的方式：

  1. 类JBIG1的基于上下文的算术编码 （使用16个像素构成的模板，其中4个为自适应）。
  2. 将halftone图像转换成灰度图像(gray scale image)，再传输灰度图像。
    将图像分割成由$m_b$行$n_b$列像素构成的块。
    对应的灰度值为块中像素之和。
    位平面(bit planes)使用基于上下文的算术编码(JBIG1)进行编码。
    灰度值作为半色调位平面字典(halftone bitmap dictionary)的索引。
    解码器使用索引恢复原图像。
