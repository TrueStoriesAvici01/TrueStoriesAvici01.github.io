递归与动态规划总结

2019-11-30

一句话简要介绍动态规划(dynamic programming)

对象：
求解最优化问题。
表现：
通常要做出一组选择来达到最优解，每次选择会生成与原问题形式相同的子问题。
？当多余一个选择子集都生成相同的子问题时，动态规划通常很有效。

与分治方法的联系

  1. 分治方法将问题划分成相互独立的子问题，递归求解子问题，再将解组合起来，从而求出原问题的解。
  2. 动态规划方法类似，但是对于子问题有重叠时（不同的子问题有公共的子子问题），动态规划只对子子问题求解一次，并将解保存到表格中（备忘录），从而避免重复计算。

性质

  1. 最优子结构：问题的最优解包含其子问题的最优解。
  2. 子问题重叠：递归求解过程中包含少量不同子问题的多次重复计算。

动态规划可分为两个部分：

  1. 递归：递归求解子问题。
  2. 备忘录：将已计算的值存储在表中。
    动态规划 = 递归 + 备忘录

动态规划的应用领域
通常用来求解最优化问题(optimization problem)。
通常有多个解，希望求得最优解（an optimal solution)

动态规划的基本步骤

  1. 刻画一个最优解的结构特征。
  2. 递归地定义最优解的值。
  3. 计算最优解的值，通常使用自底向上的方法
  4. 利用计算出的信息构造一个最优解。

例子

  • 字符串算法：最长公共子序列、最长递增子序列、最长公共子串、编辑距离等。
  • 关于图的有效求解算法：寻找图中最短距离的Bellman-Ford算法、Floyd的所有顶点间的最短路径算法等。
  • 链矩阵乘法。
  • 子集和。
  • 0/1背包问题。
  • 旅行商问题。

从一个例子开始

给定一段长为$n$的钢条，以及切割成长度为i（i = 1, 2,…, n）长度时的售价$V_i$，求如何切割可获得最佳收益。

两种划分子问题的思路

  1. 将钢条切割成两段，第一段长$l_1$，第二段长$n - l_1$，则钢条的最佳收益可表示为$P_n = P_{l_1} + P_{n-l_1}, l_1 = 1, 2,…, n$。

    原问题是求长为$n$的钢条的最佳收益，子问题为长为$l_1$和$n-l_1$的最佳收益，对两个子问题进一步求解，直到到达约束条件

  2. 将钢条切割成长度为$l_1$的一段（不再进行分割）和长度为$n - l_1$的一段（子问题，进一步切割，直到到达约束条件）。则最终收益为$P_n = V_{l_1} + P_{n - l_1}$。

动态规划的核心
记住已经解决过的子问题的解。

两种形式

  1. 自顶向下的备忘录法
  2. 自底向上

最优子问题(optimal substructure)
问题的最优解是由相关子问题的最优解组合而成，而这些子问题可以独立求解。

证明: $T(n)$表示切割长度为n的钢条的函数调用的次数，$T(n)$ = 1 + $\sum_{j = 0}^{n-1}T(j)$，$T(n) = 2^n$。
证：当n = 0时，T(0) = 1，上式成立。
设当n时上式成立，则有$T(n) = 1 + 2 ^0 + 2^1 +… + 2^{n-1} = 2^n$。
当n+1时，$T(n+1) = 1 + \sum_{j = 0}^nT(j) = 1 + 2^0 + 2^1 + … + 2^{n-1} + 2^{n} = 2^n + 2^n = 2^{n+1}$，得证。

朴素递归算法效率低下的原因

反复求解相同的子问题

动态规划的原理

  1. 对每个子问题只求解一次，并将结果保存下来。对于再次需要子问题的解时，只需查找保存的结果。
  2. 付出额外的内存空间来节省计算时间，时空权衡(time-memory trade-off)。
  3. 若子问题是输入规模的多项式函数，则可以在多项式时间内求解。

动态规划的两种实现

  1. 带备忘录的自顶向下法(top-down with memoization)
    当求解子问题，先检查是否已计算过。若未计算，则进行计算，并将结果保存；若已计算，则直接返回结果。
  2. 自底向上法(bottom-up method)
    确定子问题的规模，使得任何子问题的求解都只依赖更小的子问题。对子问题按规模进行排序，对子问题进行求解时，其依赖的子问题已经求解完毕，并已保存。

NOTE: 由于没有递归函数调用，自底向上的方法的时间复杂度具有更小的系数。

备忘录式自顶向下法的设计小贴士

  1. 用一个数组来装子问题求解的结果。
  2. 初始化该数组时要设置一个正常情况下不可能达到的初始值，从而用来区分该子问题是否已被求解。
  3. 函数首先判断子问题是否已经被求解(对数组对应的值进行检查)
    若已求解，则直接返回结果。
    若未求解，则继续执行。
  4. 检查边界条件，是否达到结束条件。
  5. 若子问题即未求解过，又未达到边界条件，则对子问题进行求解，并将结果保存到数组中。

子问题图
子问题图G = (V, E)的规模可以帮助确定动态规划的运行时间，每个子问题只求解一次，运行时间等于每个子问题求解时间之和。子问题的求解时间与子问题图中对应顶点的度成正比，子问题的数目等于子问题图的顶点数

更进一步
当不仅要求出最佳的结果，还要保存获得最佳结果的解方案。

思路：

创建一个数组，用来保存最佳结果对应的解方案。

联系数据结构相关知识，可以用栈保存结果。即，遍历可能的结果，当出现更好的结果时，先将栈中的解方案弹出，再将新的方案保存。
注：利用栈时，先向栈中预保存一个初始值，避免第一次“最优值”导致对空栈弹出。

应用动态规划方案

  1. 刻画一个最优解的结构特征
  2. 递归地定义最优解的值
  3. 计算最优解的值，通常采用自底向上的方法
  4. 利用计算出的信息构造一个最优解

刻画最优解的结构特征
利用最优子结构性质从子问题的最优解构造原问题的最优解，满足任何最优解都是子问题实例的最优解构成的

递归和动态规划相关题目

给定整数N，返回Fibonacci数列的第N项。

类似题目

  1. 给定整数N，代表台阶数，一次可以跨1个或2个台阶，返回有多少种走法？
  2. 假设农场中成熟的母牛每年只会生1头小母牛，并且永远不会死。第一年农场有1头成熟的母牛，从第二年开始，母牛开始生小母牛。每只小母牛3年之后成熟又可以生小母牛。给定整数N，求出N年后牛的数量。

  1. F(1) = F(2) = 1
  2. F(n) = F(n-1) + F(n-2)
