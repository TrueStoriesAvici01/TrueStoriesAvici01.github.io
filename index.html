数据结构与抽象(一)：组织数据

Posted on 2019-10-29

组织数据

抽象数据类型(Abstract Data Type, ADT): 描述数据集(set)及数据上操作的规范说明。

数据结构(data type): ADT在某种编程语言中的实现。

集合(collection): 包含一组对象的ADT。可能允许重复，可能不允许重复。

  1. ADT包(bag): 由一个无序集合构成，允许有重复值。
  2. ADT线性表: 对项进行编号，可以将项添加到线性表的表尾，也可将项添加到表头或者两项之间。线性表的位置不能表示项的添加时间。
  3. ADT栈(stack): 当从栈中删除项时，删除的是最后添加的项。
  4. ADT队列(queue): 从队列中删除项时，删除的是最早添加的项。
  5. ADT有序表(sorted list): 添加项时，由ADT来确定这个项在有序表中位置。
  6. ADT字典(dicionary): 含有项对，如字典中单词和对应的定义。单词充当关键字(key)，用来查找项。
  7. ADT树(tree): 根据层次组织项。
  8. ADT二叉查找树(binary search tree): 按照层次和排序来组织项。
  9. ADT图(graph): 按照项的之间的关系。

数据结构与抽象(二)：设计类

Posted on 2019-10-29

设计类

封装

封装：将数据和方法放在一个类中，隐藏类实现的细节，不需要知道类定义的内部细节，只需要知道如何为方法提供相应的参数，让方法执行正确的动作。

抽象(abstract)：关注是什么而不是如何的过程。

封装将类定义分成两个部分:

  1. 客户接口(client interface):
    1. 类的公有方法的方法头
    2. 类中公有命名常量
  2. 实现(implementation): 所有数据域和所有方法的定义组成
    1. 私有数据域
    2. 私有常量
    3. 所有的方法定义

Posted on 2019-10-29

The Internet

A nut-and-bolts description

hosts(end systems): Many devices that are being hooked up to the Internet, such as laptops, smartphones, tablets, etc.

The way end systems communicate with each other:

  1. End systems are connected together by a network of communication links(coaxial cable, copper wire, optical fiber…) and packet switches.
  2. Transmission rate: The rate of different links can transmit data, measured in bits/second.
  3. Steps:
    1. The sending end system segments the data and adds header bytes to each segment.
    2. The resulting packages (packets) are sent through the network to the destination end system.
    3. The destination end system reassmbled into the original data.

Git理论基础

Posted on 2019-10-29

Git记录的是什么

  1. Git是将每个版本独立保存
  2. 工作区域、暂存区域和Git仓库

    工作区域 (Working Directory)：日常编程时的工作目录
    暂存区域 (Stage (Index)): 保存改动的目录
    Git仓库 (Repository(HEAD)): 最终存放所有数据的位置

Git 的工作流程

  1. 在工作目录中添加、修改文件
  2. 将需要进行版本管理的文件存放到暂存区域
  3. 将暂存区域的文件提交到Git仓库

Git 管理的文件有三个状态

  • 已修改 (modified)
  • 已暂存 (staged)
  • 已提交 (committed)
