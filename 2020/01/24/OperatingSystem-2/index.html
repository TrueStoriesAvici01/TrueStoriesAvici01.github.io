操作系统概念(二):操作系统结构

2020-01-24

操作系统结构

[toc]

操作系统服务

操作系统服务

  • 用户界面(user interface,UI)：1.命令行界面(command-line interface,CLI)：文本命令;2.批处理(batch interface)：通过执行文件实现;3.图形用户界面(graphical user interface,GUI)：视窗系统。
  • 程序执行：程序装入内存并运行，能够结束执行。
  • I/O操作：用户不能直接控制I/O设备，操作系统提供进行I/O操作的方法。
  • 文件系统操作：读写文件和目录，创建和删除文件，搜索给定文件，列出文件信息，文件访问控制。
  • 通信：一个进程与另外一个进程之间交换信息。同一个计算机的两个进程或不同计算机的进程之间。通过共享内存和消息传递技术实现。
  • 错误检测：知道可能发生的错误，并采取合适的措施确保正确性和一致性的计算。
  • 资源发配:系统为多个用户中的每一个分配资源。
  • 统计:记录资源的使用情况。
  • 保护和安全：控制信息的使用。多独立进程并发执行时，一个进程不能干预另外一个进程。

操作系统的用户界面

方法

  • 命令中断(command interpreter)
  • 图形用户界面(GUI)

命令解释程序

获取并执行用户指定的下一条命令。

执行命令方法：

  1. 本身包含代码。需要转到相应的代码段执行相应的系统调用
  2. 系统程序实现命令。识别命令对应的文件并执行。

图形用户界面

采用桌面(desktop)，提供基于鼠标的窗口和菜单系统。
图标：代表程序，文件，目录和系统功能。

系统调用(system call)

获取操作系统所提供的服务的接口。

  • 程序会大量调用操作系统。
  • 可通过应用程序接口(application programming interface,API)实现。
  • 通过API调用实际的系统调用。

使用API：

  1. 可移植性：在支持同样API的系统编译运行。
  2. 简单：系统调用比API更为细致和困难。

程序设计语言的系统调用接口(system-call interface)
为操作系统的系统调用提供可用链接。
流程：

  1. 系统调用截取API的函数调用
  2. 调用操作系统中相应的系统调用
  3. 系统调用接口调用所需的操作系统内核中的系统调用
  4. 返回系统调用状态及其返回值

向操作系统传递参数

  • 通过寄存器传递参数
  • 参数保存在内存中断块和表（入栈），将块的地址通过寄存器传递

系统调用类型

分类：

  • 进程控制(process control)
  • 文件管理(file manipulation)
  • 设备管理(device manipulation)
  • 信息维护(information maintenance)
  • 通信(commuication)

进程控制

有系统调用用于中断正在执行的程序，或程序碰到问题引起的错误陷阱。会在内存中产生一条错误消息，错误消息写入磁盘，帮助确认问题(debugger)。

控制卡(control card)
批处理系统概念，发现输入错误并要终止时，根据定义的错误级别，自动决定下一个动作。

执行一个进程需要装入和执行另外一个程序。装入程序终止后，需确定控制权返回。

新程序终止时控制权返回现有程序，必须保存现有程序的内存。

单步:为了有助于调试程序，在每条指令运行后能执行一个陷阱。

程序的时间表：程序在某个位置所花的时间。通过定期的定时器中断，每次中断记录计数器的值，频繁的中断可得到所用时间的统计。

单任务系统
启动时调用命令解释程序，不创建新的进程，将程序装入内存，并改写自己的大部分，为新程序提供内存空间。指令指针设为指向程序的第一条指令。程序执行或错误引起一个陷阱或执行系统调用而终止。

多任务系统
命令解释程序在另外一个程序执行时可继续执行。

文件管理

创建和删除文件。
打开并使用文件。
读写，重定位。
管理文件。

设备管理

进程需要一定的资源才能继续运行。
如果有足够的资源，则系统允许请求，控制权返回到用户进程。否则，进程必须等待可用的资源。
一旦请求了设备之后，能如同文件一样对设备进行读写重定位。

信息维护

多数系统调用用于用户程序与操作系统之间传递信息。
操作系统维护所有进程的信息，有些系统调用可访问这些信息。

通信

分类：

  1. 消息传递模型
  2. 共享内存模型

消息传递模型(message-passing model)
通信前，必须先打开连接。接收方允许通信。然后通过系统调用来交换信息，然后终止通信。

共享内存模型(shared-memory model)
系统调用获得其他进程所有拥有的内存区域的访问权。进程需确保不会同时向同一个地方写。

信息传递：适用于少量数据交换，无需避免冲突，容易实现。
共享内存：速度和方便。

系统程序

  • 文件管理(file management)
  • 状态信息(status information)
  • 文件修改(file modification)
  • 程序设计语言支持(programming-lauguage support)
  • 程序装入和执行(program loading and execution)
  • 通信(communication)

操作系统设计和实现

设计目标：定义系统的目标和规格。受到硬件选择和系统类型的影响。

机制与策略：
策略(policy)：决定做什么
机制(mecanism)：决定如何做
策略和机制的分离对于灵活性而言非常重要。
策略可能随着时间有所改变。
策略的变化可能导致底层机制的改变。
需要通用机制，保证策略的改变只需重定义系统参数。

实现

多使用高级语言实现。

  1. 代码编写更快
  2. 更容易理解和调试
  3. 编译技术的改进只需重新编译即可改善操作系统的生成代码
  4. 可在不同的平台移植

操作系统结构

将操作系统分成若干个小模块，每个模块为定义明确的系统部分，有明确的输入，输出和功能。

简单结构

MS-DOS：

  • 应用程序能够访问基本的I/O子程序。
  • 未提供双模式和硬件保护。

UNIX操作系统：

  • 内核：一系列接口和驱动程序。（物理硬件之上和系统调用接口之下)
  • 系统程序

分层法(layered approach)

将操作系统分成若干层，最底层为硬件，最高层为用户界面。

  • 每一层利用较低层所提供的功能来实现。
  • 该层不需要知道如何实现这些操作，只需要知道这些操作能做什么。
  • 每层为较高层掩藏一定的数据结构，操作和硬件的存在。

问题：

  • 难以对层进行清晰的定义
  • 效率稍差，层层调用需要额外的开销

微内核

将非基本部分从内核中移走，将其作为系统程序或用户程序实现。
微内核一般包含最小的进程和内存管理以及通信功能。

功能：
提供客户程序和运行在用户空间的各种服务之间进行通信的能力，通信以消息传递形式提供。

特点：

  • 便于扩充操作系统，不需要修改内核。
  • 便于移植。
  • 更好的安全性和可靠性。
  • 总开销增加而导致的系统性能下降。

模块

内核有一组核心部件，在启动或运行时对附加服务的动态连接。
每个模块能够调用其他任何模块。

虚拟机(virtual machine)

将单个计算机的硬件抽象为几个不同的执行环境。
利用CPU调度和虚拟内存技术。

实现

底层机器模式：

  1. 用户模式
  2. 内核模式

虚拟机软件运行在内核模式。
虚拟机运行在用户模式。
真正机器上的从用户模式到内核模式的切换动作必须在虚拟机上引起虚拟用户模式到虚拟内核模式的转换。

优点

  1. 不同的系统资源具有完全的保护。
  2. 便于操作系统的开发。

VMware

虚拟层是VMware的核心，其将物理硬件抽象为独立的作为客户端操作系统的虚拟机运行。每个虚拟机有自己的虚拟的CPU，内存，磁盘驱动和网络接口。

Java

核心组件：

  1. 编程语言规范
  2. 应用程序接口(API)
  3. 虚拟机规范

Java编程语言

特点：

  • 面向对象，平台无关，分布式和多线程。
  • 对象由类构造来定义。
  • 每个类由编译器生成平台无关的字节码文件(class)
  • 通过垃圾回收(grabage collection)自动管理内存，将不用对象的内存回收并返回给系统。

Java API

  1. Java标准版(SE)：设计桌面程序和小程序。
  2. Java企业版(EE)：设计支持数据库的服务器程序。
  3. Java移动版(ME)：设计运行在小设备上的程序。

Java虚拟机

对计算机的抽象。

组成：

  1. 类加载器(class loader)
  2. Java解释器

流程：
类加载器从Java程序和Java API中加载类文件给Java解释器执行。

Java开发环境

包含

  • Java编译时环境
  • 运行时环境(JRE)

编译时环境将Java源文件转换成字节码(class)文件，运行时文件环境是宿主系统的Java平台。

Java操作系统

JX操作系统使用Java编写。

  • 由域(domain)来组织其系统。
  • 每个域表示一个独立的JVM。
  • 每个域中包含对象创建时发配内存的堆，线程和垃圾回收。
  • 零域是一个负责底层细节的微内核。
  • 零域是由C和汇编语言编写的。
  • 其余部分是由Java编写。
  • 域之间通过门户来运行。
  • 域之间和域内部依赖Java语言的类型安全。

操作系统生成

对于特定的计算机环境，配置和生成系统，称为系统生成(system generation,SYSGEN)

操作系统需要确定：

  • CPU
  • 可用内存
  • 可用设备
  • 操作系统选项

系统启动

引导程序(bootstrap program)/引导装载程序(bootstrap loader)
定位内核，将其装入内存，开始执行。

流程：
CPU接收到重置事件后，定义内存位置的指令寄存器被重新装载，并由次开始执行。

引导程序：

  • 运行诊断程序确定机器状态
  • 确诊通过后，程序按启动步骤继续进行
  • 系统的所有部分被初始化
  • 操作系统启动
