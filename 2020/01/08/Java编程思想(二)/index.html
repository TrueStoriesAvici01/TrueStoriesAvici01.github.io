Java的五个存储数据位置

  1. 寄存器: 最快的存储区。
  2. 堆栈: 位于RAM中，较快，对象引用存于此，需要知道生命周期。
  3. 堆: 通用内存池(RAM中)，存放所有的Java对象，不需要知道生命周期，执行时进行存储分配。
  4. 常量存储: 存放在程序代码内部。
  5. 非RAM存储: 数据在程序没有运行时也存在。如流对象和持久化对象。

Java中的基本类型

基本类型 大小 最小值 最大值 包装器类型
boolean - - - Boolean
char 16 bits Unicode 0 Unicode $2^{16}-1$ Character
byte 8 bits -128 +127 Byte

boolean类型仅定义为取字面值true或false。

对于常用的一系列基本类型，可以通过new在堆中创建一个对象

高精度数字:

  1. BigInteger: 支持任意精度的整数
  2. BigDecimal: 支持任何精度的定点数

Java中的数组

Java 确保数组会被初始化，而且不能在它的范围之外被访问。
以每个数组上少量的内存开销及运行时的下标检查为代价

当创建一个数组对象时，实际上是创建一个引用数组，并且每个引用会自动初始化为一个特定值，其有关键词null，Java可以识别没有指向对象的引用，从而报错。
在使用任何引用前，必须为其指定一个对象

对象的销毁

*变量的生命周期时程序设计中的重要部分*

作用域

作用域(scope): 决定了在其内定义的变量名的可见性和生命周期。
作用域由花括号的位置决定：在作用域里定义的变量只可用于作用域结束之前。

NOTE: Java中空格、制表符、换行不会影响程序的执行结果

对象的的作用域

当用new创建一个Java对象时，其可以存活于作用域之外。

引用s在作用域终点就消失了，而s指向的String对象仍然占据内存空间。

Java利用*垃圾回收器*监视`new`创建的所有对象，释放不会再被引用的对象。

方法、参数和返回值

Java中的方法类似于C中的函数

方法由：名称、参数、返回值、方法体 组成。

NOTES:

  1. 返回类型：调用方法之后从方法返回的值。
  2. 参数列表：要传给方法的参数类型和名称。
    方法名和参数类型唯一确定一个方法

Java中的方法只能作为类的一部分来创建，方法只有通过其定义在的对象才能被调用。

参数列表

参数列表中传递的是引用。

返回值：

  1. 有返回值：方法定义时加上返回值类型，方法内部使用return语句加上要返回的值。
  2. 没有返回值：方法定义时使用void，return只是退出方法。

static关键字

static关键字的作用：

当声明一个事物为static时，这个域或方法与对应的类的任何对象实例无关（不同的对象实例使用的是同一个，该事物只占用一份存储空间）

static的事物的访问：

可以不创建类的任何对象而调用static方法或访问static域。

  1. new创建对象，通过对象引用static变量
  2. 通过其类名直接引用（仅限于static成员）

static的事物定义：

将static放在定义之前，将字段或方法设定为static。

static对象不需要创建任何东西，可以直接使用。
