Java 中形参和实参的分析：

首先定义一个测试类Study：

输出的结果

1

分析

  1. 第一个结果 “0” 是因为如果没有对于没有定义的int型参数默认为0；
  2. 第二个结果 “1” 是因为在调用方法changeI() 时，它的作用是将i 赋值为1，这里的i = 1 可以理解是一个行为，i 并不是一个局部变量，所以在调用结束之后i的值发生变化。
  3. 第三个结果为 “2” 是与第二个类似，只不过是对于数组进行操作。
  4. 第四个结果是 “1”，第五个结果为 “2”。这里需要和结果2进行区分：changeI1(int i)中的i 为局部变量，即：在调用完成之后i的值会消失，从结果可以发现：i = 3这个statement并不是当成第二个结果中对于属性i的一种行为，而是与其输入形参i相对应：对于输入的实参s.j[0]赋值为3，而不会对于Study类中的i进行改变。
