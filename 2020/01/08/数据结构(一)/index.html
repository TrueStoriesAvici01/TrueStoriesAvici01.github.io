数据结构(一)：绪论

2020-01-08

计算

排序(sorting): 将n个整数按大小次序排成一个非降序列的操作。

冒泡算法

局部有序和全局有序

有序序列中每一对相邻元素都是顺序的。
所有相邻元素均顺序的序列，也必然整体有序。

算法步骤:(通过改进局部有序性实现整体的有序)

  1. 一趟扫描交换：从前向后依次检查每一对相邻元素，若发现逆序则交换两者的位置。（长度为n的序列，需要n-1次比较和不超过n-1次交换）
  2. 多次扫描：进行多次扫描交换，直到序列中不再含有任何逆序的相邻元素。

实现

算法

基于特定的计算模型，旨在解决某一信息处理问题而设计的一个指令序列

算法的要素

  1. 输入与输出：

    输入（input）：对所求解问题特定实例的描述
    输出（output）：经过计算和处理之后得到的信息，即针对输入问题实例的答案

  2. 基本操作、确定性与可行性

    算法应可描述为由若干语义明确的基本操作组成的指令序列，且每一基本操作在对应的计算模型中均可兑现。

  3. 有穷性与正确性

    有穷性（finiteness）：任意算法都应在执行有限次基本操作之后终止并给出输出。
    正确性（correctness）：算法所给出的输出应该能够符合由问题本身在事先确定的条件。
    证明算法的有穷性和正确性：
    从适当的角度找出算法具有的某种不变性和单调性。
    单调性：问题的有效规模随着算法的推进不断递减
    不变性：不仅在算法初始状态下自然满足，而且与最终的正确性呼应

  4. 退化与鲁棒性

    退化情况（degeneracy）：算法应该能够处理各种极端的输入实例。
    鲁棒性（robustness）：算法能够尽可能充分地应对退化情况。

  5. 重用性

    算法的总体框架能否方便地推广至其他场合。

算法效率

可计算性（computability）
大量的应用问题不可能设计出必然终止的算法。

难解性（intractability）
问题的最低求解时间成本远远超过目前实际系统所能提供的计算能力。

计算效率

  1. 衡量尺度：时间和空间方面度量算法的计算成本
  2. 归纳算法设计中的一般性规律和技巧

数据结构
以“数据”为研究对象，建立支持高效算法的数据处理策略。

复杂度度量

时间复杂度

时间复杂度（time complexity）：

算法的执行时间可表示为输入规模的一个函数，特定算法处理规模为n的问题所需的时间记为T(n).
在规模为n的所有输入中选择执行时间最长者作为T(n).

渐进复杂度

渐进分析（asymptotic analysis）：

关注时间复杂度的总体变化趋势和增长速度。

环境差异
不同的软硬件对计算算法的时间有影响，应该避免。

基本操作

指令语句可分解为若干次基本操作
将T(n)定义为算法所执行基本操作的总次数：T(n)决定于组成算法的所有语句各自的执行次数，以及其中含有基本操作的数目。

最坏、最好与平均情况

  1. 最坏情况（worst case）：算法在处理任何序列所需的时间不会超过的时间复杂度。
  2. 最好情况（best case）：算法的最少执行时间
  3. 平均情况（average case）：按照某种约定的概率分布对所有输入对应的计算时间加权平均。

大O记号（big-O notation）

若存在正的常数c和函数f(n),使得对任何n >> 2 都有
T(n) <= c·f(n)
可认为f(n)为T(n)增长速度的一个渐进上界，记为：
T(n) = O(f(n))
性质

  1. 对于任一常数c > 0, 有 O(f(n)) = O(c·f(n)) 函数各项正的常系数可等同于1
  2. 对于任意常数a > b > 0, 有O(n^a + n^b) = O(n^a) 多项式中的低次项可忽略，只保留最高次项

大Ω记号（big-omega notation）
对算法的复杂度最好情况做出估计

若存在正的常数c和函数g(n)，使得对于任何n >> 2都有
T(n) >= c·g(n)
则在n足够大之后，g(n)给出T(n)的一个渐进下界：
T(n) = Ω(g(n))

大Θ记号（big-theta notation）
从渐进的趋势看，T(n)介于Ω(g(n))与O(f(n))之间，若出现g(n) = f(n), 可用Θ表示。

若存在正的常数c1 < c2 和函数h(n)，使得对于任何n >> 2都有
c1·h(n) <= T(n) <= c2·h(n)
认为在n足够大后，h(n)给出T(n)的一个确界：
T(n) = Θ(h(n))

空间复杂度(space complexity)

时间复杂度本身就是空间复杂度的一个上界。

复杂度分析

常数O(1)

问题与算法

复杂度

  1. 无论输入的数据多大，仅取其中的三个元素，计较大小后输出，综合的运行时间为：T(n) = O(1)
  2. 称为“常数时间复杂度算法”（constant-time algorithm）或“就地算法”（in-place algorithm）。
  3. 仅含一次或常数次基本操作的算法均属于时间复杂度为O(1)的算法。
  4. 一般不含循环、分支、子程序调用。

对数O(logn)

问题与算法
问题：任意非负整数，统计器二进制展开中1的个数
算法：将n进行二进制展开，每次右移一位，检查最低位是否为1，是则个数加一，否则不加，直至移完。

复杂度

  1. 主要操作来源于while循环。每次循环，n减少为原来的1/2，则需要logn次就结束，其他运算为常数次O(1)，则总的时间复杂度取决于logn。
  2. 此类算法称为具有“对数时间复杂度”（logarithmic-time algorithm）。

对数多项式复杂度：运行时间可以表示为T(n) = O(log^c n)形式的算法（常数c>0）统称为“对数多项式时间复杂度的算法”（polylogarithmic-time algorithm）。

线性O(n)

问题与算法
问题：计算给定n个整数的总和。
算法：循环计算

复杂度

  1. 求解n个数之和，每次将一个数加到累计和中，单次循环的复杂度为O(1)，总共要循环n个，则总的基本操作的时间复杂度为O(n)。
  2. 运行时间可以表示为T(n) = O(n)形式的算法，统称为“线性时间复杂度算法”（linear-time algorithm）。

多项式O(polynomial(n))

运行时间可以表示为 T(n) = O(f(n))的形式，而且f(x)为多项式，则对应的算法称为“多项式时间复杂度算法”（polynomial-time algorithm）。
实际应用中认为是可接受的，若问题存在该范围内的一个算法，则此问题是可有效求解的或易解的（tractable）。

指数O(2^n)

问题与算法

复杂度

  1. 若以输入指数n的二进制位数r = logn作为输入规模，运行时间为O(2^r)。
  2. 运行时间可以表示为 T(n) = O(a^n)形式的算法(a > 1)，均属于“指数时间复杂度算法”（exponential-time algorithm）。

通常认为：指数复杂度算法无法真正应用到实际问题，不是有效算法。对于不存在多项式复杂度算法的问题，称为难解的（intractable）问题。

复杂度层次

从低到高: O(1) < O(log^ n) < O(loglog n) < O(log n) < O(sqrt(n)) < O(n) < O(nlog^ n) < O(nloglog n) < O(n^2) < O(n^3) < O(n^c) < O(2^n)

输入规模

用以描述输入所需的空间规模。
以输入参数n本身的数值作为基准的复杂度称为伪复杂度。

递归

递归：函数和过程进行自我调用。
递归调用（recursive call）：某一方法调用自身。
是对实际问题中反复出现的结构和形式做高度概括。

线性递归

数组求和

线性递归（linear recursion）：每一层次上至多只有一个实例，且构成一个线性的次序关系。
减而治之（decrease-and-conquer）：递归每深入一层，待求解问题的规模都缩减一个常数，直至减少到一个小问题。

递归分析

递归跟踪（recursion trace）

  1. 算法的每一递归实例都表示为一个方框，注明该实例调用的参数
  2. 若实例M调用实例N，则在M与N对应的方框之间添加一条有向联线

NOTES:

  1. 递归实例创建、销毁的时间近似于常数。
  2. 只需要统计各递归实例中非递归调用部分所需的时间。
  3. 单词递归的基本操作数和递归的层数。

递归方程（recurrence equation）
通过对递归模式的数学归纳，导出复杂度定界函数的递推方程（组）及其边界条件，对递归方程（组）求解。

寻找相邻两次递归的递推关系，对其进行求解。

递归模式

多递归基
对每一类可能出现的非凡情况，都需设置对应的递归基，保证有穷性。

多向递归
递归调用可能有多种选择的分支。

求解幂函数的算法
power2(n) = 1 if n=0
power2(n/2)^2 * 2 if n > 0 and odd
power2(n/2)^2 if n > 0 and even

递归消除

空间成本
递归算法需要更多的空间，影响运行速度。

尾递归及其消除

尾递归（tail recursion）：递归调用在递归实例中以最后一步操作的形式出现。只有当该算法任一实例都终止于这一递归调用时，才属于尾递归。

二分递归

分而治之（dividi-and-conquer）
将问题分解为若干规模更小的子问题，再通过递归机制分别求解。

多路递归（multi-way recursion）
每一递归实例都可能做多次递归。
若是将问题一分为二，则为“二分递归”（binary recursion）

数组求和
思路：以居中的元素为界将数组一分为二，递归地对子数组分别求和，将子数组之和相加。

Fibonacci数列
fibonacci数列第n项fib(n)的递归形式：
fib(n) = n 若 n <= 1
fib(n-1) + fib(n-2) 若 n >= 2

算法：

Fibonacci数列需要运行O(2^n)的时间复杂度。
原因：计算过程中出现的递归实例的重复度高。

优化策略

  1. 制表/记忆（tabulation/memoization）自顶向下，每遇到一个子问题，首先检查其是否计算过，以期通过直接查阅记录获得解答，避免重复计算。
  2. 动态规划（dynamic programming）从递归基出发，自底而上递推地得出各子问题的解，直到最终原问题的解。

（不懂）：
Fibonacci数列：线性递归

时间复杂度为 O(n)。

Fibonacci数列：迭代

时间复杂度为 O(n)，常数规模的空间复杂度。

抽象数据类型

封装：将数据项与相关的操作结合为一个整体，将数据结构的外部特性与其内部实现分离，提供一致而标准的对外接口。

抽象数据类型（abstract data type，ADT）：通过封装，数据集合与对应的操作所构成。
