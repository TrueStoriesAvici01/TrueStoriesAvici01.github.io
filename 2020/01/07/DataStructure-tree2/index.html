高效二叉查找树

最优二叉查找树

定义

  • 二叉树中有n个元素，分别表示为$a_1,a_2,…,a_n$，满足$a_1 < a_2 < … < a_n$。
  • 查找$a_i$的概率为$p_i$。
  • 查找在$a_i$和$a_{i+1}$处失败的概率为$q_i$。
  • $T_{ij}$为关键字$a_{i+1}, …, a_j$构成的最优二叉查找树，$T_{ii}$：空树。
  • $c_{ij}$：$T_{ij}$的代价，$c_{ii} = 0$。
  • $r_{ij}$：$T_{ij}$的树根。
  • $w_{ij}$：$T_{ij}$的权值，$w_{ij} = q_i + \sum_{k=i+1}^j(q_k + p_k)$。

目标函数
设$T_{ij}$的根为$k,\ i < k \leq j$，则有：
$c_{ij} = p_k + c_{i,k-1} + c_{k, j} + w_{i, k-1} + w_{k,j} = w_{ij} + c_{i,k-1} + c_{kj}$
$w_{i,j} = w_{i,j-1} + w_{j,j} + p_j$

求解
先求解$w_{i,j}$，再计算对应的$c_{ij}$，从而确定$T_{ij}$对应的根。
$i,j$的确定顺序是：先计算$j = i$，再计算$j = i+1$，再计算$j = i + 2$ …

AVL树

定义

高度平衡：空树是平衡树，或：T不是空树，有左子树$T_L$和右子树$T_R$，则$T$高度平衡当且仅当：1. $T_L$和$T_R$高度平衡；2. $|h_L - h_R| \leq 1$，$h_L$、$h_R$分别是$T_L$，$T_R$的高度。

平衡因子：二叉树结点$T$的平衡因子$bf(T)$定义为$h_L-h_R$，$h_L$，$h_R$分别是左，右子树$T_L$，$T_R$的高度。AVL树中每个结点$T$的$bf(T) = -1,0,1$。

操作

插入导致的不平衡，需要四种旋转：LL，RR，LR，RL。
设新结点为N，其最近的祖先平衡因子为$\pm 2$的结点为A。

  • LL：新结点N插入到A的左子树的左子树
  • LR：新结点N插入到A的左子树的右子树
  • RL：新结点N插入到A的右子树的左子树
  • RR：新结点N插入到A的右子树的右子树

LL和RR，LR和RL是对称的，可以从其中两个推出另外一个

是否需要调整
向AVL树插入一个结点，首先需要确定是否需要进行调整操作，若需要，则再判断何种旋转。

判断依据：
从插入新结点的父结点开始，沿着向根结点的路径中各个祖先结点，分别检查插入新结点的影响：

  • 对于其父结点，若没有孩子结点，调整插入结点后父结点的平衡因子并依次向上调整祖先结点的平衡因子；若有孩子，则调整父结点的平衡因子，此时AVL树已平衡。
  • 对于需要调整祖先结点的平衡因子的情况：若插入结点位于祖先结点的左子树，则平衡因子+1；否则，平衡因子-1。
  • 当调整的祖先结点为根结点或调整后平衡因子为0，+2，-2时，停止向上调整。
  • 对于停止调整的结点，若其平衡因子为+2 或 -2时，需要以此为参考进行旋转。

LL旋转
设：结点A和结点B需要进行LL旋转，其中B是A的左子结点。Br是结点B的右子树。

步骤：

  • Br成为A的左子树
  • A成为B的右子结点
  • 指向A的指针改成指向B

旋转前后子树的高度不变，所以不需要向上再进行调整。

适用：
插入的结点位于需要调整结点的左子树，并且插入结点是其父的左子结点

LR旋转
设：需要对结点A，B和C进行调整。其中：
B是A的左子结点，C是B的右子结点，Cl是C的左子树，Cr是C的右子树，新插入的结点在C的子树中。

步骤：

  • B的右子结点指向Cl
  • A的左子结点指向C
  • C的左子结点指向B
  • A的左子结点指向子树Cr
  • 指向A指针指向B
  • B的右子结点指向A

适用：
插入结点位于需要调整结点的左子树，并且插入结点是其父的右子树。

插入

  • 首先：依据二叉查找树的性质插入结点。
  • 判断是否需要旋转。
  • 若需要旋转，则判断旋转种类，在按照对应的步骤进行旋转。

删除

  • 若删除的结点是叶子结点，则直接删除。
  • 若删除的非叶子结点，则用其后继结点替代，等价于删除后继结点（若有右子树，则为右子树的最左端结点；若没有，则为祖先结点中距离其最近的并且有右子树，该右子树的最左端端点；若都没有右子树，则使用其左子结点替代）
  • 然后从被删除结点的父结点开始，向上调整各祖先结点的平衡因子，判断是否需要旋转，若需要，则按照对应的旋转的方式进行调整。

复杂度

操作 时间复杂度
查找 O(log n)
删除 O(log n)
插入 O(log n)

红黑树

为什么使用红黑树

  1. 插入一个结点引起的不平衡，AVL和RB-Tree都是最多使用两次旋转操作，为O(1)。而删除结点引起的不平衡，最坏情况下，AVL需要维护从被删结点到根结点的路径上所有结点的平衡，为O(log n)。而RB-Tree最多需要三次选择，为O(1)。
  2. AVL结构更加平衡，插入、删除容易引起不平衡，在大量需要插入，删除操作的情况下，需要的旋转的操作更加频繁，而RB-Tree效率更高。
  3. RB-Tree的统计性能更好。

定义

是二叉树，其中每个结点有颜色属性：红色或黑色。
并满足以下性质：

  • 根结点与外部结点都是黑结点
  • 所有从根到外部结点的路径上都不存在连续两个红结点
  • 所有从根到外部结点的路径上黑色结点的个数都相同

外部结点：一个结点的左右子结点为空，则用一个结点替代。

性质

  • 从根到外部结点的路径长度(length)是该路径上的结点个数，则对于任意两个从根到外部结点的的长度$l_1$和$l_2$，总满足：$l_1 \leq 2l_2$。

表示

红黑树中每个结点含有的属性有：

  • 颜色(color)
  • 键值(key)
  • 左子结点(left)
  • 右子结点(right)
  • 父结点(parent)

操作

查找

与AVL树的查找相同，逐个结点进行比较，若大于当前结点，则从其右子结点开始查找;否则从其左子结点开始查找。

插入

注：调整时以参考结点作为调整操作的默认点，插入时一般以新插入的结点为参考结点。

步骤：

  • 若插入时树的根结点为空，或其父结点为黑色，则直接插入。
  • 若插入结点（参考结点）的父结点为红色，则：
    • 若父结点的兄弟结点（叔结点）为红色，则：设父结点和叔结点为黑色，祖父结点为红色。然后视祖父结点为参考结点，继续向上进行调整。
    • 若其父结点的兄弟结点为黑色，则：
      • 若参考结点为其父结点的左子结点,父结点是祖父结点的左子结点，则：对掉祖父结点和父结点的颜色;右旋祖父结点。
      • 若参考结点为其父结点的右子结点，父结点是祖父结点的右子结点,则：对掉祖父结点和父结点的颜色;左旋祖父结点。
      • 若参考结点为其父结点的右子结点,父结点是祖父结点的左子结点,则：左旋父结点（此时参考结点成为原来的父结点的父结点，原来的父结点成为参考的左子结点，参考结点的新的父结点为原来的祖父结点);对掉参考结点与新的父结点（原祖父结点）的颜色;右旋参考结点的新父结点。
      • 若参考结点为其父结点的左子结点，父结点是祖父结点的右子结点,则：右旋父结点;对掉参考结点与新父结点的颜色;左旋参考结点的新父结点。

删除

注：调整时以参考结点作为调整操作的默认点，删除时以实际删除的结点作为参考结点。
实际删除的结点为D，其后继结点（右子结点）为N，D的父结点为P，D的兄弟结点为S，S的两个孩子结点为Sl和Sr。

步骤：

  • 若D为红色，则：直接删除D。
  • 若D为黑色,N为红色,则：用N替换D，并将N设为黑色。
  • 若D为黑色,N为黑色（外部结点）,则：（以下用N代替D作为初始参考点)
    • 若S为红色,则：旋转结点P，S;对掉P，S的颜色;从参考点继续进行调整。
    • 若S为黑色,则：
      • 若P为黑色，则：将S的颜色改为红色;将P作为参考点，向上进行调整。
      • 若P为红色，则：
        • 若Sl和Sr为黑色,则：对掉P和S的颜色。
        • 若Sl为红色,Sr为黑色,则：旋转Sl和S;对掉Sl和S的颜色;从参考点开始继续调整。
        • 若Sl为任意颜色,Sr为红色,则：旋转P，S;对掉P，S的颜色;设Sr的颜色为黑色。
  • 根结点的颜色设为黑色。

伸展树(Splay tree)

特点

  • 单次访问的复杂度可能为O(N)
  • 连续访问的平摊复杂度为O(log N)
  • Splay树基于的假设是：单次的O(N)是可接受的，但是其发生的频率较低，在一系列的操作下，单次操作的平摊的复杂度是O(log N)。
  • Splay树的基本想法是：当一个结点被访问后，会被调整到根结点处。
  • 应用Splay树的一个实际情况：当一个结点被访问后，其很可能在短时间内被再次访问。
  • Splay树的一个优势是：不需要保存高度或平衡因子等信息，节省空间，使用方便。

操作

伸展(Splaying)

分为四种情况，其中两两对称。
设待调整的结点为X，其父结点为P，祖父结点为G。

  1. X为P的左孩子，P为G为左孩子。
  2. X为P的右孩子，P为G的右孩子。
  3. X为P的左孩子，P为G的右孩子。
  4. X为P的右孩子，P为G的左孩子。

1 和 2 对称，3 和 4 对称。

仅考虑非对称情况，有两种splaying操作：

  • Zig-zig
  • Zig-zag

Zig-zig
适用于情况1，2.

操作：

  • 对G进行单旋，使P成为G的父结点。
  • 对P进行单旋，使X成为P的父结点。

Zig-zag
适用于情况3，4.

操作：

  • 对P单旋，使X成为P的父结点。
  • 对G进行单旋，使X成为G的父结点。

特点
将访问的结点调整成根结点，并且大多数结点的深度减少了约一半。

访问结点

操作：

  • 首先查找到结点所在的位置。
  • 若结点为根结点，则直接访问，无需splaying。
  • 若结点只有父结点,则：对父结点进行单旋，使父结点成为其孩子。
  • 若结点有父结点和祖父结点，则：判断祖父，父以及该结点属于的情况，然后进行对应的splaying。重复对结点进行splaying，直到结点成为根结点。

插入

操作：

  • 首先查找插入位置的父结点，将父结点splaying到根结点。
  • 若插入的结点大于根结点，则：插入的结点的左孩子是根结点，右孩子是根结点的右子树，插入的结点成为根结点。
  • 若插入的结点小于根结点，则：插入的结点的右孩子是根结点，左孩子是根结点的左子树，插入的结点成为根结点。
  • 若插入的结点与根结点相等，则无需插入。

删除

操作：

  • 首先查找要删除的结点，若不存在，则将其前序结点splaying到根结点。
  • 若存在，则使用后序结点进行替换，再对该结点splaying至根结点。
