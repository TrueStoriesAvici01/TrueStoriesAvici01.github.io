路由协议

路由控制的定义

路由控制

路由器根据控制表（Routing Table）转发数据包，由收到的数据包中的目标IP地址和路由控制表确定下一个路由器。

分类：

  • 静态路由
  • 动态路由

静态路由
有使用者手工操作，新增网络需要手动增加，发生故障需要手动设置恢复。

动态路由
设置好路由协议。新增路由器则只需在该路由器进行动态路由设置即可。发生故障路由器可自动选择可行的路径。
路由器将自己已知的网络连接信息发送给其他路由器，直到整个网络都了解。

路由控制范围

自治系统(AS: Autonomous System)/路由选择域(Routing Domain)：
制定自己的路由策略，并以此为准的一个或多个网络群体。

分类：

  • 外部网关协议EGP()
  • 内部网关协议IGP(Interior Gateway Protocol)

依据EGP在区域网络之间进行路由选择。
根据IGP在区域内部进行主机识别。

路由算法

分类：

  • 距离向量（Distance-Vector）算法
  • 链路状态（Link-State）算法

距离向量算法

根据距离和方向决定目标网络或目标主机位置。

路由器之间可以相互交换目标网络的方向及其距离。

链路状态算法

了解网络整体连接状态的基础上生成路由控制表。
每个路由器保持相同的信息。

主要路由协议

路由协议 下一层协议 方式 适用范围 循环检测
RIP UDP 距离向量 域内 No
OSFP IP 链路状态 域内 Yes
EGP IP 距离向量 对外连接 No
BGP TCP 路径向量 对外连接 Yes

RIP

RIP(Routing Information Protocol)

广播路由控制信息

  • 30秒一次，将自己已知的路由信息广播出去
  • 将已知的路由信息记过一跳之后继续广播
  • 逐步传播路由信息
  • 等待6次，若未收到路由信息，则关闭连接

根据距离向量确定路由

距离（Metrics）的单位为跳数，即：所经过路由器的个数。

若同一个网络如果有多条路径，则选择距离较短的一个。

使用子网掩码时的RIP处理

  • 从接口的IP地址对应的分类得到的网络地址后，与流过此路由器的IP地址对应的分类进行比较。如果相同，则以接口的网络地址长度为准。
  • 若两者不同，则以IP地址的分类确认网络地址长度为准。

RIP中路由变更时处理

基本行为

  • 将所知的路由信息定期进行广播
  • 一旦认为网络断开，数据将无法流过路由器，其他路由器可以得知网络已断

存在的问题
无限计数（Counting to Infinity）：收到自己发出去的消息，对于断开的网络未及时更新而收到之前发送给其他路由器的路由信息。

解决

  • 最长距离不超过16,从时间上进行控制
  • 规定路由器不再将所收到的路由器消息原路返回给发送端（水平分割（Split Horizon））。

对于有环路的网络，可采取：

  • 毒性逆转(Poisoned Reverse)：当网络中发生断开时，将这个无法通信的消息传播出去，即发送一个距离16的消息。
  • 触发更新(Triggered Updatt)：当路由消息发生变化时，不是等待30秒，而是立即发送出去。

RIP2

特点

  • 使用多播
  • 支持子网掩码
  • 路由选择域(同一个网络中使用多个逻辑上独立的RIP)
  • 外部路由标志(通过BGP获得的路由信息传递给AS)
  • 生份验证密钥

OSPF

OSPF是链路状态型路由协议

  • 路由器之间通过交换链路状态生成网络拓扑信息，由拓扑信息生成控制表。
  • 给每条链路赋予权值，选择权重最小的路径作为最终的路由。

基础知识

包名 功能
问候(HELLO) 确认相邻路由器，确定指定路由器
数据库描述(Database Description) 链路状态数据库的摘要信息
链路状态请求(Link State Request) 请求从数据库中获取链路状态信息
链路状态更新(Link State Update) 更新链路状态数据库中的链路状态信息
链路状态确认应答(Link State Acknowledgement) 链路状态信息更新的确认应答

工作原理

流程：

  • 通过发送问候包确认是否连接
  • 每个路由器通过数据库描述包相互发送路由器摘要信息和版本信息
  • 若版本较老，则首先发送一个链路状态请求包请求路由控制信息
  • 然后通过链路状态更新包接受路由状态信息
  • 最后再通过链路状态确认包通知已接受路由控制信息

HELLO包：
每10秒发送一个，允许等待3次，第4次认为连接断开，进行连接断开或恢复连接操作。发送链路状态更新包。

链路状态更新包：

  • 网络LSA：以网络为中心生成的信息
  • 路由器LSA：以路由器为中心生成的信息

路由器由Dijkstra算法生成相应的路由控制表。

将区域分层化进行细化管理

区域

  • 将连接在一起的网络和主机划分成小组，使一个自治系统(AS)内可以拥有多个区域。
  • 多个区域需要连接一个主干区域(Backbone Area)
  • 连接区域和主干区域的路由器为边界路由器
  • 区域内部的路由器为内部路由器
  • 只与主干区域连接的路由器为主干路由器
  • 与外部连接的路由器为AS边界路由器
  • 区域内路由器持有本区域网络拓扑
  • 区域外路径只能从区域边界路由器获知
  • 区域边界路由器将自己到区域内的路由器的距离信息发送给其他区域路由器
  • 区域出口的边界路由器只有一个，为末端区域，不需要发送区域外的路由信息

BGP

BGP(Border Gateway Protocol)，边界网关协议：

  • 连接不同组织机构的一种协议。
  • 属于外部网关协议(EGP)。
  • 用于ISP之间相连接。

ISP，区域网络将每个网络域分成一个个自治系统(AS:Autonomous System)，每个AS发配一个AS编号。

BGP是路径向量协议

BGP扬声器:
根据BGP交换路由控制信息的路由器。

AS路径信息访问列表(AS Path List)：
BGP数据包传输时，生成一个中途经过所有AS的编码列表。
对于多条路径，选择AS数量最少的路径。

路径向量型协议(Path Vector)：
更具所要经过的路径信息访问列表进行路由控制。

MPLS

转发IP数据包时使用了标记交换技术，对每个IP包都设定一个标记，然后根据标记再进行转发。

MPLS(Multi Protocol Label Switching)：多协议标记交换技术。

网络基本动作

标记交换路由器(LSR,Label Switching Router)：MPLS网络中实现MPLS功能的路由器。
标记边缘路由器（LER,Label Edge Router）：与外部网络连接的LSR。

  • 数据包进入MPLS时，其IP首部的前面被追加32比特垫片头。
  • MPLS网络中，根据垫片头中的标记进行下一步转发。
  • 数据离开MPLS时，垫片头被去除。

标记交换路径(LSP,Label Switch Path)：MPLS中目标地址和数据包都要通过由标记决定的同一个路径。

分类：

  • 一对一的点对点LSP
  • 一对多绑定的合并LSP

扩展：

  • 采用标记发配协议(LDP,Label Distribution Protocol)
  • 通过协议捎带信息

优势:

  • 转发速度快。(使用固定长度的标记信息)
  • 利用标记生成虚拟的路径。
