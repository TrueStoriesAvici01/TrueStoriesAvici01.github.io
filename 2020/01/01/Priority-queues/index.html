优先队列

堆

用处：
实现优先级队列。

特点：
删除操作以队列元素的优先级高低为准。
如：总是删除优先级最高或最低的元素。
插入操作不限制优先级。

大根堆(max-heap)

定义：
是完全二叉树或大根树。树种每个结点的关键字都不小于孩子结点的关键字。

小根堆(min-heap)相反（每个结点的关键字都不大于孩子结点的关键字）

注：大根堆是完全二叉树，适合使用数组存储。

操作：

插入
插入元素之后，可能不满足定义，需要检查调整次序。
从插入的位置开始，向上检查上下两个元素的大小关系是否满足要求，满足则继续向上查找，否则对调两者的次序再向上检查。直到检查到树根。

删除
从删除的位置，寻找其子结点中最大（或小根堆则为最小）结点进行替换，直到没有子孙。若被移动的叶子结点不是树的最后一个结点（即，最后一层的最右结点），则将最后结点移到该叶子结点。

单端优先级队列与双端优先级队列

单端优先级队列

在最小（大）值优先级队列的基础上进行扩展：

  • 易融合(meld)优先级队列：把两个优先级队列合并成一个优先级队列。（左倾树，二项式堆）
  • 删除队列中任意元素，减小（增大）队列中元素的优先级。（Fibonacci堆，匹配堆）

左倾树

分类：

  • 高度左倾树(HBLT: height baised leftist tree)
  • 权值左倾树(WBLT: weight biased leftist tree)

扩展二叉树：所有空子树用方结点表示，称为外部结点；原结点称为内部结点。

x为扩展二叉树的结点，sd(x)为x到外部结点的最短路径，有：

  • 若x是外部结点，sd(x) = 0
  • 若x是内部结点，sd(x) = 1+min{sd(left(x)),sd(right(x))}

left(x)(right(x))是x的左(右)孩子。

定义：

  • 左倾树：若非空，左倾树中所有内部结点x，有：
    sd(left(x)) <= sd(right(x))
  • 小根左倾树（大根左倾树）：每个结点的关键字值都不大于（小于）其孩子结点（若存在）的关键字。

操作：
插入，删除最小值（最大值）要使用融合操作。
插入：构造只含一个元素的小根左倾树，将两个树融合。
删除：将非空树的树根的左右子树融合，则最小值（最大值）则被删除。

融合操作：

  • 初始指针分别指向两个小根左倾树a和b的树根。
  • 比较指针指向的两个结点的大小，记录后，两个指针指向当前结点的右孩子结点。
  • 较大的节点作为较小节点的右结点，原结点的右子树成为新树。
  • 迭代，直到有一个指针指向的结点没有右孩子。
  • 从结束的结点开始，向上检查其父结点是否满足最小堆左倾树要求，不满足则对调两个子树。最终结束于根节点。

性质：

  • $n \geq 2^{sd(x)} - 1$
  • 从根沿最右一条路径到外部结点的路径长度是所有从根到外部结点路径长度的最小值，为$sd(r) \leq \log_2(n+1)$

r：树根；n：树的内部结点个数。

权值左倾树

结点x的权值w(x)为x子树的结点数目。（外部结点的权值为0）

  • 若x为外部结点，w(x) = 0
  • 若x为内部结点，w(x)为孩子结点权值之和再加1

定义

权值左倾树(WBLT)：每个结点的左孩子结点的权值w大于等于右孩子结点的权值w。

性质

  • $rd(x) \leq log_2(w(x) + 1)$

x：任意的内部结点
rd(x)：从x沿最右支路到外部结点的路径长度。

差异

权值左倾树的融合操作只需一次自顶向下的操作。而高度左倾树在融合过程中结点的新的路径长度位置，需要随后更新。

二项式堆(Binomial Heap)

定义
最小二项式堆（最大二项式堆）：小根树（大根树）的聚集。
最小二项式树称为B-堆。

结点组成：
* degree：存放该结点所有孩子的个数
* child：指向该结点的任意一个孩子
* link：指向孩子兄弟结点的单向循环链表（结点的所有孩子存放在一个单向循环链表中）
* data：结点的数据域

二项式树：

  • 序(order)为0的二项式树有一个结点
  • 序为k的二项式树是由两个序为k-1的二项式树构成，其中一个一棵树作为另外一个树的根结点的左子树。

序为k的二项式树的性质：

  • 有$2^k$个结点
  • 树的深度为k
  • 深度为i的结点有C(k,i)个结点(i = 0,…,k)
  • 根的度为k，其子结点的度分别为k-1, k-2,…,0

二项式堆

是二项式树的集合，每个二项式树满足小根堆特性（父结点小于等于其子结点），并且不存在相同度的二项式树。

性质：
n个结点的二项式堆共有$\log_2(n)$个二项式树。

操作：
所有操作都基于合并union()，即，将两个二项式堆合并成一个二项式堆

  • 插入
  • 查找最小关键字
  • 抽取最小关键字
  • 减少关键字的值
  • 删除关键字
  • 合并两个二项堆

插入:
构造一个由新结点构成的堆,然后调用合并算法union()将两个堆进行合并.

查找最小关键字:
只要遍历包含根结点的单向链表,找出最小关键字.

抽取最小关键字:
若树根链表为空,则无最小关键字.
若非空,则将查找到最小关键字,然后其子树构造成一个新堆,在原堆中将对应的二项式树删除,然后调用合并算法将两个堆合并.

减少关键字的值:
指针指向减少值的结点,然后比较其父结点和当前结点的大小,若不满足小根堆性质,则调换两个节点的值,指针移到父结点.依次比较父子结点值的关系,直到指针移到根结点为止.

删除关键字:
若关键字对应的结点为叶子节点,则直接删除.
否则,其子节点构成新堆,对原结点及其子结点删除,调用合并算法将两个堆进行合并.

合并算法:

  • 将两个的根结点链表合并成一个新链表.
  • 按照根结点的序的大小对二项式树进行排序.
  • 按序对各个二项式树进行处理:
  • 若当前二项式树的序号唯一,则不作处理.
  • 若相同序号的二项式树有两个,则将两个二项式树合并(根节点的关键字的值较大的树作为较小的树的树根的左子树),然后从合并后的新树开始,向后检查.
  • 若相同序号的二项式树有三个,则将位于后两个二项式树进行合并,然后从合并后的新树开始,向后检查.

Fibonacci堆

由一系列小根树构成.(每个结点的关键字大于等于其父结点的关键字)

构成:

  • 结点构成
  • 堆构成

结点构成:

  • key: 结点的键值(数据域)
  • degree: 结点的度(子结点的个数)
  • parent: 结点的父结点
  • child: 结点的子结点个数
  • mark: 是否失去孩子

堆构成:

  • nodeNum: 堆中结点的个数
  • min: 指向根链表中最小的节点.
    根链表: Fibonacci堆中所有树的树根构成的一个链表.

注:
多个结点构成树,多棵树构成Fibonacci堆.

操作:

  • 创建一个新堆
  • 插入一个结点
  • 寻找最小结点
  • 两个堆合并
  • 抽取最小结点
  • 关键字减值
  • 删除一个结点

创建一个新堆
初始化为:

  • 结点个数为0
  • min指针指向null

插入一个结点
就是将新节点直接插入到根链表中

  • 初始化一个结点
  • 若 min指向null
  • 则 将min指向新结点
  • 否则 将新结点插入原堆的根链表中; 若 插入的结点小于当前堆的min,则 min指向新结点
  • 节点个数加一

寻找最小结点
直接返回min指针指向的节点

两个堆合并
将两个堆的根链表合并

  • 创建一个新堆
  • 新堆的min指向第一个堆的min
  • 将两个堆的根链表合并
  • 若 堆1的min为null 或 堆2的min非空并且堆2的min小于堆1的min
  • 则 新堆的min指向对2的min
  • 新堆的结点个数为两个旧堆之和
  • 返回 新堆

抽取最小结点
删去最小结点,将最小结点的孩子加入根链表中,并对堆中的树进行合并

  • 创建新指针z指向min结点
  • 若 z非null
  • 则 将z的子结点加入到根链表中,然后从根链表中删除z
  • 并且 若 z是其自己兄弟
  • 则 令min为null(堆为空)
  • 否则 令min指向z的右兄弟,并调用辅助程序Consolidate()调整
  • 结点个数减一
  • 返回 z

辅助程序Consolidate():

  • 创建新数组A[],并初始化
  • 对于 根链表中的各个结点
  • 令x指向当前结点,d为x的度
  • 若 A[d]非null,
  • 则 令y指向A[d],并调用link()将y合并到x,令A[d]为null,使得A[d+1]指向x
  • 令min指向null
  • 对 A[i]中每个元素
  • 若 A[i]非空
  • 则 将A[i]指向的根结点加入根链表H中,并且若 指向的结点小于min结点,则min指向新的结点.

辅助程序link()

  • 从根链表移除y
  • 使y称为x的孩子,增加x的度
  • y的mark设为false

配偶堆

定义
最小配偶堆是一颗小根树.

分类:

  • 最小配偶堆
  • 最大配偶堆

操作:

  • 融合与插入
  • 减小关键字值
  • 删除最小元
  • 删除任意元

融合与插入
比较两棵小根树的树根关键字,根关键字较大的树成为根关键字较小的树的最左边子树.

减少关键字值
若N是根或N的新关键字值小于其父结点的关键字,则

  1. 把以N为根的子树从原树中删除,堆中现有两棵小根树
  2. 融合两棵小根树

删除最小元
将树根删除,若树根的子树超过2,则融合成一棵:
两遍配偶堆融合

  1. 从左到右两两融合小根树
  2. 从右到左,将小根树依次融合成一颗.
    多遍配偶堆融合
  3. 是FIFO队列,将所有树入队
  4. 从队列头出列两棵树,融合成一棵树后放置列尾,重复直至一棵树.

删除任意元

  1. 把以N为根的子树从原树中删除
  2. 删除结点N,然后把N的子树融合成一棵最小树.若配偶堆是两遍配偶堆,则使用两遍融合法,否则使用多遍融合法.
  3. 将1,2的两棵树融合成一颗小根树.

对称最小-最大堆

定义
对称最小-最大堆(SMMH:symmetric min-max heap):
是一颗完全二叉树,除根结点外,每个结点都包含数据元素.

element(N)表示以N为根的子树,但不包含N的所有数据,若非空,则满足:

  • N的左孩子结点为element(N)的最小元.
  • N的右孩子结点为element(N)的最大元.

性质:

  • 每个节点都小于等于右兄弟结点.
  • 结点N有祖父,祖父结点左孩子小于等于N中的数据.

表示:
一维数组存储,位置0和1不放数据.
最小元存放在根的左孩子中,最大元存放在根的右孩子中.

操作:
插入:

  1. 将完全二叉树的长度加一,为待插入的数据创建一个新结点E.
  2. 若E为其父的左子结点,则依次检查E是否小于其祖父的左子结点,小于则两者对掉,否则向上检查,直到其祖父为根结点;若E为其父的右子结点,依次检查E是否大于其父,若大于,则两结点对掉,否则向上检查,直到其父为根节点.

删除最小元

  1. 将树的最后一个结点x删除,设置最小结点为E.
  2. 比较E的左孩子(若存在)和其父的右孩子的左孩子结点(若存在)的大小,若x小于两者中最小的,将E和两者较小的进行对掉.
  3. 循环直到没有两个结点都不存在或没有比x小的结点,将x插入E所在的位置.
  4. 若E有右兄弟结点,比较两个结点,若E较大,则调换两结点.

区间树

定义
是一颗完全二叉树.
树中的每个结点,除最后一个结点外,都存放两个数据.a <= b.
每个结点P的左,右孩子(若存在)结点的区间被结点P的区间所包含.

性质

  1. 结点的左端点构成小根堆,结点的右端点构成大根堆.
  2. 若堆中元素为奇数,则最后一个结点只含一个元素,且该元素同时属于大,小根堆.
  3. 若根有两个元素,则左端点为区间堆的最小值,右端点是区间堆的最大值.若只有一个元素,则区间堆只有一个元素.
  4. n个元素的区间堆高度为O(log n)

操作:
插入

  1. 若堆中的元素个数为偶数个,则在堆中创建一个新结点.
  2. 若堆中的元素个数为奇数个,则直接在最后一个结点中插入元素.
  3. 从插入元素的结点开始,检查当前结点和其父结点的关系.
  4. 若插入的元素小于其父结点的最小值(或大于其父的最大值),则调换两个值,并向上继续检查.
  5. 若插入结点在其父结点区间内,则不需要调整.

删除最小元

  1. 若为空堆,则无需删除.
  2. 若只有一个元素,则返回该元素,将堆置空.
  3. 若不只一个元素,则返回根结点的左端数据,并删除该数据.
  4. 将堆的最后一个结点的左端数据删除,并插入到根结点的左端.若此时最后结点为空,则删除最后结点.
  5. 从根结点向下,检查两个孩子结点的左端数据的大小,若当前结点小于两个孩子结点左端数据的最小值,则交换两个数据.
  6. 若存在左端数据大于右端数据,则交换左右端数据,并开始比较其孩子结点的右端数据的大小,若大于孩子右端数据的最大值,则交换两个数据.
  7. 向下检查,直到叶子结点为止.

复杂度
取最大,小元:O(1)
插入,删除最大,小元:O(log n)
初始化: O(n)

双端优先级队列(DEPQ)

特点：

  • 返回最小优先级的元素
  • 返回最大优先级的元素
  • 插入任意优先级的元素
  • 删除最小优先级的元素
  • 删除最大优先级的元素

应用：
适应外部排序的快速排序。

快速排序：

  1. 将数据分三组L，M，R。
  2. 中间M只含一个元素（pivot），左边一组L中所有元素都不大于(<=)M，右边一组R中的所有元素都不小于(>=)M。
  3. 本次划分完后，分别递归快排左边一组L和右边一组R。

外部排序：
数据规模大，无法一次放入内存进行排序。

使用DEPQ对快排和外排进行结合：

  1. 读入尽可能多的数据放入DEPQ中，并且DEPQ中的数据要逐步更新，最终其中存放的数据就是所有数据的中间组M。
  2. 对DEPQ之外的元素逐个进行处理，若不大于(<=)DEPQ中的最小元素，则元素放入左边一组；若不小于(>=)DEPQ中的最大的元素，则放入右边一组；否则从DEPQ中删除最小（最大）元素，删除的元素归到左（右）边一组中，新元素插入到DEPQ中。
  3. 依序输出DEPQ，该数据为中间一组M。
  4. 递归排序左边一组和右边一组。
