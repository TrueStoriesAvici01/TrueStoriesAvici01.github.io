应用协议

应用层协议概述

应用层协议是为了实现某种应用而设计和创造的协议。

包含：
会话层：管理通信连接
表示层：转换数据格式
应用层：对端主机交互

远程登录

从自己的本地计算机登录到网络另外一端计算功能的应用。

分类：

  • TELNET
  • SSH

TELNET

利用TCP的一条连接，通过这一条连接向主机发送文字命令并在主机上执行。

分类：

  • 仿真终端功能
  • 协商选项机制

流程：

  1. 键入文字命令
  2. 进行行模式或透明模式处理后将进一步中的命令传送给telnetd守护进程
  3. 向Shell发送文字命令
  4. 解释从Shell收到的命令，执行程序，获取结果
  5. 获取从Shell返回的结果
  6. 进行行模式或透明模式等处理后结果返回给TELNET客户端
  7. 显示最终结果

选项
提供交互和协商的功能。

仿真终端（NVT)：

  1. 行模式：以一行命令为一个单位发送给服务端
  2. 透明模式：以一个字符为单位发送给服务端

telnet命令
telnet主机名 TCP端口号

TCP端口号 服务
21 FTP
25 SMTP
110 POP3

SSH

加密的远程登录系统。

功能：

  • 使用更强的认证机制。
  • 可以转发文件。
  • 可以使用端口转发功能。

端口转发
将特定端口号所收到的消息转发到特定的IP地址和端口号。

示例：
客户端连接POP3服务器。
SSH客户端程序，SSH服务器相当于网关。

  • 邮件程序使用一般TCP通信连接SSH客户端。
  • SSH客户端通过SSH加密通信转发给SSH服务端程序。
  • SSH服务端程序使用一般TCP通信连接POP3服务端程序。

文件传输

FTP工作机制
使用两条TCP连接：

  1. 用于控制
  2. 用于数据传输

控制：

  • 登录用户名和密码的验证，发送文件的名称，发送方式…
  • 通过ASCII吗字符串发送请求和接收应答。
  • 使用的为TCP21号端口。
  • 数据传送完毕后传输数据的TCP断开。
  • 通常数据传输的TCP和控制的连接按照相反的方向建立。
  • 控制用的连接保持连接，直到没有新的命令一段时间。
  • 使用端口20进行数据传输。

电子邮件

工作机制

SMTP(Simple Mail Transfer Protocol)：

  • 提供电子邮件服务的协议为。
  • 使用TCP协议。

邮件服务器

  1. 发送端将邮件发送给服务器。
  2. 由DNS的MX记录，发送邮件给对方邮件服务器。
  3. 根据邮件软件的设置，从邮件服务器接收邮件。

邮件地址

MX记录
电子邮件的发送地址右DNS进行管理，DNS中注册有邮件地址及其发送地址对应的邮件服务器的域名。

MIME

电子邮件所能发送的数据格式扩展成MIME(Multipurpose Internet Mail Extensions)，可以发送多种格式。

组成：

  • 首部
  • 正文

SMTP

  • 发送电子邮件的协议。
  • 使用TCP的25号端口。
  • 建立TCP连接后进行控制和应答以及数据的发送。
  • 指令和应答的最后追加换行指令(CR,LF)。
  • 无法验证发送者，需要使用POP before SMTP或SMTP认证(SMTP Authentication)

POP

  • 用于接收电子邮件的协议。
  • 客户端从POP服务器接收对方发送的邮件。
  • 在客户端和服务器之间建立TCP连接。
  • 使用ASCII吗字符串。
  • 两种应答:
    • 正常：+OK
    • 异常：-ERR

IMAP

接收电子邮件的协议。

  • POP：邮件是由客户端进行管理。
  • IMAP：邮件是由服务器进行管理。

优势：

  1. 不需要下载所有邮件。
  2. 不同计算机可以保持同步。
  3. 适用于异构终端。

WWW

基本概念

  • 访问信息的手段与位置(URI,Uniform Resource Identifier)
  • 信息的表现形式(HTML, HyperText Markup Language)
  • 信息转发(HTTP, HyperText Transfer Protocol)

URI
Uniform Resource Identifier
不局限于表示互联网资源，可标识所有资源的标识码。

HTML
记述Web页的一种语言，指定浏览器中显示格式。
具有纯文本，能附加链接。

XML：使用标签表示一个数据。

HTTP
访问Web网页的URI时。
默认使用80端口。

  • 客户端向服务器80端口建立一个TCP连接。
  • 在TCP连接上进行请求和应答以及数据报文的发送。

版本：

  • HTTP1.0: 每一个命令和应答都触发一个TCP连接的建立与断开。
  • HTTP1.1: 允许一个TCP连接上发送多个命令和应答。

JavaScript
嵌入在HTML的编程语言。
在客户端中执行。
可用于验证客户端输入字符串，操作HTML或XML的逻辑结构，动态显示Web页的内容和页面风格。

CGI
Web服务器调用外部程序时所使用的一种服务器端应用的规范。
客户端输入的数据会传给触发运行的另外一个程序，运行结束后将生成的HTML再返回客户端。
可以返回动态的信息。

Cookie
Web应用为获取用户信息，保存的用户信息。

网络管理

SNMP

  • 在TCP/IP的网络管理中使用SNMP收集必要的信息。
  • 基于UDP/IP的协议。
  • 管理端为管理器，被管理端为代理。
  • SNMPv3中分为：
    • 消息处理
    • 用户安全
    • 访问控制

工作机制：

  • 动作检查：查询请求，信息请求应答
  • 修改设置：设置请求，应答
  • 事件通知：事件通知。

MIB
SNMP中交互的信息是MIB（Management Information Base）
MIB是在树状结构的数据库中为每个项目附加编号的一种信息结构。

RMON
MIB由监控网络中某个设备接口的众多参数构成。
RMON：由监控网络上线路的众多参数构成。
以终端为单位，也可以以协议为单位进行监控。
用于收集网络扩展和变更时期更为有意义的数据。

其他应用层协议

多媒体通信实现技术
需求：可有少许丢包，希望延时较小，注重时延性。

实时多媒体通信多采用UDP，加上H.323和SIP协议，以及RTP协议。

H.323
用于IP网上传输音频，视频的协议。

组件：

  • 终端
  • 网关
  • 网闸
  • 多点控制单元

SIP
Session Initiation Protocol：

  • 解析对方地址
  • 呼叫对方号码
  • 传送的媒体信息进行处理
  • 中断会话和数据转发

RTP
为每个报文附加时间戳和序列号。

P2P
终端或主机不经过服务器直接一对一相互通信。
主机具有客户端和服务端，对等关系提供服务。

LDAP
Lightweight Directory Access Protocol:
访问目录服务的一种协议。
目录服务：
网络中存在的一种提供相关资源的数据库的服务。

格式：

  • 目录树的格式
  • 数据格式
  • 命名规则
  • 目录访问顺序
  • 安全认证
