操作系统概念(一):导论

2020-01-22

导论

操作系统做什么

操作系统(operating system)：管理计算机硬件的程序，为应用程序提供基础，为计算机硬件和计算机用户之间的中介。

计算机系统组成

  • 计算机硬件
  • 操作系统
  • 系统软件与应用程序
  • 用户

或

  • 硬件
  • 软件
  • 数据

硬件：中央处理器，内存，输入/输出设备等。（为系统提供基本的计算资源)
应用程序：规定了使用这些资源的方式从而解决用户的计算问题。
操作系统：控制和协调用户的应用程序对硬件的使用。

设计目的

  • 用户：使用户使用方便。对于大型机，是使资源利用率最大化;对于工作站，是使个人可用性和资源利用率的折中。
  • 系统:资源发配器，管理各种资源，对于冲突的资源请求，决定为各个程序和用户发配资源。

操作系统：一直运行在计算机上的程序（内核），其他程序为系统程序和应用程序

计算机系统组织

计算机系统操作

概述

  • CPU与若干设备控制器通过总线相连。
  • 总线提供对共享内存的访问。
  • 设备控制器负责特定的设备。
  • CPU与设备控制器并发工作，竞争内存周期。
  • 内存控制器协调对内存的同步访问功能。

计算机运行：

  • 运行引导程序(bootstrap program)(初始化程序,位于固件中,初始化系统的所有部分，找到操作系统内核，并装入内存)
  • 操作系统开始执行第一个程序，等待事件发生。

中断(interrupt)

  • 硬件：通过系统总线向CPU发送信号，触发中断。
  • 软件：通过执行系统调用(system call)，触发中断。

中断流程：

  1. 暂停当前任务
  2. 转入中断服务程序开始位置
  3. 中断服务程序执行
  4. CPU重新执行被中断的计算

中断中的控制权转移：
调用通用例程检查中断信息，例程调用相应的中断处理程序。

中断中状态保存：

  • 固定位置保存中断地址
  • 返回系统栈中的地址

存储结构

程序必须装入内存中才能运行。

指令执行周期：

  1. 从内存获取指令
  2. 将指令保存在指令寄存器(instruction register)
  3. 解码指令，从内存获取操作数，将操作数保存在内部寄存器中
  4. 完成对操作数的执行后，结果存回内存
  • 易失存储器(volatile storage)：断电后丢失其中的内容
  • 非易失存储器(nonvoolatile storage):断电不会丢失内容

存储设备层次：
寄存器 <—> 高速缓存 <—> 主存 <—> 电子磁盘 <—> 磁盘 <—> 光盘

I/O结构

  • 一个CPU和多个设备控制器通过共同的总线连接。
  • 设备控制器维护本地缓冲存储器和一组专用的寄存器。
  • 设备控制器负责控制外围设备与本地缓冲存储器之间的数据传递。
  • 设备驱动程序提供设备与操作系统直接的统一接口。

流程：

  1. 驱动程序在设备控制器中装载适当的寄存器。
  2. 设备控制器检查寄存器。
  3. 控制器从设备向本地缓冲区传输数据。
  4. 数据传输完成后通过中断通知驱动程序已完成操作。
  5. 驱动程序返回到操作系统的控制。

直接内存访问(direct memory access,DMA)
设备控制器在本地缓冲和内存之间传送一整块数据，每块只产生一个中断。CPU在设备控制器执行操作时可完成其他工作。

计算机系统体系结构

单处理器系统

有一个主CPU能够执行一个通用指令集。

多处理器系统

并行系统(parallel system)/紧耦合系统(tightly coupled system)：
两个或多个紧密通信的处理器，共享计算机总线，时钟，内存和外设。

优势：

  • 增加吞吐量：N个处理器不能达到单个处理器的N倍。
  • 规模经济：共享外设，存储，节省成本。
  • 增加可靠性：单个处理器故障系统依旧可用。

类型：

  • 非对称处理(asymmetric multiprocessing)：主处理器控制系统，其他处理器或向主处理器要任务或做预先定义的任务。
  • 对称多处理(symmetric multiprocessing,SMP)：所有处理器对等，每个处理器都要完成操作系统中的所有任务。

集群系统

clustered system：将两个或多个独立的系统耦合起来，计算机共享存储并通过局域网连接或内部连接。

目的：
提供高可用性(high availability)，单个或多个系统出错，服务仍然继续。

分类：

  • 非对称集群(asymmetric clustering)：一台机器处于热备份模式(hot-standby mode)，另外一台运行，热备份主机监控活动服务器，若服务器失效，则热备份主机成为活动服务器。
  • 对称模式(symmetric mode)：两个或更多主机都运行应用，相互监视。

并行集群：允许多个主机访问共享存储上的相同的数据，并对文件访问内提供控制和加锁。

操作系统结构

多道程序设计
通过组织作业以使CPU总有一个作业可执行。

实现：

  • 同时将多个任务保存在内存中，作业集为作业池中作业集的子集。
  • 系统从内存中选择一个作业开始执行。
  • 当前作业需要等待另外一个任务时，切换到其他作业。
  • 若等待的作业满足要求，继续执行。

分时系统（多任务系统）
多个用户同时共享计算机，系统在不同用户之间快速切换，为每个用户提供一小部分分时计算机。

名词解释：

  • 进程(process)：装入到内存中并执行的程序。
  • 作业池(job pool)：所有驻留在磁盘中等待发配内存的作业组成。
  • 作业调度(job scheduling)：多个作业调入内存中，系统在作业中做出选择。
  • CPU调度(CPU scheduling)：多个任务需要同时执行，系统从中选择。
  • 虚拟内存(virtual memory)：允许一个执行的任务不必在内存中。

操作系统操作

中断驱动：
没有进程或I/O设备服务，操作系统将等待。事件的发生通过中断或陷阱引起。

陷阱(异常)：
一种软件中断，源于出错或用户程序的特别请求。(中断服务程序用来处理中断)

双重模式操作

目的：区分操作系统代码和用户定义代码，确保操作系统正常执行。

模式:

  1. 用户模式(user mode)
  2. 内核模式(kernel mode)/(管理模式(supervisor mode),系统模式(system mode),特权模式(privileged mode))

模式位(mode bit)：
用来区分为操作系统所执行的任务还是用户所执行的任务。

操作系统运行流程：

  1. 系统引导，硬件处于内核模式。
  2. 装入操作系统，以用户模式执行用户进程。
  3. 出现陷阱或中断，硬件从用户模式切换到内核模式。
  4. 系统将控制权交还给用户程序时切换到用户模式。

指令执行生命周期：

  • 控制发生在操作系统中，指令以内核模式来执行。
  • 控制权转交给一个用户程序序后，转换为用户模式。
  • 通过中断、陷阱或系统调用将控制权返回给操作系统。
  • 系统调用后，硬件将其作为软件中断。
  • 控制权由中断向量转交给操作系统的中断处理程序，转换为内核模式。
  • 内核检查为何种内核模式，请求服务的类型，通过寄存器，堆栈等传递其他信息。
  • 内核检查参数的合法性。
  • 执行请求。
  • 将控制权返还给系统调用之后的指令。

错误处理：

  1. 用户程序出现错误。
  2. 硬件向操作系统发出陷阱。
  3. 通过中断向量将控制权转交给操作系统。
  4. 操作系统进行异常终止。
  5. 给出恰当的出错信息，并将内存转存。

定时器(timer):
目的：
防止用户程序陷入死循环或不调用系统服务，不将控制权返回给操作系统。

实现：
可变定时器(variable timer):固定速率的时钟和计数器实现。每个时钟周期，计数器递减。减0时产生中断。

流程：

  • 设置好定时器
  • 将控制权交给用户
  • 若发生中断，则控制权自动交给操作系统

进程管理

进程：处于执行中的程序。

  • 进程需要一定的资源，在创建进程或执行进程时进行发配。
  • 进程接收各种输入。
  • 进程可执行系统调用。
  • 进程终止时，操作系统收回可再用的资源。

程序计数器(program counter)：明确下一个执行的指令，从而进程的执行是连续的。

组成：

  • 进程是系统工作的单元。
  • 系统由多个进程组成。
  • 由操作系统进程和用户进程构成，可并发执行。

活动：

  • 创建或删除系统和用户进程
  • 挂起和重启进程
  • 进程同步
  • 进程通信
  • 死锁处理机制

内存管理

内存是CPU和I/O能够直接寻址和访问的唯一大容量存储器

与硬盘的关系

  • 数据：通过CPU生成的I/O调用传送到内存中。
  • 指令：指令必须在内存中。

程序执行

  • 先变成绝对地址转入内存
  • 进程通过绝对地址访问内存中断指令和数据
  • 程序终止，内存空间释放

内存管理活动

  • 记录内存中正在执行的部分和执行的主体
  • 决定装入内存中的进程
  • 根据需要分配和释放内存

存储管理

文件
对存储设备的物理属性的抽象。将文件映射到物理介质上，通过存储介质访问文件。

文件系统管理

可以在多种类型的物理介质中存储信息。
文件是由创建者定义的一组相关信息的集合。

大容量存储器管理

为了断电后保存数据，备份内存。
程序存储在硬盘上，执行时才调入内存，在执行时将硬盘作为来源和目的地。

高速缓冲

信息的使用

  • 信息保存在内存中
  • 使用时临时复制到高速缓存中
  • 需要特定信息时，先检查是否在高速缓存中
    • 若在，则直接使用
    • 若不在，使用位于内存中，则复制到高速缓存中
  • 内存为外存的高速缓存，外存数据复制到内存中后使用。

高速缓存一致性(cache coherency):
多个CPU可并发执行，必须确保在一个高速缓存中数值的更新必须反映到所有其他副本所在的高速缓存。

保护和安全

保护：控制进程或用户对计算机系统资源的访问。

安全(security)：确保系统不受外部或内部攻击。

用户标识(user ID,UID)：区分不同的用户，用户登录时鉴别步骤确认ID，将ID与用户的进程和线程关联。

组标识(group identifier)：定义一个祖名和属于该组的用户集。

升级特权(escalate privilege)：获取对一个活动的额外特权。

分布式系统

一组物理上分开的，各个异构的计算机系统通过网络连接在一起，为用户通过各种资源的计算机的集合。

网络操作系统(network operating system)：提供网络的文件共享，允许不同计算机上的进程进行信息交换的通信方法。

计算环境

传统计算

一些联网的PC组成，服务器提供文件和打印服务。公司实现门户以从Web访问内部服务器。

客户机-服务器计算

服务器系统(server system)满足客户机系统(client system)的请求。

分类：

  • 计算服务器系统(compute-server system)：接收用户所发送的请求，将操作结果返回给客户机。
  • 文件服务器系统(file-server system)：客户机创建，更新，访问和删除文件。

对等计算

系统中所有节点是对等的，每个节点可作客户机或服务器。

流程：

  • 节点加入网络时，利用查询服务注册服务。需要服务的节点与集中查询服务联系，确定提供服务的节点，然后与之进行通信。
  • 客户机向所有网络中的其他节点广播服务请求。提供服务的节点响应发出该请求的对等节点。

基于Web的计算

负载平衡器(load balancer)：在一组相似的服务器之间实现负荷发配。
