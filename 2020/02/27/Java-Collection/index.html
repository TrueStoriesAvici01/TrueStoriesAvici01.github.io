Java Collection 接口

[toc]

简介

  • 一个集合(collection)表示一组对象(objects,或称元素),可能允许重复元素,可能有序.
  • Collection接口的实现(implementation)应提供两种构造器:无参(void)的和具有单个参数,类型为Collection的构造器.
  • 对于不支持的操作,可能抛出UnsupportedOperationException.
  • 不同的实现对于操作的对象有不同的约束,不合法的对象可能抛出异常或操作失败.
  • 不同的实现决定各自的同步策略.
  • 某些迭代遍历操作可能由于由于集合中有自引用实例而抛出异常.

方法

返回值 方法 描述
int size() 返回集合中的元素.若超过最大值,则返回最大值
boolean isEmpty() 集合中不包含元素时返回true
boolean contains(Object o) 若集合中包含指定的对象则返回true
Iterator< E > iterator() 返回集合中元素的迭代器
Object[] toArray() 返回一个包含集合中所有元素的数组.返回的数组中的元素是拷贝的新元素
< T > T[] toArray(T[] a) 返回一个包含集合中所有元素的一个数组
boolean add(E e) 确保集合中有特定元素.若新加入指定元素,则返回true;若已存在,则返回false
boolean remove(Object o) 返回true表示成功移出集合中的特定的元素
boolean containsAll(Collection< ? > c) 若集合中包含指定集合中所有的元素,返回true
boolean addAll(Collection<? extends E> c) 将特定集合中所有元素加入到集合中
boolean removeAll(Collection<?> c) 移除集合中指定元素集中的所有元素
default boolean removeIf(Predicate<? super E> filter 将集合中所有满足给定条件的元素移出
boolean retainAll(Collection<?> c 只保留集合中指定元素集的元素
void clear() 移除集合中所有元素
boolean equals(Object o) 若指定的对象与该集合相等,则返回true.一般要重写,实现值比较,否则默认为引用比较.
int hashCode() 返回集合的哈希值.重写equals方法也需重写hashCode,保证equals相等时hashCode页相等
default Spliterator< E > spliterator() 对于集合构造一个分隔器.需要记录分隔器的特征值
default stream< E > stream() 以集合为源,返回一个连续流
default Stream< E > parallelStream() 返回一个可能的并行流
