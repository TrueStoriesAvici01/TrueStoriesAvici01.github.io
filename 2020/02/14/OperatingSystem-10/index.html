操作系统概念(十):文件系统接口

2020-02-14

文件系统接口

[toc]

文件系统:

  • 一组文件:用于存储相关数据.
  • 目录结构:组织系统内的文件并提供有关文件的信息.

文件概念

操作系统将存储设备抽象,定义了逻辑存储单元(文件),再将文件映射到物理设备.

文件属性

常见属性:

  • 名称
  • 标识符:标识文件系统内文件的唯一标签.
  • 类型:被支持不同类型的文件系统所使用.
  • 位置:指向文件位置的指针.
  • 大小:文件当前的大小.
  • 保护:进行读,写,执行等访问控制信息.
  • 时间,日期和用户标识:文件创建,上次修改和上次访问相关信息.

文件操作

创建操作:

  • 创建文件:两个步骤:1.在文件系统为文件找到空间;2.在目录中为新文件创建一个条目.
  • 写文件:执行一个系统调用,指明文件名和要写入文件的内容.
  • 读文件:使用系统调用,指明文件名和读入文件块的内存位置.
  • 在文件中重定位:搜索目录相应条目,设置当前文件位置为给定值.
  • 删除文件:在目录中搜索给定名称的文件,释放文件空间并删除相应目录条目.
  • 截短文件:保留文件属性而将长度设置为0并释放空间.

打开与关闭文件

打开文件表(open-file table):
一个包含所有打开文件的信息表.

对一个文件的操作可通过表的一个索引指定文件.关闭文件使得操作系统从文件表中删除对应条目.

多进程同时打开同一文件:
采用两级内部表:单个进程的表和整个系统的表.

  • 单个进程表:单个进程打开的所有文件.每个条目中指向对应的整个系统打开的文件表.
  • 系统打开文件表:进程打开一个文件,系统打开文件表为打开文件增加相应的条目.还有打开计数器(open count),记录打开该文件的进程数,为0时该条目可从系统打开文件表删除.

打开文件的信息:

  • 文件指针:指向文件的指针.
  • 文件打开计数器:当计数器为0时表示没有进程打开文件,可以关闭文件并删除对应条目.
  • 文件磁盘位置:定位磁盘上文件位置的信息.
  • 访问权限:每个进程用一个访问模式打开文件.

文件类型

文件类型:
通过识别和支持文件类型,并按照合理方式对文件进行操作.

常在文件名内包含类型,可由文件名+圆点+扩展名组成.

文件结构

文件类型可用于表示文件的内部结构.源文件和目标文件具有一定结构,以适应相应处理程序的要求.

支持过多的文件结构导致操作系统变大.

操作系统至少支持一种结构:可执行文件结构,以便能装入和运行程序.

Macintosh操作系统要求每个文件包含两个部分:

  • 资源叉(source fork):包括用户感兴趣的信息.
  • 数据叉(data fork):包括程序代码或数据.

内部文件结构

  • 定义块的大小,其由扇区大小决定.
  • 每个块的大小是相同的.
  • 将若干个逻辑记录打包(packing),再放入物理记录中.
  • 由于按块分配的,最后一个块的部分空间一般没有用上,称为内部碎片(internal fragmentation).

文件加锁

  • 共享锁(shared lock):类似于读者锁.
  • 专用锁(exclusive lock):类似于写者锁.

强制(mandatory):一个进程获得锁后,阻止其他进程访问已加锁的文件.

建议(advisory):需要确保合适地获得和释放锁.

访问方法

顺序访问

顺序访问(sequential access):
文件信息按顺序一一加以处理.

读操作:
读取下一个文件,并自动前移文件指针.

写操作:
向文件尾部增加内容,并将指针移到新增数据之后.

直接访问

直接访问(direct access)/相对访问(relative access):

  • 文件以固定长度的逻辑记录(logical record)组成.
  • 文件作为块或记录的编号序列进行直接读写.

实现:
用户向操作系统提供相对块号(relative block number,相对于文件开始的索引).
设逻辑记录长度为L,记录N的请求可转换为从文件位置 L * N 开始的L字节的请求.

其他访问方式

创建文件索引(index,各块的指针).
为了查找文件中的记录,首先搜索索引,再根据指针直接访问文件,查找所需的记录.

对于大文件使用两层的索引:
初级索引文件包括二级索引文件的指针,二级索引文件包括真正指向数据项的指针.

目录结构

存储结构

分区/片(slice):
一个磁盘中有多个文件系统,每个部分独立用于不同的地方.

卷(volume):
不同的分区组合成更大的结构.

设备目录(device directory)/卷表(volume table of content):
记录卷上所有文件的信息(名称,位置,大小等).

目录概述

目录可看作符号表,将文件名转换成目录条目.

目录相关操作:

  • 搜索文件:搜索目录结构查找特定文件的条目.
  • 创建文件:创建新文件并加入到目录中.
  • 删除文件:将不需要的文件从目录删除.
  • 遍历目录:遍历目录内的文件及其目录中每个文件条目的内容.
  • 重命名文件:改变文件在目录结构中的位置.
  • 跟踪文件系统:访问每个目录和每个目录的每个文件.

单层结构目录

特点:

  • 所有文件都位于同一个目录下.
  • 每个文件必须具有唯一的名称.

双层结构目录

特点:

  • 每个用户都有自己的用户文件目录(user file directory, UFD).
  • 当一个用户作业开始执行时,搜索系统的主文件目录(master file directory, MFD).
  • 不同用户的UFD唯一,但可拥有系统名称的文件.
  • 有效的对用户进行隔离,解决了名称冲突问题,但不利于合作和访问其他文件.
  • 双层结构目录可看作是高度为2的树,树根时MFD,直接后代时UFD,UFD的后代(树叶)为文件本身.
  • 定义一个特殊的用户目录,包含所有系统文件.给定名称文件装入时,操作系统搜索本地UFD,若未找到,则搜索特殊用户目录.

树状结构目录

将目录结构扩展为任意高度的树.

目录包含一组文件和子目录,每个目录是一个按照特定方式访问的文件.

每个目录条目都用一位定义其为文件或子目录.

当前目录(current directory):
需要引用一个文件时,旧搜索当前目录.当所需文件不在时,需要提供路径名.

  • 绝对路径名(absolute path name):从根开始并给出路径上的目录名直到所指定的文件.
  • 相对路径名(relative path name):从当前目录开始定义路径.

删除操作:

  • 若目录为空,则直接删除.
  • 若目录非空,则:
    • 禁止删除,只有删除其中内容时才能删除目录.
    • 删除一个目录时,所有该目录的文件和子目录都被删除.

桌面文件(Desktop File):
包括所看到的所有可执行程序的名称和位置.

无环图目录

背景:
共同的子目录需要共享,共享目录或文件可同时位于文件系统的多处.

无环图(acyclic graph):允许目录含有共享子目录和文件.同一文件或子目录可出现在不同目录中.

共享文件(或子目录)只存在一个文件,任何改变都会其他用户看到.

共享文件或目录的实现:

  • 创建一个新目录条目(链接,link,为另一个文件或目录的).链接可通过使用路径名定位真正的文件从而实现解析.
  • 在共享目录中重复所有被共享文件信息.

问题:

  • 不同的文件名可能表示同一个文件,在遍历文件系统时,不希望多次重复地遍历共享目录.
  • 删除问题:1.删除文件时就删除,可能导致指针指向不存在的文件;2.对于符号链接实现共享,删除链接不影响原文件.若文件条目被删除,则文件空间被释放,链接指针无效;3.保留文件直到删除其所有引用.为每个文件保留引用列表或只保留文件引用的数量,当计数为0时,则删除文件.

通用图目录

对于已存在的树状结构目录增加链接.

问题:

  • 多次检索共享部分.解决:限制搜索时所访问目录的次数.
  • 计数不为0时不一定存在引用(自我引用).解决:使用垃圾回收.

文件系统安装

文件系统被进程使用之前必须被安装(mount).

目录结构建立在多个卷上,卷必须被安装后才能在文件系统命名空间使用.

安装步骤:操作系统获取设备名称和文件系统的安装点(mount pointer).

系统可能不允许在包含文件的目录上安装,或安装的文件系统可见,而之前已存的文件不可见.

系统可允许对同一个文件系统在不同的安装点上进行多次重复安装,或只允许对一个文件系统安装一次.

文件共享

多用户

为了实现共享和保护,操作系统采用了文件/(目录),拥有者和组.

  • 拥有者是目录最高控制权的用户,可改变属性和授权访问.
  • 组属性定义对文件拥有相同权限的用户子集.
  • 组成员和其他用户对文件可进行的操作取决于文件拥有者的定义.

远程文件系统

实现方式:

  1. 通过ftp,实现匿名访问(anonymous)和验证访问,从而在不同机器之间进行文件的人工传输.
  2. 通过分布式文件系统(distributed file system,DFS),在本机上直接访问远程目录.
  3. 通过WWW(World Wide Web),浏览器获取对远程文件的访问.

客户机-服务器模型

  • 包含文件的机器为服务器,需要访问文件的机器为客户机.
  • 客户机通过网络名称或其他标识符来指定.
  • 文件操作请求会代表用户通过网络按照DFS发送给服务器.
  • 文件打开请求与请求的用户ID一起发送.
  • 服务器使用访问检查确定用户是否有权按所需模式访问文件.

分布式信息系统

  • 分布式信息系统(distributed information system)/分布式命名服务(distributed naming service):提供用于远程计算所需信息的统一访问.
  • 域名系统(domain name system, DNS):提供主机名到网络地址的转换.
  • 网络信息服务(network information service, NIS):将用户名,主机名等信息加以集中管理.
  • 公共Internet文件系统(common Internet file system,CIFS):网络信息和用户验证信息用来创建网络登录.服务器用来确定允许或拒绝请求.
  • 轻量级目录存取协议(lightweight directory access protocol, LDAP):目录存储企业内所有计算机的所有用户和资源.用户一次验证,可访问企业所有计算机.

故障模式

本地文件系统出错原因:

  • 磁盘出错
  • 目录结构和磁盘管理信息损坏
  • 磁盘控制器故障
  • 电缆故障
  • 用户或管理员错误
  • 目录或分区被删除

远程文件系统故障处理:

  • 终止(或延迟)对故障服务器的操作.
  • 通过状态信息(state information,维护当前活动和打开的文件)从故障恢复.
  • 可通过无状态(stateless)DFS实现恢复(只有打开文件才能请求文件的读写).

一致性语义

一致性语义(consistency semantics):多用户同时访问共享文件时的语义.

UNIX语义

  • 一个用户对打开文件的写操作可被打开文件的其他用户看到.
  • 允许用户共享文件当前指针的位置.

会话语义

  • 用户对打开文件的写不能被打开同文件的其他用户看到.
  • 一旦文件关闭,其修改只能被之后打开的用户看到,已打开的用户不能看到.

不可修改共享文件语义(immutable shared file)

  • 共享的文件不可重用,文件内容不可修改.

保护

访问类型

控制访问(contolled access):
限制可进行的文件访问类型.

  • 读(read)
  • 写(write)
  • 添加(append)
  • 删除(delete)
  • 列表清单(list)

访问控制

访问控制列表(access-control list,ACL):
每个文件和目录增加ACL,给定每个用户名及其所允许的访问类型.

改进:每个文件采用三种用户类型

  • 拥有者(owner):文件的创建者
  • 组(group):对共享文件进行访问的用户组成
  • 其他(universe):所有其他用户

其他保护方式

保护:

  • 对每个文件加密.
  • 对多层目录结构,还需要保护子目录内文件.
  • 需要控制一个用户能确定一个目录是否有一个文件存在.
