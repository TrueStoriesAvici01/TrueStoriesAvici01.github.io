进程的概念和特征

进程的概念

进程控制块(Process Control Block,PCB)：描述进程的基本情况和运行状态。

进程：由程序段，数据段和进程控制块组成。

创建进程：创建进程的PCB；撤销进程：撤销进程的PCB。

PCB是进程存在的唯一标志。

进程的定义：

  • 进程是程序的一次执行过程。
  • 进程是程序和数据在处理机上顺序执行时所发生的活动。
  • 进程是程序运行过程，是系统进行资源分配和调度的一个独立单位。

进程的特征

  1. 动态性：具有创建、活动、暂停、终止过程，是动态地产生，变化和消亡的。
  2. 并发性：多个进程同时存在与内存中，在一段时间内同时运行。
  3. 独立性：能独立运行，获得资源和接受调度的单位。
  4. 异步性：进程各自独立，一不可预测的速度执行。
  5. 结构性：进程是由程序段，数据段和进程控制块组成。

进程的状态和转换

进程的状态

主要有五种状态：

  1. 创建状态：进程被创建出来。（步骤：申请PCB，将PCB写入进程相关信息，系统为进程分配必须的资源）
  2. 就绪状态：进程处于准备状态，若获得CPU，则可以运行。
  3. 运行状态：进程正在CPU上运行。单核状态下，每个时刻最多只有一个进程处于运行。
  4. 阻塞状态（等待状态）：进程由于等待某个事件而暂停（等待资源可用，等待输入…）。（不可获得CPU）
  5. 结束状态：进程执行完成或者被中断而退出运行。（进程若要结束，先设置结束状态，再进行资源释放和回收）

就绪状态与阻塞状态的区别：就绪可以获得时间片，只要获得获得时间片，就进入运行状态；阻塞状态：不可获得时间片，其等待其他事件，需要其他事件完成后，将其转换到就绪状态等待时间片。

进程的状态转换

解释：

  • 就绪—>运行：就绪状态的进程被调度，获得时间片。
  • 运行—>就绪：进程的时间片用完而为执行完成；或者在可剥夺资源的操作系统中，有更高优先级进程就绪，则挂起当前的进程，让高优先级进程执行。
  • 运行—>阻塞：进程请求某种资源或等待某种事件，进程有运行进入阻塞。此时CPU不会再分配时间片给进程。
  • 阻塞—>就绪：进程等待的资源或事件到来后，进程的状态由阻塞转换为就绪。

进程的控制

原语:进程控制用的程序段.

特点:执行期间不允许中断,是不可分割的基本单位.

进程的创建

  • 父进程创建子进程.
  • 子进程继承父进程拥有的资源.
  • 子进程撤销时,归还从父进程获得的资源.
  • 父进程撤销时,撤销其所有子进程.

创建进程的过程:

  1. 为进程分配唯一的进程标识号,申请PCB.
  2. 为进程分配所需的资源.(若资源不足,则处于”等待状态”)
  3. 初始化PCB.(初始化标志信息,处理机状态,进程优先级等)
  4. 将进程插入就绪队列,等待调度.

进程的终止

终止的原因:

  • 正常结束.(任务完成)
  • 异常结束.(存储区越界,保护错,非法指令,特权指令错)
  • 外界干扰.(系统干预,父进程终止)

过程:

  1. 由进程的标识符,检索PCB,获得进程的状态.
  2. 若进程处于运行状态,则终止进程,将资源分配给其他进程.
  3. 若进程有子进程,将子进程终止.
  4. 将进程拥有的资源或者归还给父进程,或者归还给操作系统.
  5. 将PCB从队列中删除.

进程的阻塞和唤醒

阻塞
正在执行的进程由于等待的事件未发生,自行调用阻塞原语(block),使得由运行状态转化为阻塞状态.
注:阻塞是进程自身的主动行为,只有运行状态的进程才能转化为阻塞状态.

过程:

  1. 由进程标识符找到对应的PCB.
  2. 若为运行状态,保护现场,将状态转化到阻塞状态,停止运行.
  3. 将PCB插入到对应事件的等待队列中.

唤醒
阻塞的进程等待的某种事件发生,由相关的进程调用唤醒原语(wakeup),将进程唤醒.

过程:

  1. 找到对应进程的PCB.
  2. 将其从等待队列中移出,状态转为就绪队列.
  3. 将PCB插入就绪队列中,等待调度.

注:block是由被阻塞的进程自我调用实现的,wakeup是由其他进程调用实现的.

进程的切换

从一个进程的运行转换到另一个进程的运行.

过程:

  1. 保存CPU的上下文.(程序计数器和寄存器)
  2. 更新当前进程的PCB信息.
  3. 将进程的PCB移入相应的队列中.(就绪,阻塞队列)
  4. 选择新进程,更新其PCB.
  5. 更新内存管理的数据结构.
  6. 恢复CPU的上下文.

进程的组织

进程时操作系统的资源分配和独立运行的基本单位.

组成:

  • 进程控制块
  • 程序段
  • 数据段

进程控制块(PCB)

PCB包含的内容:
|进程描述信息|进程控制和管理信息|资源分配清单|处理机相关信息|
|—-|—-|—-|—-|
|进程标识符(PID)|进程当前状态|代码段指针|通用寄存器值|
|用户标识符(UID)|进程优先级|数据段指针|地址寄存器值|
||代码运行入口地址|堆栈段指针|控制寄存器值|
||程序的外存地址|文件描述符|标志寄存器值|
||进入内存时间|键盘|状态字|
||处理机占用时间|鼠标||
||信号量使用|||

操作系统通过PCB对进程进行管理和控制.

PCB的组织方式

  • 链接方式:将相同状态的PCB链接成一个队列,不同的状态对应不同的队列.(也可以按照阻塞原因的不同,排成多个阻塞队列.)
  • 索引方式:将同一状态的进程组织在一个索引表中,索引表项指向相应的PCB.(不同状态对应不同的索引表,如就绪索引表和阻塞索引表)

程序段

可以被多个进程共享,多个程序运行同一个程序.

数据段

进程的中间数据或结果.

进程的通信

进程通信:进程之间的信息交换.

分类:

  • 共享内存
  • 消息传递
  • 管道通信

共享内存

存在一块可以直接访问的共享空间,通过对共享空间进行读/写操作实现进程之间的信息交换.
在对共享空间进行读写操作时,需要使用同步互斥工具(P,V操作)对共享空间进行控制.

特点:

  • 效率高:只拷贝两次数据(一次从输入文件到共享内存区,一次从共享内存区到输出文件).
  • 进程间使用共享内存时,会一直保持共享内存区,直到通信结束.共享内存中的内容是解除映射时写回文件的.
  • 设置好共享内存区后,不再涉及到内核的系统调用.

使用共享内存的方式:

  1. 服务器使用信号量获取某个共享内存对象.
  2. 服务器将数据从输入文件读到共享内存区对象.
  3. 服务器读入完毕后使用信号量通知客户.
  4. 客户将这些数据从该共享内存区对象写到输出文件中.

消息传递

  • 进程间的数据交换通过格式化的消息实现的.
  • 利用操作系统提供的消息传递方法实现进程通信.

分类:

  • 直接通信方式
  • 间接通信方式

直接消息传递
发送进程直接将消息发送给接收进程,并将消息挂在接收进程的消息缓冲队列中,接收进程从消息缓冲队列中取得消息.

若没有缓冲机制,则发送进程执行send,发送进程阻塞,直到接收进程执行receive进行接收.

间接通信方式
发送进程将消息发送到信箱,接收进程从信箱取得消息.
也称为信箱通信方式.

过程:
发送进程:
检查信箱是否已满,若已满,则发送进程阻塞;若未满,则将信息存入信箱,若有进程等待取信,则唤醒正在等待信件的进程.

接收进程:
检查信箱是否有信件,若有,则取出信件.若有进程等待将信件放入信箱,则唤醒该进程;若没有信件,则接收进程阻塞.

管道通信

管道:用于连接读进程和写进程以实现进程间通信的一个共享文件(pipe).
写进程以字符流形式向管道送入数据,读进程从管道接收数据.

特点:

  1. 为半双工的,数据只能向一个方向流动;若需要双方通信,则要建立两个管道.
  2. 匿名管道只能用于父子进程或者兄弟进程之间通信.
  3. 单独构成文件系统:管道对于两端的进程是一个文件,其单独构成一个文件系统,存在于内存中.

分类:

  • 无名管道(pipe):用于相关进程(父子进程)之间通信,通过pipe()创建打开,最后使用的进程关闭.
  • 命名管道(fifo):在磁盘上有对应的节点,没有数据块.通过mknode()系统调用建立.任何进程都可以打开和读写操作.

管道是一个内核缓冲区,进程以先进先出从缓冲区取出数据.缓冲区可以看做一个循环读列,每个数据只能被读一次.

线程的概念和多线程模型

线程的基本概念

引入线程,减少并发执行所付出的时空开销,提高操作系统的并发性能.

特点:

  • 线程时程序执行的最小单位.
  • 组成:线程ID,程序计数器,寄存器集合和堆栈.
  • 线程是系统独立调度和分派的基本单位.
  • 线程与同一进程的其他线程共享进程拥有的资源.
  • 线程可以创建或销毁另一个线程.
  • 多个线程并发执行.
  • 线程有就绪,阻塞和运行三种状态.

进程作为系统资源分配的单元,则线程作为CPU分配单元.

线程与进程的比较

进程 线程
调度 拥有资源和独立调度的基本单位 线程:独立调度的基本单位,进程:资源拥有的基本单位
拥有资源 进程拥有资源 进程拥有资源,线程访问所属进程的资源
并发性 进程间可以并发 进程间可以并发,线程间也可以并发
系统开销 创建或撤销进程要分配或回收资源,进程间切换需要CPU环境的保存和新进程CPU环境的设置 线程切换只需保存和设置少量寄存器,同一进程的多个线程共享地址空间,线程间同步和通信容易
地址空间和其他资源 进程间地址空间相互独立 同一进程的线程共享进程资源
通信 需要进程同步和互斥实现 线程间可直接读/写进程数据段进行通信

线程的属性

属性:

  1. 每个线程有一个唯一的标识符和线程控制块(记录线程执行的寄存器和栈等现场状态).
  2. 不同的线程可以执行相同的程序.
  3. 同一进程的各个线程共享该进程所拥有的资源.
  4. 多个线程可以并发执行.
  5. 线程的生命周期经历阻塞态,就绪态,运行态等状态变化.

线程的实现方式

线程的分类：
|名称|描述|
|—-|—-|
|用户级线程(User-Level Thread,ULT)|由应用程序支持的线程实现，对内核不可见|
|内核级线程(Kernel-Level Thread,KLT)|内核支持的线程|

组合线程(Hybrid Multithreading):将用户线程和内核线程的组合。

用户级线程

  • 线程的管理由应用程序完成，内核不知道多线程的存在。
  • 由应用程序的一个线程通过线程库生成新线程，切换速度快。
  • 线程不具有线程上下文，无法被调度到内核。
  • 任意时刻每个进程上只有一个线程运行，只有一个处理器内核分配给该进程。
  • 特点：同一进程的多个线程对应于一个内核线程，任一时刻只有其中的一个线程与内核线程对应。

优点：

  • 可在不支持线程的操作系统上实现。
  • 创建，切换，销毁线程的代价低。
  • 线程管理算法灵活。
  • 不需要上下文切换，线程调用快捷。
  • 不需要内核调度，控制简单。

缺点：

  • 其中一个线程发生阻塞时，系统阻塞整个进程从而阻塞所有线程。
  • 进程内部没有时钟中断，不能用轮转调度方法。
  • 同一进程下的所有线程无法并行。

内核级线程

  • 线程的创建和销毁在内核下完成的，由操作系统负责管理，通过系统调用完成。
  • 每个进程中的所有线程都有与之对应的内核线程，内核为每个线程维护上下文信息。
  • 内核线程为每个对应的线程设置一个线程控制块(TCB)，内核根据控制块感知线程的存在，并进行控制。

优点：

  • 多处理器系统中，同一进程的多个线程可以并行执行。
  • 进程的一个线程被阻塞，不影响同一进程中的其他线程的执行。

缺点：

  • 线程的调度都是系统调用，代价高。
  • 信号一般是发送给进程的，需要使用订阅发布模型。

用户级线程和内核级线程的区别

  • 内核级线程是内核可以感知的，而用户级线程内核不可感知。
  • 内核级线程创建，撤销和调度线程需要内核提供支持，而用户级线程不需要内核支持。
  • 用户级线程执行系统调用会导致所属进程被中断，而内核级线程只会中断该线程。
  • 用户级线程内，CPU调度以进程为单位，同一进程的多个线程是由用户程序控制运行；内核级线程，CPU调度是以线程为单位，内核负责线程的调度。
  • 用户级线程是运行在用户态下，而内核级线程可以运行在任何状态下。

组合方式

  • 线程的创建是由用户程序完成的，用户程序线程的调度和同步是由用户程序完成的。
  • 同一个进程的多个用户级线程映射到少量(小于等于用户级线程个数)的内核级线程上。
