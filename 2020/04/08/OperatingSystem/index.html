计算机操作系统概述

基本概念

操作系统(Operating System, OS)

  • 控制和管理计算机硬件和软件。
  • 组织调度计算机工作和资源的分配。
  • 提供给用户和其他软件的接口和环境的程序集合。

特征

  • 并发
  • 共享
  • 虚拟
  • 异步

并发(Concurrence)

  • 并发：多个事件同一时间间隔发生。
  • 系统同时存在多个运行的程序。
  • 在一段时间内，宏观上多道程序同时执行。在某一时刻，单处理机上只有一道程序执行。微观上为分时交替执行。
  • 并行：系统具有同时运算或操作的特性。同一时刻完成多种工作。

共享(Sharing)

资源供内存中多个并发的进程共同使用。

方式：

  1. 互斥共享方式
  2. 同时访问方式

互斥共享方法

  • 一段时间内只允许一个进程访问该资源。
  • 当其他进程访问正在被使用的资源时，必须等待至进程释放资源后才允许访问。
  • 该类资源被称为临界资源，有栈，变量，表格等。

同时访问方式

  • 允许一段时间内多个进程“同时”对其进行访问。
  • 微观上，多个进程分时交替对资源进行访问。
  • 具体有：磁盘设备，重入码编写的文件。

注：

  • 共享以并发为条件，不允许并发执行，则无法资源共享。
  • 若不能对共享进行管理，则无法并发执行。

虚拟(Virtual)

  • 虚拟：把一个物理上的实体变成若干个逻辑上的对应物。
  • 操作系统实现了：虚拟处理器，虚拟内存，虚拟外部设备。
  • 虚拟技术：时分复用技术（处理器的时分共享）；空分复用技术（虚拟存储器）

异步(Asynchronism)

异步性：多个程序并发执行，资源的有限导致进程并非一直执行，而是以不可预料的速度向前推进。

操作系统的目标和功能

操作系统作为计算机系统资源的管理者

  1. 处理机管理：就是对进程的管理：进程控制，进程同步，进程通信，死锁处理，处理机调度。
  2. 存储器管理：内存分配，地址映射，内存保护与共享，内存扩充。
  3. 文件管理：文件存储空间的管理，目录管理及文件读写管理和保护。
  4. 设备管理：缓冲管理，设备分配，设备处理和虚拟设备。

操作系统作为计算机硬件系统之间的接口

操作系统提供两种接口：

  • 命令接口（控制作业的执行）
  • 程序接口（请求操作系统服务）

命令接口

  • 联机命令接口（交互命令接口）：适用于分时或实时系统的接口。
  • 脱机命令接口（批处理命令接口）：适用于批处理系统。

程序接口

  • 由一组系统调用命令组成。
  • 使用系统调用请求操作系统为其提供服务。

操作系统的发展与分类

手工操作阶段

所有工作需要人工干预。

批处理阶段

单道批处理系统

  • 作业的处理是成批进行的。
  • 内存中始终保持一道作业。

特征：

  • 自动性：自动逐个依次运行。
  • 顺序性：作业执行的顺序和进入内存的顺序一致。
  • 单道性：内存中只有一个程序运行。

多道批处理系统

  • 多个程序同时进入内存并运行。
  • 多个程序共享系统的软硬件资源。

特点：

  • 多道：内存存放多道程序。
  • 宏观上并行：多道程序都处于运行状态。
  • 微观上串行：内存中多道程序轮流占用CPU执行。
  • 资源利用率高，系统吞吐量大，但是用户响应时间长。

分时操作系统

  • 把处理器的运行时间分成很短的时间片。
  • 作业分配给时间片内不能完成计算，作业暂停运行。处理器分配给其他作业。
  • 多道批处理：作业自动控制无需人工干预；分时系统：人机交互。

特征：

  • 同时性（多路性）：多个用户同时使用一台计算机。
  • 交互性：用户通过终端直接控制程序运行。
  • 独立性：多个用户间彼此独立进行操作。
  • 及时性：应用请求在很短时间内获得响应。

实时操作系统

硬实时系统：提供绝对保证，某个特定动作在规定时间内完成。
软实时系统：接受偶尔违反时间规定。

特点：

  • 实时性
  • 可靠性

网络操作系统和分布式计算机系统

网络操作系统：计算机网络中各台计算机有机地结合起来。
特点：网络中各种资源的共享以及各计算机之间的通信。

分布式计算机系统：

  • 任意两台计算机通过通信方式交换信息。
  • 每台计算机具有同等地位。
  • 每台计算机资源为所有用户共享。
  • 计算机间并行工作，协同完成。

特点：

  • 分布性
  • 并行性

个人计算机操作系统

目前应用最广泛。

操作系统的运行机制

程序分类：

  • 操作系统内核程序（核心态，管态）
  • 应用程序（用户态，目态）

操作系统内核：

  1. 时钟管理：
    • 提供标准系统时间；
    • 通过时钟中断实现进程切换；
    • 采用时间片轮转调度实现分时系统。
  2. 中断机制：
    • 键盘鼠标信息的输入；
    • 进程的管理和调度；
    • 系统功能的调用；
    • 设备驱动；
    • 文件访问；
  3. 原语：
    • 特点：
      • 处于操作系统底层；
      • 运行具有原子性；
      • 运行时间短，调用频繁；
    • 具有以上特点的程序称为原语(Atomic Operaton)；
    • 原语的特征：关闭中断；
  4. 系统控制的数据结构及处理：
    • 基本操作：
      • 进程管理：进程状态管理，进程调度和分派，创建和撤销进程控制块；
      • 存储器管理：存储器空间的分配和回收，内存信息保护等。
      • 设备管理：缓冲区管理，设备分配和回收。

中断和异常

  • 操作系统内核工作在内核态。
  • 用户程序工作在用户态。
  • 通过中断和异常从用户态进入内核态。

中断(Interruption)/外中断

  • CPU执行指令以外的事件的发生。
  • 通常是与当前程序无关的事件。
  • 如：设备发出I/O结束中断；时钟中断表示固定的时间片已到。

异常(Exception)/内中断/例外/陷入(Trap)

  • 源自CPU执行指令内部的事件。
  • 异常不能被屏蔽，需立即处理。
  • 如：非法操作码，地址越界，算术溢出，陷入指令等。

系统调用

系统调用：用户程序调用操作系统提供的一些子功能。

功能分类：

  • 设备管理：设备请求和释放，设备启动。
  • 文件管理：文件的读写，创建和删除。
  • 进程控制：进程的创建，撤销和唤醒。
  • 进程通信：进程间的消息传递或信号传递。
  • 内存管理：内存的分配，回收，获取作业占用内存大小及起始地址。

用户态—>核心态：

  • 系统调用
  • 发生中断
  • 用户程序发生错误状态
  • 用户程序执行特权指令

操作系统的体系结构

大内核

操作系统的主要功能模块作为一个紧密联系的整体运行在核心态。

微内核

将内核中最基本的功能保留在内核，将不需要在核心态执行的功能移到用户态执行。
移出内核后划分成若干服务程序，相互独立。
问题：频繁在核心态和用户态之间切换，执行开销大。

小结

并行性：多个事件在同一时刻发生。
并发性：多个事件在同一时间间隔内发生。

特权指令：具有特殊权限的指令，使用不当导致系统崩溃。
非特权指令：用户态下可以使用的指令。用户态使用特权指令会产生中断阻止使用特权指令。

访管指令：可以在用户态下执行的指令。
在用户态下使用访管指令请求系统服务，产生访管中断，转换为核心态。
