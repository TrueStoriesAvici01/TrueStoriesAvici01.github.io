内存管理的概念

内存管理的功能

  • 内存空间的分配与回收
  • 地址转换:程序中的逻辑地址与内存中的物理地址不一致,需要将逻辑地址转换为对应的物理地址.
  • 内存空间的扩充:利用虚拟存储技术或自动覆盖技术,从逻辑上扩充内存.
  • 存储保护:保障各个作业不互干扰.

程序装入和链接

将程序变为内存中的进程，需要经过如下步骤：

  1. 编译：将源代码编译成若干目标模块。
  2. 链接：将目标模块和库函数链接在一起，形成完整的装入模块。
  3. 装入：将装入模块装入内存运行。

程序链接方式：

  • 静态链接：运行前，将目标模块和库函数链接成一个完整的程序。
  • 装入时动态链接：装入内存时，采用边装入边链接方式。
  • 运行时动态链接：在程序执行时需要目标模块时才进行链接。

程序装入方式：

  • 绝对装入：编译程序产生绝对地址。程序装入内存时，其逻辑地址和实际内存地址相同。
  • 可重定位装入：目标模块的起始地址从0开始，模块中的其他地址都是相对于起始地址的。程序装入内存中时，对地址进行修改，称为重定位。一般一次装入完成，称为静态重定位。
  • 动态重定位：动态运行时装入。模块装入内存中不将相对地址转换为绝对地址，而是等执行时转换。需要重定位寄存器。可以将程序分配到不连续的存储区。

逻辑地址空间和物理地址空间

逻辑地址空间：
链接后，将多个模块链接成一个完整的目标程序。各模块按照各自顺序构成同一的从0开始编码的逻辑地址空间。不同的程序可以有相同的逻辑地址，因为可以映射到不同的内存位置。

物理地址空间：
内存中的物理单元的集合。是地址转换的最终地址。运行时通过地址转换将逻辑地址转换为物理地址，称为地址重定向。

内存保护

通过重定位寄存器和界地址寄存器实现。

  • 重定位寄存器：程序的最小物理地址值。
  • 界地址寄存器：逻辑地址值。

程序执行前会初始化重定位寄存器和界地址寄存器。
程序的每个逻辑地址都和两个寄存器进行比较，保证没有发生地址越界后获得物理地址，交由内存。

内存覆盖与内存交换

扩充内存的方式：

  • 内存覆盖
  • 内存交换

内存覆盖

  • 将用户空间分成固定区和覆盖区。
  • 经常活跃的部分放在固定区，其余部分按照调用先后顺序放入覆盖区，其他部分放在外存中。调用前从外存中调入覆盖区。

内存交换

换出：将等待状态的程序从内存移到外存。
换入：将准备运行的程序从外存移到内存。

实现：
进程的时间片用完时，将该进程换出，将另外有个进程换入当前进程的内存空间。并将时间片分配给内存中的进程。

注：

  • 需要备份存储，提供对内存映像的直接访问。
  • 需要执行时间大于交换时间。交换时间受到转移时间影响，从而受限于交换的内存空间大小。
  • 换出进程必须为空闲状态。
  • 交换一般在内存不足时启动，负载降低时停用。

交换与覆盖

  • 交换：不同进程之间进行。
  • 覆盖：同一进程内。

内存连续分配管理方式

连续分配方式：分配一个连续的内存空间。

分类：

  • 单一连续分配
  • 固定分区分配
  • 动态分区分配

单一连续分配

内存分为：

  • 系统区（操作系统使用，位于低地址）
  • 用户区（为用户提供）

优点：

  • 无外部碎片
  • 简单
    缺点：
  • 适用于单用户单任务
  • 有内部碎片
  • 存储空间利用率低

固定分区分配

  • 将内存分成若干个固定大小的分区，每个分区只装一个作业。
  • 有空闲分区则从外存的后备作业队列中选择合适大小的作业装入。

分区的划分：

  • 分区大小相同
  • 分区大小不同：（大量小分区，适量中分区，少量大分区）

分区说明表
包含：

  • 起始地址
  • 分区大小
  • 分区状态（是否已分配）

问题：

  • 程序太大无法放入任一分区
  • 程序小于分区大小，分区内部空间浪费（内部碎片）
  • 不能实现多进程共享主存区，空间利用率低。

动态分区分配(可变分区分配)

特点：

  • 不预先划分分区，等进程装入内存，由进程的大小倒台地建立分区。

问题：

  • 在内存中产生大量碎片，内存利用率下降（外部碎片）
  • 利用紧凑(Compaction)技术解决，需要动态重定位寄存器支持，费时。

动态分区分配策略

  • 首次适应算法(First Fit)：将空闲分区以地址递增的次序链接，分配内存时顺序查找第一个满足大小的空闲分区。
  • 最佳适应算法(Best Fit)：空闲分区按容量递增形成分区链，找到第一个满足要求的空闲分区。
  • 最坏适应算法(Worst Fit)/(最大适应算法,Largest Fit)：空闲分区以容量递减次序链接，找到第一个满足要求的空闲分区。
  • 邻近适应算法(Next Fit)/循环首次适应算法：类似于首次适应算法，分配内存是从上次查找结束开始继续查找。

策略比较：

  • 首次适应算法一般是最好和最快的。
  • 首次适应算法可能导致低地址出现大量小空闲分区。
  • 邻近适应算法会导致内存末尾分配时产生小碎片。
  • 最佳适应算法会导致最多的外部碎片。
  • 最坏适应算法导致没有可用的大内存块，性能差。

内存非连续分配管理方式

非连续分配:
将一个程序装入不相邻的内存分区.

分类:

  • 分页存储管理
  • 分段存储管理

分页存储管理方式分类:

  • 基本分页存储管理方式
  • 请求分页存储管理方式

基本分页存储管理方式

思想:

  • 将主存划分成大小相等且固定的块,作为主存的基本单位.
  • 每个进程以块为单位进行划分,执行时,以块为单位逐个申请主存中的块.

注:块的大小较小,进程只有为最后一个块申请空间时才产生内部碎片,相对而言很小.

基本概念

  • 页(Page):进程中的块.
  • 页框/页帧(Page Frame):内存中的块.
  • 块(Block):外存中同样大小的分区.
  • 地址结构:由两部分组成(页内偏移量W和页号P).设地址长度为32位,则0-11位为页内地址(页内偏移量);12-31位为页号.
  • 页表:每个进程有一张页表,记录页号对应的物理块号.(页表中每个页号对应一个物理块号,实现页号到物理地址的映射)

页表示意图:

页面的大小应为2的幂次并且大小适中:

  • 太小:页面数过多,页表过长,占用内存.并且增加地址转换开销.
  • 太大:页内碎片增加,降低内存的利用率.

基本地址变换机构

目的:将逻辑地址转换为内存中的物理地址.

实现:
通过页表寄存器(PTR,存放页表在内存的起始地址F和页表长度M).
进程未执行时,页表的起始地址和长度位于进程控制块中;进程开始时,页表起始地址和长度存入页表寄存器.

逻辑地址A转换为物理地址E的过程:(单个页面大小为L,页表存放的起始地址为F,页表长度为M)

  1. 计算页号P和页内偏移量W(P=A/L,W=A%L).
  2. 比较页号P和页表长度M.(若P>=M,则越界中断,否则继续)
  3. 页表中页号为P的项的地址=页表起始地址F+页号P*页表项长度,取出该页表项内容b,其为物理块号.
  4. E=b*L+W,利用获得的物理地址E访问内存.

分页存储管理的地址变换机构:

问题:

  • 每次访存操作都需要进行逻辑地址到物理地址的转换,速度降低.
  • 每个进程都有页表,降低内存利用率.

具有快表的地址变换机构

基本地址变换机构的问题:
存取一条指令至少访问两次内存:一次访问页表(确定物理地址),一次存取指令.

实现:
地址变换机构增设快表(联想寄存器,TLB,并行查找的高速缓冲存储器).
快表:存储若干个经常访问的页表项.

地址变换过程:

  1. 由逻辑地址获得页号送入快表,对比快表中的页号.
  2. 若有匹配的页号,则直接取出对应的页框号,与页内偏移量组成物理地址.
  3. 若没有匹配,则访问主存中的页表,读出对应的页表项并存入快表.(若快表已满,则进行替换)

具有快表的地址变换机构:

两级页表

实现:
对页表也进行地址映射,建立上一级页表,用于存储页表的映射关系.
进程执行时,通过页表的上一级页表获得进程的页表和进程的页面.

示例:
32位系统中,每个页面大小为$2^{12}$.二级页面大小为$2^{10}$,顶级页表大小为$2^{10}$.

查找逻辑地址为0x20021406的内容:

  • 逻辑地址:0x20021406(0010 0000 0000 0010 0100 0000 0110B)
  • 顶级页表:0x80(00 1000 0000B)
  • 二级页表:0x21(00 0010 0001B)
  • 页内偏移量:0x406(0100 0000 0110B)
  1. 首先,查找顶级页表中地址为0x80内容,其为对应的二级页表地址.
  2. 再由二级页表地址,查找该二级页表中第0x21项,其内容为页框的地址.
  3. 在目标页框中由页内偏移量0x406读取对应项的内容.

对于64位操作系统:
设页面大小为4KB($2^12$),页表项大小为8B.
则每个页框存储$2^9$(4KB/8B)个页表项.
则上一级页表字段为9位.
则页表的级数为64=12+9+9+9+9+9+7.

基本分段存储管理方式

分段

特点:

  • 按照用户进程中的自然段划分逻辑分区:如分为:主程序,两个子程序,栈和数据段.
  • 每个段从0开始编码,分配一段连续的地址空间.(段内连续,段间非连续)
  • 逻辑地址由段号S和段内偏移量W组成.

段表

  • 段表:用于将逻辑地址映射到内存空间.
  • 每个段表项对应进程的一个段.
  • 段表项由段号,段长和本段在主存的起始地址组成.

地址变换机构

段表寄存器:存放段表起始地址F和段表长度M.从逻辑地址映射到物理地址.

段表地址映射示意图:

地址变换过程:(逻辑地址A,物理地址E)

  1. 逻辑地址A中取前几位为段号S,后几位为段内偏移量S.
  2. 比较段号S和段表长度M,若S>M,则产生越界中断,否则继续.
  3. 段表中段号S对应的段表表项地址=段表起始地址F+段号S*段表项长度,取出该段表项的前几位得到段C.若段内偏移量W>=C,产生越界中断,否则继续执行.
  4. 取出该段表项中的起始地址b,得物理地址E=b+W.

段的共享与保护

  • 分段系统:两个作业的段表对应的表项指向共享段的同一个物理地址.(防止两个作业同时修改共享段数据)
  • 分页系统:1.存取控制保护;2.地址越界保护.(用段表寄存器中的段表长度与逻辑地址中的段号比较,段表项中的段长和逻辑地址中的段内偏移比较.)

段页式管理方式

地址空间先分成若干个逻辑段,每个段有段号.然后将每个段分成若干个大小固定的页.
内存的分配以存储块为单位.

段页式管理方式:

逻辑地址组成:

  • 段号S
  • 页号P
  • 页内偏移量W

特点:

  • 每个进程都有一张段表,每个分段都有一张页表.
  • 段表表项包括段号,页表长度和页表起始地址.
  • 页表表项包括页号和块号.
  • 段表寄存器中存储段表起始地址F和段表长度M.

地址变换流程:

  • 通过段表查找页表起始地址.
  • 通过页表查找页帧号.
  • 由页帧中和页内偏移量组成物理地址.

虚拟内存的概念，特征以及虚拟内存的实现

传统存储管理方式的特征

  1. 一次性：作业一次装入内存。
  2. 驻留性：作业被装入内存后，一直驻留在内存中。

局部性定理

  1. 时间局部性：程序的某个指令被执行后，该指令可能再次执行。
  2. 空间局部性：程序访问某个存储单元，不久后，其附近的存储单元也将被访问。

虚拟存储器的定义和特征

定义：
程序装入时，将程序的一部分装入内存。
执行时，所访问的数据不在内存时，操作系统将所需的部分调入内存后继续执行。
操作系统将内存中暂时不使用的内容换出内到外存。

特征：

  • 多次性：无需一次将程序全部装入内存，允许分成多次调入内存。
  • 对换性：无需作业常驻内存，运行时换入和换出。
  • 虚拟性：从逻辑上扩充内存。

虚拟内存技术的实现

实现方式：

  • 请求分页存储管理
  • 请求分段存储管理
  • 请求段页存储管理

硬件需求：

  • 一定容量的的内存和外存。
  • 页表(段表)机制。
  • 中断机构。
  • 地址转换机构。

请求分页管理方式实现虚拟内存

请求分页系统：

  • 只将需要的一部分页面调入内存，便启动作业。
  • 作业执行时，所需的页面不存在时，通过调页功能将其调入。并通过置换功能将暂时不需要的页面换出。

页表机制

页表项构成：

  • 页号
  • 物理块号
  • 状态位P：指示是否调入内存
  • 访问字段A：页面被访问次数（或多长时间未访问）
  • 修改位M：指示调入内存后是否修改
  • 外存地址：指示页面在外存的地址

缺页中断机构

流程：

  • 若页面不在内存，产生缺页中断，请求调入所缺的页。
  • 缺页的进程阻塞。
  • 若内存有空闲块，则将调入的页装入该块，修改相应的页表项。
  • 若没有空闲块，则淘汰某页（若被修改，需要写回外存）

地址变换机构

请求分页的地址转换过程：

流程：

  1. 先检索快表
  2. 若找到页，则修改页表访问位，由页表项给出的物理块号和页内地址形成物理地址。
  3. 若没有找到，则去内存查找页表，再对比页表状态位P，查看是否调入内存，未调入则产生缺页中断，请求调入内存。

页面置换算法

内存无空闲空间时，从内存调出一组数据。

算法种类：

  1. 最佳置换算法(OPT)
  2. 先进先出页面置换算法(FIFO)
  3. 最近最久未使用置换算法(LRU)
  4. 时钟置换算法(CLOCK)

最佳置换算法

特点：
选择淘汰的页面是以后永不使用的，或者再最长时间内不再被访问的页面。

示例：

先进先出页面置换算法

特点：
淘汰最早进入内存的页面，即在内存中驻留时间最久的页面。

实现：
将调入内存的页面按先后顺序链接成队列，设置一个指针指向最早的页面。

示例：

问题：
可能出现Belady异常（增加分配的物理块时导致页故障数反而增加）

最近最久未使用置换算法(LRU)

特点：
选择淘汰最近最长时间未访问的页面。

实现：
每个页面设置一个访问字段，记录自上次被访问以来经历过的时间。

示例：

时钟置换算法(CLOCK)/最近未用算法(Not Recently Used,NRU)

特点：

  • 每个帧关联一个附加位（使用位）
  • 某页首次装入内存后被访问时，使用位设为1.
  • 将待替换的候选帧看成一个循环缓冲区。
  • 某一页被替换时，指针指向下一帧。
  • 替换一页时，查找使用位为0的帧，并将遇到的使用位为1的帧设为0。

改进型CLOCK置换算法：增加修改位。

每个帧的状态可分为：

  • 最近未访问，也未修改(u=0,m=0)
  • 最近被访问，但未修改(u=1,m=0)
  • 最近未访问，但已修改(u=0,m=1)
  • 最近被访问，被修改(u=1,m=1)

算法步骤：

  1. 指针扫描缓冲区，对使用位不做修改，选择遇到的第一个(u=0,m=0)的帧进行替换。
  2. 第1步失败，重新扫描，查找第一个(u=0,m=1)的帧，用于替换，跳过的帧的使用位设置为0。
  3. 第2步失败，重新扫描，重复第1步和第2步，直到找到。

页面分配策略

驻留集大小

影响因素：

  • 分配给每个进程的存储量小，可驻留的进程数多，提高处理机效率。
  • 进程的页数少，导致页错误率较高。
  • 给定进程分配更多的空间不会影响错误率。

策略：

  1. 固定分配局部置换：
    • 每个进程分配固定数量的物理块。
    • 发生缺页时，从进程自己在内存的页面中选择一页换出。
    • 难以确定分配的物理块数：太多资源利用率低，太少缺页中断多。
  2. 可变分配全局置换：
    • 每个进程分配一定的物理块。系统也保持一定空闲物理块队列。
    • 缺页时，从空闲物理块队列中取出一个块分配。
  3. 可变分配局部置换：
    • 每个进程分配一定数量的物理块。
    • 缺页时，从该进程在内存的空间中选择一个页面换出。
    • 若频繁缺页，则为该进程分配物理块。若缺页率特别低，则减少分配给该进程的物理块。

调入页面的时机

  1. 预调页策略：将预计在不久之后会被访问的页面预先调入内存。需指出应先调入哪些页。
  2. 请求调页策略：进程运行时需要访问页面不在内存中时发出请求，系统将所需的页面调入内存。（过多的I/O开销）

从何处调入页面

外存分类：

  • 文件区：存放文件，离散分配。
  • 对换区：存放对调页面，采用连续分配。

情况：

  1. 足够的对换区空间：全部从对换区调入所需页面。
  2. 缺少足够的对调区空间：不会修改的文件从文件区调入。没有修改则无需换出。修改的部分换出到对换区。
  3. UNIX方式：未运行的页面从文件区调入。曾经运行又被换出的页面放在对换区。

页面抖动(颠簸)和工作集(驻留集)

页面抖动(颠簸)

抖动（颠簸）：

  • 频繁的页面调度行为。（刚刚换入的页面被换出主存，刚刚换出的页面又要换入主存）
  • 一个进程在换页上用的时间多于执行时间。

原因：进程频繁访问的页面数高于可用的物理页帧数目。

工作集(驻留集)

工作集：某段时间内，进程经常访问的页面集合。

工作集模型原理：

  • 为进程分配大于工作集的物理块。
  • 有空闲的物理块时调入进程。
  • 所有工作集之和大于可用物理块数，则暂停一个进程，将其页面调出后物理块分配给其他进程。
